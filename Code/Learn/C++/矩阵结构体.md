# 矩阵结构体与重载

---

### 1. 运算符重载的基本语法

在 `struct` 内部，重载二进制运算符（有两个操作数的运算符，如 `*`）的通用格式：

```cpp
返回类型 operator 运算符 (const 参数类型& 变量名) const {
    // 逻辑实现
}
```

*   **`operator*`**：这是关键字，告诉编译器你要定义乘法。
*   **`const Matrix& b`**：通过**常量引用**传递参数。这样既不会产生多余的拷贝（省空间/时间），又保证了不会在函数内部意外修改操作数 `b`。
*   **末尾的 `const`**：这是一个非常关键的细节。它表示这个函数**不会修改当前对象（即乘号左边的矩阵 `A`）**。在快速幂逻辑中，这非常安全。

### 2. 矩阵乘法重载的“工业级”实现

我们以你正在写的这道题为例，实现一个既符合语法规范，又具备**性能优化**的矩阵重载：

```cpp
struct Matrix {
    long long v[25][25]; // 根据数据范围，25 足够
    int size;

    // 构造函数：初始化矩阵
    Matrix(int s) : size(s) {
        for (int i = 0; i < size; i++)
            for (int j = 0; j < size; j++) v[i][j] = 0;
    }

    // 重载乘法运算符 *
    Matrix operator*(const Matrix& b) const {
        Matrix res(size); // 创建一个结果矩阵

        for (int i = 0; i < size; i++) {
            for (int k = 0; k < size; k++) { // 注意：k 循环被提到了中间
                
                // 【核心优化 1：剪枝】
                // 如果左矩阵当前项为 0，则这一整行的乘法计算都可以跳过
                if (v[i][k] == 0) continue; 

                for (int j = 0; j < size; j++) {
                    // 【核心优化 2：缓存友好】
                    // 在最内层循环 j，使得我们连续访问 res.v[i][j] 和 b.v[k][j]
                    // 这符合 CPU 缓存的行优先存储，比跨行访问快 5~10 倍
                    res.v[i][j] = (res.v[i][j] + v[i][k] * b.v[k][j]) % 998244353;
                }
            }
        }
        return res;
    }
};
```

---

### 3. 为什么要这样写？（复盘笔记）

#### A. 语义的简化
如果没有重载，你的快速幂代码是这样的：
`res = multiply(res, base);`
有了重载，代码变成了：
`res = res * base;`
这使得你在处理像 $V_n = M^{n-m} \cdot V_m$ 这种数学公式时，代码几乎就是公式的投影。

#### B. 循环顺序的艺术 (i-k-j)
标准的矩阵乘法定义通常是三层循环 $i, j, k$。但在计算机中，**$i, k, j$ 顺序是最快的**。
*   **原理**：C++ 的二维数组在内存中是按行排列的。
*   当我们固定 $i$ 和 $k$，在最内层变动 $j$ 时，地址是连续增加的。
*   CPU 发现你在读连续地址，会提前把后面的数据加载进 **L1/L2 Cache**。如果 $j$ 在内层，速度会起飞。

#### C. 剪枝优化
这道压轴题的矩阵其实是一个**稀疏矩阵**（有很多 0，尤其是杨辉三角部分和 $f$ 的平移部分）。
*   `if (v[i][k] == 0) continue;` 这一行代码在处理大矩阵时，能直接消掉大量的无效计算。

### 4. 延伸：如何重载快速幂？

有了运算符重载，你的快速幂函数可以写得非常通用：

```cpp
Matrix qpow(Matrix a, long long b) {
    Matrix res(a.size);
    // 初始化为单位矩阵 I
    for (int i = 0; i < a.size; i++) res.v[i][i] = 1;

    while (b > 0) {
        if (b & 1) res = res * a; // 就像计算 res = res * a_int 一样自然
        a = a * a;               // 就像计算 a = a * a 一样自然
        b >>= 1;
    }
    return res;
}
```

### 5. 学习笔记总结（可存入 Git）

> **[C++ 语法笔记：矩阵结构体与运算符重载]**
>
> 1. **结构体封装**：将二维数组和其维度 `size` 封装在一起，避免在函数调用时传递复杂的指针。
> 2. **`operator*` 规范**：
>    - 使用 `const` 引用传参，防止拷贝。
>    - 函数声明加 `const`，防止修改操作数。
>    - 内部创建临时 `res` 矩阵并返回，不影响原矩阵。
> 3. **高性能实现 (IKJ 模式)**：
>    - 将 `j` 循环放在最内层，利用 CPU 缓存行。
>    - 引入 `v[i][k] == 0` 判断，实现稀疏矩阵剪枝。
> 4. **复用性**：一旦定义好 `Matrix` 结构体和其 `*` 重载，它就是一个独立的模块。在任何需要矩阵运算（如路径统计、递推求值、动态规划优化）的题目中，都可以直接粘贴使用。
