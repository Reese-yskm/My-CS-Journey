# 一、文件读写到底在干嘛？

你之前写的程序，大多数是：

```text
键盘输入 → 程序处理 → 屏幕输出
```

比如：

```python
name = input()
print(name)
```

但文件读写变成了：

```text
文件内容 → 程序读取 → 程序处理 → 写回文件
```

比如：

```text
input.txt  →  Python程序  →  output.txt
```

这就很有用了。

比如以后你可以做：

```text
错题信息 → 自动生成 Markdown 错题卡
单词列表 → 统计词频
代码文件 → 批量整理格式
LaTeX题目 → 自动生成模板
```

------

# 二、最基本的写文件

先看最小例子：

```python
with open("test.txt", "w", encoding="utf-8") as f:
    f.write("Hello Python!\n")
    f.write("这是我写入文件的第二行。\n")
```

运行后，会在当前目录生成一个：

```text
test.txt
```

里面内容是：

```text
Hello Python!
这是我写入文件的第二行。
```

这里重点是：

```python
open("test.txt", "w", encoding="utf-8")
```

意思是：

```text
打开 test.txt
模式是 w，也就是 write，写入
编码用 utf-8，避免中文乱码
```

------

# 三、为什么要用 with？

你会经常看到这种写法：

```python
with open("test.txt", "w", encoding="utf-8") as f:
    f.write("hello")
```

它等价于一种更原始的写法：

```python
f = open("test.txt", "w", encoding="utf-8")
f.write("hello")
f.close()
```

区别是：

```python
with
```

会帮你自动关闭文件。

所以以后你先记住一句话：

> Python 文件读写，优先使用 `with open(...) as f:`。

------

# 四、写文件的三种常用模式

## 1. `"w"`：覆盖写入

```python
with open("note.txt", "w", encoding="utf-8") as f:
    f.write("第一版内容\n")
```

如果文件不存在，就创建文件。
如果文件已经存在，会清空原来的内容。

所以 `"w"` 有点像：

```text
重新写一个文件
```

------

## 2. `"a"`：追加写入

```python
with open("note.txt", "a", encoding="utf-8") as f:
    f.write("追加一行内容\n")
```

`"a"` 是 append，表示追加。

如果文件原来有内容，不会清空，而是在后面继续写。

适合做：

```text
日记
错题记录
日志
学习打卡记录
```

------

## 3. `"r"`：读取文件

```python
with open("note.txt", "r", encoding="utf-8") as f:
    content = f.read()

print(content)
```

`"r"` 是 read，表示读取。

注意：如果文件不存在，`"r"` 会报错。

------

# 五、读取文件的三种方式

假设 `note.txt` 内容是：

```text
第一行
第二行
第三行
```

## 方式一：一次性全部读取

```python
with open("note.txt", "r", encoding="utf-8") as f:
    content = f.read()

print(content)
```

得到的是一个完整字符串。

适合小文件。

------

## 方式二：按行读取成列表

```python
with open("note.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()

print(lines)
```

得到：

```python
["第一行\n", "第二行\n", "第三行\n"]
```

注意每一行后面的 `\n` 还在。

------

## 方式三：一行一行处理

```python
with open("note.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())
```

这里：

```python
line.strip()
```

是去掉行首行尾的空白字符，最常见的是去掉换行符 `\n`。

这个方式最常用，也最稳。

# 六、你现在最应该掌握的模板

## 模板 1：写入一个 Markdown 文件

```python
title = "格林公式错题"
problem_type = "平面曲线积分"
mistake = "极坐标积分结果写错"
summary = "遇到闭合曲线积分，先检查能否用格林公式。"

content = f"""# {title}

## 题型
{problem_type}

## 我的错误
{mistake}

## 一句话总结
{summary}
"""

with open("problem_note.md", "w", encoding="utf-8") as f:
    f.write(content)
```

这里的：

```python
f"""..."""
```

是格式化字符串，可以把变量嵌进去。

比如：

```python
name = "小王"
print(f"你好，{name}")
```

输出：

```text
你好，小王
```

------

## 模板 2：追加一条记录

```python
title = input("题目名：")
mistake = input("错误原因：")

content = f"""
# {title}

## 错误原因
{mistake}

---
"""

with open("mistakes.md", "a", encoding="utf-8") as f:
    f.write(content)
```

每运行一次，就往 `mistakes.md` 后面追加一条记录。

这已经很接近一个小工具了。

------

## 模板 3：读取文本并统计行数

```python
with open("note.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()

print("行数：", len(lines))
```

------

## 模板 4：读取文本并统计词频

```python
from collections import Counter

with open("article.txt", "r", encoding="utf-8") as f:
    text = f.read()

words = text.split()
cnt = Counter(words)

for word, times in cnt.items():
    print(word, times)
```

这适合英文文本。中文分词先不用管。

# 七、最容易踩的坑

## 坑 1：文件位置找不到

比如你写：

```python
open("note.txt", "r", encoding="utf-8")
```

Python 会去哪里找？

它会去“当前运行目录”找，不一定是你代码文件所在目录。

你在 VS Code 里面看终端，当前目录大概是：

```text
D:\My-CS-Journey\Code\Learn\Python
```

所以 `note.txt` 会默认在这个目录下面。

建议你初学时就把 `.py` 文件和 `.txt/.md` 文件放在同一个文件夹里。

------

## 坑 2：中文乱码

一定加：

```python
encoding="utf-8"
```

不要偷懒。

尤其是你会写中文 Markdown，不加很容易乱码。

------

## 坑 3：忘记换行

```python
f.write("第一行")
f.write("第二行")
```

写出来是：

```text
第一行第二行
```

你要自己加：

```python
f.write("第一行\n")
f.write("第二行\n")
```

------

## 坑 4：`w` 会清空原文件

```python
open("mistakes.md", "w", encoding="utf-8")
```

每运行一次，原内容都会被覆盖。

如果你想“新增一条”，用：

```python
open("mistakes.md", "a", encoding="utf-8")
```

# 九、你现在可以先写任务 1

我建议你先写这个，难度刚好：


```python
learned = input("今天学了什么：")

{problem}

{summary}
```
