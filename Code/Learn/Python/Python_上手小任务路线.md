# Python 上手小任务路线

> 目标：不是刷难题，而是通过一组难度逐步上升的小任务，把 Python 基础语法真正用顺。
>
> 适合对象：已经有 C++ 基础，但刚开始写 Python 时容易“不知道怎么下手”的学习者。

---

# 使用方式

建议顺序完成，不要一上来追求写得很短。

每个任务都按这个流程来：

1. 先用最朴素的 Python 写出来；
2. 跑通样例；
3. 再回头想：这个地方 Python 有没有更自然的写法；
4. 最后把自己卡住的点记录下来。

如果一题卡住超过 15 分钟，可以先看提示，不要硬耗。

# 练习时常见错误

## 1. 忘记冒号

错误：

```python
if x > 0
    print(x)
```

正确：

```python
if x > 0:
    print(x)
```

---

## 2. 缩进混乱

Python 不用大括号，缩进就是结构。

```python
for x in nums:
    if x > 0:
        print(x)
```

---

## 3. input 读到的是字符串

```python
x = input()
```

这里的 `x` 是字符串，不是整数。

需要转换：

```python
x = int(input())
```

---

## 4. range 右端点取不到

```python
range(1, 5)
```

实际是：

```text
1, 2, 3, 4
```

如果要从 1 到 n，要写：

```python
range(1, n + 1)
```

---

## 5. 字典访问不存在的 key 会报错

可能报错：

```python
cnt[x] += 1
```

更稳的写法：

```python
cnt[x] = cnt.get(x, 0) + 1
```

---

## 6. 列表赋值不是复制

```python
a = [1, 2, 3]
b = a
b.append(4)
print(a)
```

此时 `a` 也会变。

如果要复制：

```python
b = a.copy()
```

或：

```python
b = a[:]
```

# 最后建议

你现在学 Python，不需要追求“写得像高手”。

第一阶段只追求三件事：

1. 能写出来；
2. 能跑通；
3. 能看懂自己写的代码。

等这些任务做完一轮，再回头学更 Pythonic 的写法，比如列表推导式、`Counter`、`defaultdict`、异常处理、命令行参数等，就会顺很多。
