
# 队列知识点笔记

队列（Queue）是最基础、也最常用的线性数据结构之一。  
它的核心特点只有一句话：

## 队列是 **先进先出**（FIFO, First In First Out）

也就是说：

- 最早进入队列的元素，最先离开
- 新元素总是在队尾进入
- 删除元素总是在队头进行

---

# 一、队列的基本概念

## 1. 基本术语
- **队头（front）**：删除元素的一端
- **队尾（rear）**：插入元素的一端
- **入队（enqueue）**：在队尾插入元素
- **出队（dequeue）**：从队头删除元素
- **判空**：队列中是否没有元素
- **取队头元素**：查看最早进入但尚未出队的元素

---

## 2. 队列和栈的区别

### 栈（Stack）
- 后进先出（LIFO）

### 队列（Queue）
- 先进先出（FIFO）

所以：

- 栈更适合处理“最近进入的先处理”的问题
- 队列更适合处理“按进入顺序依次处理”的问题

---

# 二、队列的顺序存储实现

顺序队列通常使用数组实现。

假设有数组：

```cpp
int q[MAXN];

再配两个变量：

- `front`：指向队头位置
- `rear`：指向队尾的下一个位置（常见写法之一）

------

## 1. 最朴素的顺序队列

### 基本思想

- 入队：`q[rear++] = x`
- 出队：`front++`

### 模板代码

```cpp
struct Queue {
    int q[1000];
    int front, rear;

    Queue() {
        front = 0;
        rear = 0;
    }

    bool empty() {
        return front == rear;
    }

    void push(int x) {
        q[rear++] = x;
    }

    void pop() {
        if (!empty()) {
            front++;
        }
    }

    int top() {
        return q[front];
    }
};
```

------

## 2. 这种写法的问题：假溢出

比如数组长度是 10：

- 入队 10 个元素
- 再出队 5 个元素

这时前面虽然空了 5 个位置，但如果 `rear` 已经走到数组末尾，继续入队会越界。

也就是说：

## 数组前面明明有空位，但普通顺序队列却不能再利用

这就叫：

## **假溢出**

------

# 三、循环队列

为了解决假溢出，就要引入：

## 循环队列

它的核心思想是：

## 把数组逻辑上看成一个环

当下标走到末尾时，可以回到开头继续使用。

------

## 1. 循环队列的核心技巧

数组长度设为 `N`，下标更新时使用：

```cpp
(next_index) = (current_index + 1) % N
```

这样：

- 如果 `current_index` 还没到末尾，就正常加一
- 如果已经到末尾，就回到 0

------

## 2. 判空和判满

循环队列最关键的是：

### 判空

```cpp
front == rear
```

### 判满

常见写法是**浪费一个位置**，规定：

```cpp
(rear + 1) % N == front
```

为什么要浪费一个位置？

因为如果不浪费一个位置，那么：

- `front == rear`
  既可能表示空，也可能表示满

会冲突。

所以常见做法是：

## 队列最多只放 `N-1` 个元素

这样就能区分：

- 空：`front == rear`
- 满：`(rear + 1) % N == front`

------

## 3. 循环队列模板代码

```cpp
struct CircularQueue {
    int q[1000];
    int front, rear;
    int capacity; // 数组总长度

    CircularQueue() {
        front = 0;
        rear = 0;
        capacity = 1000;
    }

    bool empty() {
        return front == rear;
    }

    bool full() {
        return (rear + 1) % capacity == front;
    }

    void push(int x) {
        if (full()) return;
        q[rear] = x;
        rear = (rear + 1) % capacity;
    }

    void pop() {
        if (empty()) return;
        front = (front + 1) % capacity;
    }

    int top() {
        return q[front];
    }
};
```

------

## 4. 循环队列最容易错的地方

### （1）判满条件写错

不要写成：

```cpp
rear == front
```

那是判空。

------

### （2）忘记取模

比如：

```cpp
rear++;
```

这样就失去了“循环”的意义。

必须写成：

```cpp
rear = (rear + 1) % capacity;
```

------

### （3）容量和实际可存元素数不一样

如果数组长度是 `N`，浪费一个位置时：

## 实际最多只能存 `N-1` 个元素

------

# 四、链式队列

顺序队列的核心问题是：

- 容量固定
- 循环队列需要处理下标和判满

如果用链表实现队列，就会更自然。

## 链式队列的思想：

- 每个元素是一个链表节点
- 队头负责删除
- 队尾负责插入

------

## 1. 结构设计

通常需要：

- `front`：指向队头节点
- `rear`：指向队尾节点

节点定义：

```cpp
struct Node {
    int val;
    Node* next;
};
```

------

## 2. 链式队列模板代码

```cpp
struct Node {
    int val;
    Node* next;
    Node(int x) : val(x), next(nullptr) {}
};

struct LinkedQueue {
    Node* front;
    Node* rear;

    LinkedQueue() {
        front = nullptr;
        rear = nullptr;
    }

    bool empty() {
        return front == nullptr;
    }

    void push(int x) {
        Node* node = new Node(x);
        if (empty()) {
            front = rear = node;
        } else {
            rear->next = node;
            rear = node;
        }
    }

    void pop() {
        if (empty()) return;
        Node* temp = front;
        front = front->next;
        delete temp;
        if (front == nullptr) {
            rear = nullptr;
        }
    }

    int top() {
        return front->val;
    }
};
```

------

## 3. 链式队列的关键理解

### 入队

总是在尾部插入：

```cpp
rear->next = node;
rear = node;
```

### 出队

总是在头部删除：

```cpp
front = front->next;
```

------

## 4. 为什么链式队列不会假溢出

因为链表节点不是连续放在固定数组里的。
只要内存够，就可以继续申请新节点。

所以链式队列没有“数组前面空了但不能用”的问题。

------

# 五、双端队列（Deque）

双端队列是“升级版队列”。

## 定义：

允许在两端都进行插入和删除。

也就是说可以：

- 队头插入
- 队头删除
- 队尾插入
- 队尾删除

------

## 1. 双端队列的作用

它常用于：

- 单调队列
- 滑动窗口问题
- 某些模拟题
- 0-1 BFS（后面图论里会遇到）

------

## 2. STL 中的双端队列

C++ 里直接有：

```cpp
deque<int> dq;
```

常用操作：

```cpp
dq.push_back(x);   // 尾插
dq.push_front(x);  // 头插

dq.pop_back();     // 尾删
dq.pop_front();    // 头删

dq.front();        // 取队头
dq.back();         // 取队尾
dq.empty();        // 判空
dq.size();         // 元素个数
```

------

# 六、队列的典型应用

------

## 1. 层序遍历（树）

层序遍历其实就是：

## 从上到下，一层一层访问节点

为什么天然适合用队列？

因为：

- 根先进入队列
- 访问根后，把左右孩子按顺序放进去
- 再依次处理队头元素

这正好符合先进先出。

### 模板代码

```cpp
vector<vector<int>> levelOrder(TreeNode* root) {
    vector<vector<int>> ans;
    if (root == nullptr) return ans;

    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        int sz = q.size();
        vector<int> level;

        for (int i = 0; i < sz; i++) {
            TreeNode* cur = q.front();
            q.pop();

            level.push_back(cur->val);

            if (cur->left) q.push(cur->left);
            if (cur->right) q.push(cur->right);
        }

        ans.push_back(level);
    }

    return ans;
}
```

------

## 2. BFS（广度优先搜索）

BFS 本质上就是：

## 从起点开始，一层一层向外扩展

为什么必须用队列？

因为你要保证：

- 先扩展距离近的
- 再扩展距离远的

这正是队列的顺序。

### 图 / 网格 BFS 基本模板

```cpp
queue<pair<int, int>> q;
q.push({sx, sy});
vis[sx][sy] = true;

while (!q.empty()) {
    auto [x, y] = q.front();
    q.pop();

    for (int d = 0; d < 4; d++) {
        int nx = x + dx[d];
        int ny = y + dy[d];

        if (合法 && !vis[nx][ny]) {
            vis[nx][ny] = true;
            q.push({nx, ny});
        }
    }
}
```

------

## 3. 维护“时间窗口” / “滑动范围”

比如“最近多少秒的请求数”，就可以：

- 新请求入队
- 队头过期就出队

这是典型的队列模型。

------

# 七、STL 中的 queue

C++ 中直接提供了队列容器适配器：

```cpp
queue<int> q;
```

常用操作：

```cpp
q.push(x);      // 入队
q.pop();        // 出队（不返回值）
q.front();      // 队头元素
q.back();       // 队尾元素
q.empty();      // 判空
q.size();       // 元素数量
```

注意：

## `queue.pop()` 只删除，不返回值

所以通常要先：

```cpp
int x = q.front();
q.pop();
```

# 九、队列部分最容易错的点

## 1. front / rear 含义不统一

写代码前一定先固定你自己的定义：

- `front` 指向队头元素
- `rear` 指向队尾下一个位置

只要定义统一，后面就不会乱。

------

## 2. 循环队列判满条件记错

最常见写法是：

```cpp
(rear + 1) % capacity == front
```

------

## 3. STL queue 的 pop 不返回值

不要写：

```cpp
int x = q.pop(); // 错
```

正确写法：

```cpp
int x = q.front();
q.pop();
```

------

## 4. BFS 忘记及时标记 visited

通常应在入队时就标记：

```cpp
vis[v] = true;
q.push(v);
```

不要等出队再标，不然可能重复入队。
