### 贪心专题学习笔记

### 一、贪心是什么

贪心算法的核心思想是：

在每一步都做出当前看来最优的选择，并希望这些局部最优选择最终能组成全局最优解。

简单说就是：

每一步都选“眼前最合适”的方案。

但是贪心最难的地方在于：

不是所有“看起来眼前最优”的选择都真的能推出全局最优。

所以贪心题真正重要的不是代码，而是：

这个贪心策略为什么是对的？

------

### 二、贪心和 DP 的区别

DP 通常是：

把所有可能状态都记录下来，然后从小问题推到大问题。

贪心通常是：

找到一个局部选择规则，每一步只保留一个最优方向，不再回头。

可以这样对比：

| 类型 | 核心思想           | 是否枚举多个状态 | 难点                   |
| ---- | ------------------ | ---------------- | ---------------------- |
| DP   | 保存子问题答案     | 通常会           | 状态定义、初始化、转移 |
| 贪心 | 每步做局部最优选择 | 通常不会         | 贪心策略正确性证明     |

DP 更像是“全面考虑”。

贪心更像是“找到一个一定不会错的选择规则”。

------

### 三、什么时候可能用贪心

看到题目时，如果有这些特征，可以考虑贪心：

1. 要求最少次数、最多数量、最大收益、最小代价。
2. 每一步选择之后，问题规模会变小。
3. 当前选择一旦确定，后面通常不需要反悔。
4. 题目中有明显的排序依据，比如按右端点、按起点、按差值、按收益、按消耗等。
5. 可以通过“交换论证”说明：如果最优解没有选当前贪心选择，也可以把它换成当前选择且不会变差。

但要注意：

看到“最大”“最小”不一定就是贪心，也可能是 DP。

关键是看能不能找到一个可靠的局部选择规则。

------

### 四、贪心题的标准思考流程

以后做贪心题，可以按这个流程来：

#### 1. 明确目标

先问：

题目到底要最大化什么，或者最小化什么？

例如：

- 最多能安排多少个区间？
- 最少需要多少箭？
- 最少跳几步到终点？
- 最大能获得多少利润？
- 能不能完成一圈？

------

#### 2. 找局部选择

尝试问：

当前这一步，我应该优先选什么？

常见选择标准有：

- 选结束最早的
- 选起点最小的
- 选代价最小的
- 选收益最大的
- 选差值最大的
- 选当前能覆盖最远的
- 选当前最紧迫的

------

#### 3. 检查是否需要排序

很多贪心题的第一步是排序。

常见排序方式：

| 场景           | 常见排序                             |
| -------------- | ------------------------------------ |
| 区间不重叠     | 按右端点升序                         |
| 合并区间       | 按左端点升序                         |
| 用最少箭射气球 | 按右端点升序                         |
| 分发饼干       | 孩子胃口、饼干尺寸都升序             |
| 排队/重建      | 根据题意设计多关键字排序             |
| 任务调度       | 常常按截止时间、收益、持续时间等排序 |

贪心题里，排序往往就是在制造“局部最优的顺序”。

------

#### 4. 想清楚为什么不能反悔

这是贪心最关键的一步。

你要能解释：

为什么我现在做这个选择，不会影响以后拿到最优答案？

常用证明方法：

##### 方法一：交换论证

假设有一个最优解没有采用我的贪心选择。

我尝试把最优解中的某个选择换成我的贪心选择。

如果换完以后答案不变差，说明一定存在一个包含贪心选择的最优解。

因此贪心选择是安全的。

##### 方法二：领先证明

证明贪心算法在每一步之后，都不会比任何其他最优策略更差。

例如跳跃游戏中，贪心维护当前能到达的最远位置。

只要当前最远覆盖范围不落后，最终就不会输。

##### 方法三：局部不可逆证明

证明当前这个选择如果不做，后面再做只会更差，或者失去机会。

比如区间调度中，选择结束最早的区间，可以给后面留下最多空间。

------

### 五、常见贪心模型

### 1. 区间贪心

这类题通常有一堆区间：

```text
[start, end]
```

常见问题：

- 最多选多少个不重叠区间
- 最少删除多少个区间使剩下不重叠
- 最少用几支箭射爆所有气球
- 合并区间

典型策略：

如果目标是尽量多选不重叠区间，通常按右端点升序排序。

原因：

选择结束越早的区间，越能给后面的区间留下空间。

典型题：

- LeetCode 435 无重叠区间
- LeetCode 452 用最少数量的箭引爆气球
- LeetCode 56 合并区间

------

### 2. 覆盖范围贪心

这类题通常关注：

当前最多能覆盖到哪里？

典型思想：

不断维护一个当前能到达的最远边界。

常见变量：

```cpp
int curEnd;     // 当前这一步能覆盖到的边界
int farthest;   // 在当前范围内，下一步最多能到达的位置
```

典型题：

- LeetCode 45 跳跃游戏 II
- LeetCode 55 跳跃游戏
- LeetCode 763 划分字母区间

这类题的重点是：

不是每一步真的跳到某个位置，而是看当前范围内能把下一步扩展到多远。

------

### 3. 排序 + 双指针贪心

这类题常常先排序，然后用两个指针分配资源。

典型思想：

小资源尽量满足小需求。

例如分发饼干：

- 胃口小的孩子优先满足
- 小饼干尽量给胃口小的孩子
- 如果当前饼干满足不了当前孩子，就换更大的饼干

典型题：

- LeetCode 455 分发饼干
- LeetCode 881 救生艇

------

### 4. 差值贪心

这类题通常有两种选择，每个元素选 A 或选 B。

关键是比较：

```text
选 A 比选 B 多赚多少 / 少亏多少
```

常见做法：

按差值排序。

典型题：

- LeetCode 1005 K 次取反后最大化的数组和
- LeetCode 1029 两地调度

核心思想：

如果一个选择对某个人更“划算”，就优先把它分配到对应方向。

------

### 5. 堆 + 贪心

有些贪心题需要动态维护当前最优候选。

这时会用堆。

常见模式：

- 当前可选任务中，选收益最大的
- 当前资源不够时，删除代价最大的
- 当前会议冲突时，保留结束最早的

典型题：

- LeetCode 253 会议室 II
- LeetCode 871 最低加油次数
- LeetCode 630 课程表 III

这类题的特征是：

当前可选集合会动态变化，因此只排序一次不够，需要堆维护最优候选。

------

### 6. 字符串贪心

这类题常常要构造字典序最小、最大，或者满足某种局部条件。

常见思想：

- 能删就删
- 能选更小的就选更小的
- 维护单调栈
- 当前字符是否必须保留

典型题：

- LeetCode 316 去除重复字母
- LeetCode 402 移掉 K 位数字
- LeetCode 738 单调递增的数字

这里经常会和“单调栈”结合。

------

### 六、贪心证明常用模板

### 模板一：交换论证

可以这样写：

假设存在一个最优解，它没有选择贪心算法当前选择的元素 A，而是选择了另一个元素 B。

根据贪心选择的性质，A 不比 B 差。

将最优解中的 B 替换成 A 后：

- 方案仍然合法
- 答案不会变差

所以一定存在一个包含 A 的最优解。

因此当前贪心选择是安全的。

------

### 模板二：区间右端点证明

以“最多选不重叠区间”为例：

每次选择右端点最小的区间。

因为右端点越小，留给后续区间的空间越大。

如果某个最优解第一步没有选择这个右端点最小的区间，而选择了另一个结束更晚的区间，那么可以把它换成右端点更小的区间。

换完后：

- 不会和后面的区间产生更多冲突
- 选择数量不会减少

所以选择右端点最小的区间是安全的。

------

### 模板三：覆盖范围证明

以跳跃游戏为例：

我们不关心具体每一步跳到哪里，只关心当前步数内能覆盖到的最远位置。

如果在当前可达范围内，某个位置能把下一步扩展得最远，那么选择它不会比其他选择差。

因为后续所有能到达的位置，都包含在这个最远覆盖范围内。

所以每一轮扩展最远覆盖边界是合理的。

------

### 七、贪心最容易错的地方

### 1. 把“看起来合理”当成“贪心正确”

例如：

每次选最大的、最小的、最近的、最便宜的，不一定正确。

必须能说明为什么这个选择不会影响全局最优。

------

### 2. 排序依据选错

贪心题里排序依据非常关键。

例如区间调度：

如果按左端点排序，可能不是最优。

通常要按右端点排序，因为目标是给后面留下更多空间。

------

### 3. 忽略反例

如果一个贪心策略想不出证明，就要主动找反例。

常用找反例方法：

- 构造小规模数据
- 构造局部最优和全局最优冲突的情况
- 构造极端情况，例如全相等、一个特别大、一个特别小

------

### 4. 和 DP 混淆

有些题看起来能贪心，但实际要 DP。

如果当前选择会明显影响后面很多状态，并且无法用简单交换证明安全，就要考虑 DP。

例如：

- 打家劫舍不能贪心选大的
- 最长递增子序列不能简单局部选最小或最大
- 背包不能直接按价值或重量贪心

------

### 九、做贪心题时的自问清单

每做一道贪心题，先问：

1. 目标是最大化还是最小化什么？
2. 当前这一步应该优先选择什么？
3. 为什么这个选择不会让后面变差？
4. 是否需要排序？
5. 排序依据是什么？
6. 如果策略错了，能不能构造反例？
7. 这个题是不是其实需要 DP？
8. 代码中是否需要维护边界、当前最优、剩余资源？

------

### 十、一句话总结

贪心不是“每次选最大/最小”这么简单。

真正的贪心是：

找到一个可以被证明安全的局部最优选择。

做贪心题时，重点不是代码模板，而是：

1. 找到选择规则；
2. 说明为什么这个规则不会错；
3. 用排序、双指针、堆等工具把规则实现出来。

## 十一、典型题目及知识

### 区间问题

* **遇到重叠时，删除谁不重要，重要的是保留下来的区间右端点要尽量小。**

* 435 无重叠区间：
  重叠时要删一个，所以保留右端点更小的。

  452 引爆气球：
  重叠时可以共用一支箭，所以维护公共右边界，取 min。

  56 合并区间：
  重叠时要合并，所以右端点取 max。

* 划分字母区间

```cpp
class Solution {
public:
    vector<int> partitionLabels(string s) {
        /*
            题目：763. 划分字母区间

            要把字符串 s 划分成尽可能多的片段，
            并且要求：同一个字母最多只能出现在一个片段中。

            也就是说：
            如果一个字母出现在某个片段里，
            那么这个字母在整个字符串中的所有出现位置，
            都必须被包含在这个片段里。

            --------------------------------------------------
            核心转化：

            对于每个字符 c，
            它在字符串中可能出现多次。

            如果 c 第一次出现在 l，最后一次出现在 r，
            那么任何包含 c 的片段，都必须至少覆盖区间 [l, r]。

            所以这题可以转化成：

                每个字符对应一个“出现区间”，
                我们从左往右扫描，合并必须放在一起的区间。

            不过实际代码里，不需要记录第一次出现位置。
            因为我们从左往右扫描时，当前片段的左端点自然知道。
            真正关键的是：

                当前片段最远必须延伸到哪里？

            这个最远位置由片段内所有字符的“最后一次出现位置”决定。
        */

        vector<int> last(26, 0);

        /*
            先记录每个字符最后一次出现的位置。

            last[c] 表示：
                字符 c 在字符串 s 中最后一次出现的下标。

            为什么只需要最后出现位置？

            因为从左往右扫描时，如果当前片段中出现了字符 c，
            那么为了保证 c 不出现在别的片段里，
            当前片段必须至少覆盖到 last[c]。
        */
        for (int i = 0; i < s.size(); i++) {
            last[s[i] - 'a'] = i;
        }

        vector<int> ans;

        /*
            start 表示当前片段的起点。

            end 表示当前片段必须覆盖到的最远位置。

            一开始当前片段从 0 开始，
            最远边界也先设为 0。
        */
        int start = 0;
        int end = 0;

        /*
            从左往右扫描字符串。

            每遇到一个字符 s[i]，
            就要检查这个字符最后一次出现在哪里。

            如果它最后一次出现的位置更靠后，
            那么当前片段的右边界就必须扩展。
        */
        for (int i = 0; i < s.size(); i++) {
            int idx = s[i] - 'a';

            /*
                当前字符 s[i] 最后一次出现的位置是 last[idx]。

                当前片段必须覆盖当前片段中所有字符的最后出现位置，
                所以 end 要取最大值。
            */
            end = max(end, last[idx]);

            /*
                如果扫描到 i == end，
                说明从 start 到 end 这一段中出现过的所有字符，
                它们的最后一次出现位置都没有超过 end。

                换句话说：
                这一段里的字符不会再出现在后面了。

                所以这里可以安全切一刀。
            */
            if (i == end) {
                ans.push_back(end - start + 1);

                /*
                    下一个片段从 i + 1 开始。
                */
                start = i + 1;
            }
        }

        return ans;
    }
};
```

### 导弹拦截

核心：**一个系统是一条不上升链；要用最少系统覆盖全部导弹，就是用最少条不上升链覆盖序列。根据 Dilworth 定理，这个数量等于最长严格上升子序列的长度。**

初步个人思路：

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    /*
        题目：导弹拦截

        输入若干个导弹高度，以 0 结束。

        第一问：
            一套系统最多可以拦截多少导弹？

            由于后面拦截的导弹高度不能高于前一枚，
            所以一套系统可以拦截的导弹序列必须满足：

                h1 >= h2 >= h3 >= ...

            也就是“最长不上升子序列”。

        第二问：
            如果要拦截所有导弹，最少需要多少套系统？

            每套系统内部都必须是不上升序列。
    */

    vector<int> nums;
    int a;

    cin >> a;
    while (a)
    {
        nums.push_back(a);
        cin >> a;
    }

    int len = nums.size();

    /*
        第一问：一套系统最多拦截多少导弹

        dp[i] 表示：
            以 nums[i] 作为最后一枚导弹时，
            能形成的最长不上升子序列长度。

        例如：
            nums[j] >= nums[i]

        说明 nums[i] 可以接在 nums[j] 后面，
        因为后面的导弹高度不能高于前面的导弹高度。

        所以可以转移：

            dp[i] = max(dp[i], dp[j] + 1)
    */
    vector<int> dp(len, 1);

    /*
        如果至少有一枚导弹，那么答案最少是 1。
        不能初始化成 -1，否则只有一个导弹时会出错。
    */
    int max_missile = 1;

    for (int i = 1; i < len; i++)
    {
        for (int j = 0; j < i; j++)
        {
            /*
                注意这里必须是 >=，不是 >。

                因为题目说“不能高于前一发”，
                也就是可以等于前一发。

                所以这是最长“不上升”子序列，
                不是最长“严格下降”子序列。
            */
            if (nums[j] >= nums[i])
            {
                dp[i] = max(dp[i], dp[j] + 1);
            }
        }

        max_missile = max(max_missile, dp[i]);
    }

    cout << max_missile << "\n";

    /*
        第二问：拦截所有导弹最少需要多少套系统

        res[i] 表示：
            第 i 套系统当前最后拦截的导弹高度。

        如果当前导弹高度 x <= res[i]，
        说明第 i 套系统可以继续拦截这枚导弹。

        你的贪心逻辑是：
            从已有系统中找一个能接住当前导弹的系统；
            如果能接，就把它放进去，并更新这套系统的当前高度；
            如果没有系统能接，就新开一套系统。
    */
    vector<int> res;

    for (auto &x : nums)
    {
        if (res.empty())
        {
            // 还没有任何系统，必须新开一套
            res.push_back(x);
        }
        else
        {
            bool flag = false;

            /*
                尝试把当前导弹放到已有系统中。

                如果 x <= v，说明当前系统最后拦截高度 v
                仍然可以接住高度为 x 的导弹。
            */
            for (auto &v : res)
            {
                if (x <= v)
                {
                    /*
                        接住之后，这套系统的当前最高可接高度更新为 x。

                        因为下一枚导弹必须 <= x 才能继续被这套系统接住。
                    */
                    v = x;
                    flag = true;
                    break;
                }
            }

            /*
                如果所有已有系统都接不了当前导弹，
                就必须新开一套系统。
            */
            if (flag == false)
            {
                res.push_back(x);
            }
        }
    }

    cout << res.size() << "\n";

    return 0;
}
```

优化版（$n\lg n$）

```cpp
#include <bits/stdc++.h>
using namespace std;

/*
    优化版：导弹拦截

    输入：
        若干个正整数，表示导弹高度；
        以 0 结束。

    输出：
        第一行：一套系统最多能拦截多少导弹
        第二行：拦截所有导弹最少需要多少套系统

    --------------------------------------------------

    第一问：
        求最长不上升子序列长度。

        不上升：
            a1 >= a2 >= a3 >= ...

        可以用 O(n log n) 优化。

    第二问：
        求最少需要多少个不上升序列覆盖整个序列。

        可以贪心：
            每个系统维护当前最后拦截的高度；
            当前导弹尽量放入一个“刚好能接住它”的系统中。

        也可以理解为：
            最少不上升序列个数 = 最长上升子序列长度。
*/

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    vector<int> nums;
    int x;

    while (cin >> x && x != 0)
    {
        nums.push_back(x);
    }

    int n = nums.size();

    if (n == 0)
    {
        cout << 0 << "\n";
        cout << 0 << "\n";
        return 0;
    }

    /*
        ==================================================
        第一问：最长不上升子序列长度
        ==================================================

        我们维护一个数组 tail。

        tail[len - 1] 表示：
            长度为 len 的不上升子序列中，
            结尾高度尽可能大的那个结尾高度。

        为什么要“尽可能大”？

        因为对于不上升序列来说，
        结尾越大，后面越容易继续接更小或相等的导弹。

        当前导弹高度为 h。

        如果 h 可以接到某个不上升序列后面，
        需要满足：
            前一个结尾 >= h

        为了维护 tail，可以用二分。

        不过直接写不上升的二分容易绕。
        更简单的办法是：

            把每个高度 h 变成 -h。

        原序列的不上升：
            h1 >= h2 >= h3

        取负后变成：
            -h1 <= -h2 <= -h3

        也就是“不下降子序列”。

        所以第一问可以转化为：
            求 -nums 的最长不下降子序列长度。

        最长不下降子序列用 upper_bound。
    */
    vector<int> tail1;

    for (int h : nums)
    {
        int y = -h;

        /*
            求最长不下降子序列：

            使用 upper_bound 找到第一个 > y 的位置。

            如果所有元素都 <= y，
            说明 y 可以接在最后面，长度增加。

            否则，用 y 替换第一个 > y 的位置，
            让对应长度的结尾尽可能小。
        */
        auto it = upper_bound(tail1.begin(), tail1.end(), y);

        if (it == tail1.end())
        {
            tail1.push_back(y);
        }
        else
        {
            *it = y;
        }
    }

    cout << tail1.size() << "\n";

    /*
        ==================================================
        第二问：最少需要多少套系统 本质Dilworth 定理
        ==================================================

        每套系统内部必须是不上升序列。

        systems[i] 表示：
            第 i 套系统当前最后拦截的导弹高度。

        当前导弹高度为 h。

        如果 systems[i] >= h，
        那么第 i 套系统可以继续接住当前导弹。

        贪心策略：
            在所有能接住 h 的系统中，
            选择当前末尾高度最小的那一套。

        为什么？

            假设有几套系统都能接住 h：

                末尾高度分别是 100, 80, 60
                当前 h = 50

            三套都能接。

            应该放到 60 那套，而不是 100 那套。

            因为 100 那套更“宽松”，以后还能接更高的导弹；
            60 那套更“紧”，用它来接 50 更不浪费。

        所以我们希望找：
            第一个 >= h 的系统末尾高度。

        这正好可以用 lower_bound。

        为了使用 lower_bound，systems 需要保持升序。
    */
    vector<int> systems;

    for (int h : nums)
    {
        /*
            找到第一个 >= h 的位置。

            这个位置表示：
                能接住当前导弹的系统中，
                当前末尾高度最小的一套。
        */
        auto it = lower_bound(systems.begin(), systems.end(), h);

        if (it == systems.end())
        {
            /*
                如果找不到任何 >= h 的系统，
                说明所有已有系统最后的高度都 < h。

                当前导弹太高，不能接在任何已有系统后面，
                必须新开一套系统。
            */
            systems.push_back(h);
        }
        else
        {
            /*
                找到一套系统可以接住当前导弹。

                接住之后，这套系统的末尾高度变成 h。
            */
            *it = h;
        }
    }

    cout << systems.size() << "\n";

    return 0;
}
```

### 滑动窗口加单调队列前置基础

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

/*
    ============================================================
    滑动窗口、单调队列、单调栈基础总结
    ============================================================

    这几个工具经常一起出现，但用途不完全一样。

    1. 滑动窗口
       适合处理“连续区间”问题。

       常见关键词：
           连续子数组
           子串
           长度不超过 k
           长度恰好 k
           至多/至少多少个
           窗口内最大值/最小值

    2. 单调队列
       适合在滑动窗口中维护最大值/最小值。

       常见问题：
           每个长度为 k 的窗口最大值
           每个长度为 k 的窗口最小值
           长度不超过 k 的最大子段和
           前缀和窗口最小值

    3. 单调栈
       适合找每个元素左边/右边第一个比它大或小的元素。

       常见问题：
           下一个更大元素
           每日温度
           柱状图最大矩形
           接雨水

    注意：
        “队列”对应窗口，因为窗口有左右边界，会过期。
        “栈”对应最近的更大/更小关系，因为它只关心一侧的支配关系。
*/


/*
    ============================================================
    一、普通固定长度滑动窗口
    ============================================================

    问题：
        给定数组 a，求长度恰好为 k 的最大连续子数组和。

    思路：
        维护一个长度为 k 的窗口和。

        每次右边加入一个元素；
        如果窗口长度超过 k，就左边移出一个元素。

    时间复杂度：
        O(n)
*/

ll maxSumExactlyK_Window(const vector<ll>& a, int k)
{
    int n = a.size();

    if (k <= 0 || k > n)
    {
        return LLONG_MIN;
    }

    ll windowSum = 0;
    ll ans = LLONG_MIN;

    for (int right = 0; right < n; right++)
    {
        /*
            右端点进入窗口。
        */
        windowSum += a[right];

        /*
            如果窗口长度超过 k，
            就把最左边的元素移出去。

            当前窗口长度是：
                right - left + 1

            这里不用显式写 left，
            因为当 right >= k 时，
            应该移出的元素下标就是 right - k。
        */
        if (right >= k)
        {
            windowSum -= a[right - k];
        }

        /*
            当 right >= k - 1 时，
            窗口长度已经达到 k。
        */
        if (right >= k - 1)
        {
            ans = max(ans, windowSum);
        }
    }

    return ans;
}


/*
    ============================================================
    二、普通可变长度滑动窗口
    ============================================================

    问题示例：
        给定正整数数组 a，求和 >= target 的最短连续子数组长度。

    注意：
        这个模板通常要求数组元素非负或正数。

    原因：
        如果数组里有负数，右端点扩大时窗口和不一定变大，
        左端点收缩时窗口和不一定变小，
        滑动窗口的单调性就没了。

    思路：
        right 不断向右扩展；
        当窗口满足条件时，left 尽量向右收缩。
*/

int minLengthSumAtLeastTarget_Positive(const vector<int>& a, int target)
{
    int n = a.size();

    int left = 0;
    int sum = 0;
    int ans = INT_MAX;

    for (int right = 0; right < n; right++)
    {
        /*
            右端点进入窗口。
        */
        sum += a[right];

        /*
            当窗口已经满足 sum >= target 时，
            尝试收缩左端点，看看能不能让窗口更短。
        */
        while (sum >= target)
        {
            ans = min(ans, right - left + 1);

            sum -= a[left];
            left++;
        }
    }

    if (ans == INT_MAX)
    {
        return 0;
    }

    return ans;
}


/*
    ============================================================
    三、单调队列：滑动窗口最大值
    ============================================================

    问题：
        给定数组 a 和窗口长度 k，
        求每个长度为 k 的窗口中的最大值。

    核心：
        用 deque 存下标，不直接存值。

    队列维护两个性质：

        1. 下标从队首到队尾递增
           因为我们是从左到右扫描。

        2. 对应的值从队首到队尾单调递减
           这样队首永远是当前窗口最大值。

    为什么从队尾弹出较小值？

        假设当前元素 a[i] 比队尾元素更大。
        那么队尾元素不仅更小，而且更早进入窗口，
        它以后不可能再成为最大值，所以可以删除。
*/

vector<int> slidingWindowMaximum(const vector<int>& a, int k)
{
    int n = a.size();

    vector<int> ans;
    deque<int> q;

    for (int i = 0; i < n; i++)
    {
        /*
            第一步：删除已经离开窗口的下标。

            当前窗口是：
                [i - k + 1, i]

            如果 q.front() < i - k + 1，
            说明它已经不在窗口里。
        */
        while (!q.empty() && q.front() <= i - k)
        {
            q.pop_front();
        }

        /*
            第二步：维护单调递减。

            如果队尾元素 <= 当前元素，
            那么队尾元素不可能再成为最大值。
        */
        while (!q.empty() && a[q.back()] <= a[i])
        {
            q.pop_back();
        }

        /*
            第三步：当前下标入队。
        */
        q.push_back(i);

        /*
            第四步：当窗口长度达到 k 后，
            队首就是当前窗口最大值。
        */
        if (i >= k - 1)
        {
            ans.push_back(a[q.front()]);
        }
    }

    return ans;
}


/*
    ============================================================
    四、单调队列：滑动窗口最小值
    ============================================================

    和最大值完全对称。

    队列维护：
        对应值从队首到队尾单调递增。

    队首就是当前窗口最小值。
*/

vector<int> slidingWindowMinimum(const vector<int>& a, int k)
{
    int n = a.size();

    vector<int> ans;
    deque<int> q;

    for (int i = 0; i < n; i++)
    {
        /*
            删除窗口外的下标。
        */
        while (!q.empty() && q.front() <= i - k)
        {
            q.pop_front();
        }

        /*
            维护单调递增。
            如果队尾元素 >= 当前元素，
            队尾就没用了。
        */
        while (!q.empty() && a[q.back()] >= a[i])
        {
            q.pop_back();
        }

        q.push_back(i);

        if (i >= k - 1)
        {
            ans.push_back(a[q.front()]);
        }
    }

    return ans;
}


/*
    ============================================================
    五、前缀和 + 单调队列：长度不超过 k 的最大子段和
    ============================================================

    这个是你现在那道 OJ 题最需要的基础。

    问题：
        给定数组 a，求长度不超过 k 的最大连续子段和。

    核心公式：

        pre[i] = a[0] + a[1] + ... + a[i - 1]

        子数组 a[l ... r] 的和：
            pre[r + 1] - pre[l]

    用 i 表示右端点后一位，也就是 pre[i]。

    当前子数组是：
        a[l ... i - 1]

    长度是：
        i - l

    要求长度不超过 k：

        i - l <= k

    所以：

        l >= i - k

    对于固定的 i，
    合法的 l 范围是：

        max(0, i - k) <= l <= i - 1

    我们要求最大：

        pre[i] - pre[l]

    所以要找合法范围里最小的 pre[l]。

    因此：
        用单调队列维护最近 k 个前缀和的最小值。
*/

ll maxSubarrayAtMostK(const vector<ll>& a, int k)
{
    int n = a.size();

    if (k <= 0)
    {
        return LLONG_MIN;
    }

    vector<ll> pre(n + 1, 0);

    for (int i = 1; i <= n; i++)
    {
        pre[i] = pre[i - 1] + a[i - 1];
    }

    deque<int> q;

    /*
        pre[0] 对应空前缀，
        可以作为从 0 开始的子数组左端点。
    */
    q.push_back(0);

    ll ans = LLONG_MIN;

    for (int i = 1; i <= n; i++)
    {
        /*
            当前合法左前缀下标 l 必须满足：
                l >= i - k

            如果队首太小，就说明对应子数组长度超过 k。
        */
        while (!q.empty() && q.front() < i - k)
        {
            q.pop_front();
        }

        /*
            队首是当前窗口内 pre[l] 最小的位置。
            因此 pre[i] - pre[q.front()] 最大。
        */
        if (!q.empty())
        {
            ans = max(ans, pre[i] - pre[q.front()]);
        }

        /*
            把当前 pre[i] 加入队列。

            为了维护 pre 值递增：
            如果队尾的 pre >= 当前 pre[i]，
            那么队尾不如当前 i 优。

            因为当前 i 更靠后，未来能存活更久；
            而且 pre[i] 更小或相等。
        */
        while (!q.empty() && pre[q.back()] >= pre[i])
        {
            q.pop_back();
        }

        q.push_back(i);
    }

    return ans;
}


/*
    ============================================================
    六、前缀和 + 历史最小值：长度至少 k 的最大子段和
    ============================================================

    问题：
        给定数组 a，求长度至少 k 的最大连续子段和。

    仍然用：

        子数组和 = pre[i] - pre[l]

    当前右端点后一位是 i，
    子数组长度是：

        i - l

    要求：

        i - l >= k

    所以：

        l <= i - k

    对于每个 i，
    合法的 l 是：

        0, 1, 2, ..., i - k

    这个范围只会越来越大，
    所以不需要单调队列，只需要维护历史最小前缀和。
*/

ll maxSubarrayAtLeastK(const vector<ll>& a, int k)
{
    int n = a.size();

    if (k <= 0)
    {
        k = 1;
    }

    if (k > n)
    {
        return LLONG_MIN;
    }

    vector<ll> pre(n + 1, 0);

    for (int i = 1; i <= n; i++)
    {
        pre[i] = pre[i - 1] + a[i - 1];
    }

    ll ans = LLONG_MIN;
    ll minPre = LLONG_MAX;

    /*
        i 至少从 k 开始，
        因为长度至少 k 的子数组至少需要 k 个元素。
    */
    for (int i = k; i <= n; i++)
    {
        /*
            当右端点后一位为 i 时，
            合法左前缀最大可以到 i - k。

            所以把 pre[i-k] 纳入候选。
        */
        minPre = min(minPre, pre[i - k]);

        /*
            当前最大子段和：
                pre[i] - 历史最小合法前缀
        */
        ans = max(ans, pre[i] - minPre);
    }

    return ans;
}


/*
    ============================================================
    七、长度大于 k 的最大子段和
    ============================================================

    长度大于 k：

        len > k

    等价于：

        len >= k + 1

    所以直接调用“长度至少 k + 1”的函数。
*/

ll maxSubarrayGreaterThanK(const vector<ll>& a, int k)
{
    return maxSubarrayAtLeastK(a, k + 1);
}


/*
    ============================================================
    八、单调栈：下一个更大元素
    ============================================================

    问题：
        对每个位置 i，找到它右边第一个比 a[i] 大的元素。

    单调栈维护：
        一个从栈底到栈顶单调递减的下标栈。

    当当前元素 a[i] 比栈顶元素大时，
    当前元素就是栈顶元素的“下一个更大元素”。
*/

vector<int> nextGreaterElement(const vector<int>& a)
{
    int n = a.size();

    vector<int> ans(n, -1);
    stack<int> st;

    for (int i = 0; i < n; i++)
    {
        /*
            当前元素 a[i] 比栈顶元素更大，
            说明 a[i] 是栈顶下标的下一个更大元素。
        */
        while (!st.empty() && a[i] > a[st.top()])
        {
            ans[st.top()] = a[i];
            st.pop();
        }

        /*
            当前下标入栈，等待未来更大的元素。
        */
        st.push(i);
    }

    return ans;
}


/*
    ============================================================
    九、单调栈：下一个更小元素
    ============================================================

    问题：
        对每个位置 i，找到它右边第一个比 a[i] 小的元素。

    和下一个更大元素对称。

    单调栈维护：
        从栈底到栈顶单调递增的下标栈。
*/

vector<int> nextSmallerElement(const vector<int>& a)
{
    int n = a.size();

    vector<int> ans(n, -1);
    stack<int> st;

    for (int i = 0; i < n; i++)
    {
        while (!st.empty() && a[i] < a[st.top()])
        {
            ans[st.top()] = a[i];
            st.pop();
        }

        st.push(i);
    }

    return ans;
}


/*
    ============================================================
    十、单调队列 vs 单调栈区别总结
    ============================================================

    单调队列：

        关键词：
            滑动窗口
            固定长度窗口
            长度不超过 k
            窗口最大值
            窗口最小值

        特点：
            需要从队首删除“过期元素”。
            因为窗口会向右滑动。

        常用容器：
            deque<int>

    ------------------------------------------------------------

    单调栈：

        关键词：
            下一个更大元素
            下一个更小元素
            左边第一个更大/更小
            柱状图最大矩形
            每日温度

        特点：
            不需要处理窗口过期。
            主要处理“谁被当前元素解决了”。

        常用容器：
            stack<int>
            或 vector<int> 模拟栈

    ------------------------------------------------------------

    一句话：

        要维护一个滑动范围内的最大/最小值，用单调队列。

        要找某个元素左边/右边第一个更大/更小，用单调栈。
*/


/*
    ============================================================
    十一、测试主函数
    ============================================================

    输入：
        n k
        a1 a2 ... an

    输出：
        长度恰好 k 的最大子段和
        长度不超过 k 的最大子段和
        长度至少 k 的最大子段和
        长度大于 k 的最大子段和
        每个长度 k 窗口最大值
        每个长度 k 窗口最小值

    你可以用这个主函数自己随便造数据观察。
*/

int main()
{
    ios::sync_with_stdio(false);
```

### 限制长度的最大连续子数组和

核心思想：

```cpp
最大连续子段和 = 当前前缀和 - 前面某个最小前缀和
```

长度限制只是限制这个最小前缀和在那个位置选而已

```cpp
长度 <= k：
前缀下标只能在最近 k 个里选
=> 单调队列维护窗口最小值

长度 > k：
前缀下标只能在 r-k-1 及以前选
=> 维护历史最小值
```

代码如下：

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

/*
    限制长度的最大连续子段和专题

    普通最大子段和可以用 Kadane 算法。

    但是如果题目要求：

        1. 子段长度不超过 k
        2. 子段长度至少 k
        3. 子段长度大于 k
        4. 子段长度恰好 k

    那么普通 Kadane 就不够用了。

    这时更通用的方法是：

        前缀和 + 最小前缀和

    核心公式：

        子数组 [l, r] 的和
        = pre[r + 1] - pre[l]

    也就是说：

        固定右端点时，
        要让子段和最大，
        就要在合法的左端点范围内，
        找到最小的 pre[l]。

    所以问题变成：

        在某个下标范围内维护最小前缀和。
*/


/*
    一、求长度不超过 k 的最大连续子段和

    给定数组 a，求：

        max sum(a[l...r])

    要求：

        r - l + 1 <= k

    --------------------------------------------------

    前缀和定义：

        pre[0] = 0
        pre[i] = a[0] + a[1] + ... + a[i - 1]

    那么：

        a[l] + ... + a[r] = pre[r + 1] - pre[l]

    为了方便，下面用 i 表示前缀下标。

    当右端点对应 pre[i] 时，
    子数组右端点是 i - 1。

    子数组是：

        a[l ... i - 1]

    长度是：

        i - l

    要求长度不超过 k：

        i - l <= k

    所以：

        l >= i - k

    同时 l 必须小于 i。

    因此，对于每个 i，
    合法的 l 范围是：

        max(0, i - k) <= l <= i - 1

    我们要在这个范围内找最小的 pre[l]。

    因为这个范围是一个滑动窗口，
    所以用单调队列维护窗口内最小前缀和。
*/

ll maxSubarrayAtMostK(const vector<ll>& a, int k)
{
    int n = a.size();

    /*
        如果 k == 0，说明不允许选择任何非空子数组。
        一般最大子段和题默认子数组非空，所以这种情况要特殊处理。

        这里返回极小值，表示不存在合法非空子数组。
    */
    if (k <= 0)
    {
        return LLONG_MIN;
    }

    /*
        如果 k > n，长度不超过 k 等价于长度不超过 n，
        也就是普通最大子段和。

        不过下面的单调队列写法本身也能处理 k > n。
    */

    vector<ll> pre(n + 1, 0);

    for (int i = 1; i <= n; i++)
    {
        pre[i] = pre[i - 1] + a[i - 1];
    }

    /*
        队列中存的是前缀和下标。

        并且维护两个性质：

        1. 下标在当前合法窗口内
        2. pre[下标] 单调递增

        这样队首就是当前窗口内最小的前缀和。
    */
    deque<int> q;

    /*
        pre[0] 是最开始可以作为左端点前缀的值。
    */
    q.push_back(0);

    ll ans = LLONG_MIN;

    /*
        枚举右端点对应的前缀下标 i。

        i 从 1 到 n。
        当前考虑的子数组形如：

            a[l ... i - 1]

        子数组和：

            pre[i] - pre[l]
    */
    for (int i = 1; i <= n; i++)
    {
        /*
            对于当前 i，合法的 l 要满足：

                l >= i - k

            如果队首下标太小，说明对应子数组长度超过 k，
            要弹出。
        */
        while (!q.empty() && q.front() < i - k)
        {
            q.pop_front();
        }

        /*
            此时 q.front() 对应合法范围内最小的 pre[l]。

            所以以 i - 1 为右端点、长度不超过 k 的最大子段和是：

                pre[i] - pre[q.front()]
        */
        if (!q.empty())
        {
            ans = max(ans, pre[i] - pre[q.front()]);
        }

        /*
            现在要把 pre[i] 加入队列，
            供后面的右端点使用。

            为了让队首始终是最小前缀和，
            如果队尾的 pre 值 >= 当前 pre[i]，
            那么队尾就没用了。

            因为：
            - 当前 i 的下标更靠后，存活时间更长
            - pre[i] 还更小或相等

            所以可以把队尾弹掉。
        */
        while (!q.empty() && pre[q.back()] >= pre[i])
        {
            q.pop_back();
        }

        q.push_back(i);
    }

    return ans;
}


/*
    二、求长度至少 k 的最大连续子段和

    给定数组 a，求：

        max sum(a[l...r])

    要求：

        r - l + 1 >= k

    --------------------------------------------------

    仍然使用前缀和：

        子数组 a[l ... i - 1] 的长度是 i - l

    要求：

        i - l >= k

    所以：

        l <= i - k

    对每个右端点 i，
    可以选择的左前缀下标是：

        0, 1, 2, ..., i - k

    这个范围只会不断扩大，
    所以不需要单调队列。

    只要维护目前为止的最小 pre[l] 即可。
*/

ll maxSubarrayAtLeastK(const vector<ll>& a, int k)
{
    int n = a.size();

    if (k <= 0)
    {
        /*
            如果长度至少 0，通常就退化得不太自然。
            对最大非空子段和来说，k 一般应当 >= 1。
        */
        k = 1;
    }

    if (k > n)
    {
        return LLONG_MIN;
    }

    vector<ll> pre(n + 1, 0);

    for (int i = 1; i <= n; i++)
    {
        pre[i] = pre[i - 1] + a[i - 1];
    }

    ll ans = LLONG_MIN;

    /*
        minPre 表示：

            当前右端点 i 下，
            所有合法左前缀 pre[l] 中的最小值。

        对于长度至少 k：

            l <= i - k
    */
    ll minPre = LLONG_MAX;

    /*
        i 是右端点后一个位置，即前缀下标。

        只有 i >= k 时，才存在长度至少 k 的子数组。
    */
    for (int i = k; i <= n; i++)
    {
        /*
            当右端点前缀是 pre[i] 时，
            合法左前缀 l 最大可以到 i - k。

            所以先把 pre[i - k] 纳入候选。
        */
        minPre = min(minPre, pre[i - k]);

        /*
            然后用当前 pre[i] 减去历史最小合法前缀。
        */
        ans = max(ans, pre[i] - minPre);
    }

    return ans;
}


/*
    三、求长度大于 k 的最大连续子段和

    长度大于 k 等价于：

        长度至少 k + 1

    所以可以直接调用 maxSubarrayAtLeastK(a, k + 1)。

    这个函数单独写出来，是因为你现在那道 OJ 题里正好会用到：

        len > k
*/

ll maxSubarrayGreaterThanK(const vector<ll>& a, int k)
{
    return maxSubarrayAtLeastK(a, k + 1);
}


/*
    四、求长度恰好 k 的最大连续子段和

    这个是最简单的限制长度问题。

    长度恰好 k 时：

        对每个 i，区间是 a[i-k ... i-1]

    直接用前缀和：

        pre[i] - pre[i-k]
*/

ll maxSubarrayExactlyK(const vector<ll>& a, int k)
{
    int n = a.size();

    if (k <= 0 || k > n)
    {
        return LLONG_MIN;
    }

    vector<ll> pre(n + 1, 0);

    for (int i = 1; i <= n; i++)
    {
        pre[i] = pre[i - 1] + a[i - 1];
    }

    ll ans = LLONG_MIN;

    for (int i = k; i <= n; i++)
    {
        ans = max(ans, pre[i] - pre[i - k]);
    }

    return ans;
}


/*
    五、测试主函数

    你可以自己输入：

        n k
        a1 a2 ... an

    然后它会分别输出：

        长度不超过 k 的最大子段和
        长度至少 k 的最大子段和
        长度大于 k 的最大子段和
        长度恰好 k 的最大子段和

    这只是为了练习模板。
*/

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    cin >> n >> k;

    vector<ll> a(n);

    for (int i = 0; i < n; i++)
    {
        cin >> a[i];
    }

    cout << "length <= k: " << maxSubarrayAtMostK(a, k) << "\n";
    cout << "length >= k: " << maxSubarrayAtLeastK(a, k) << "\n";
    cout << "length >  k: " << maxSubarrayGreaterThanK(a, k) << "\n";
    cout << "length == k: " << maxSubarrayExactlyK(a, k) << "\n";

    return 0;
}
```
