# 二叉树知识点笔记

二叉树是数据结构学习中的一个分水岭。

在链表、栈、队列这些线性结构里，元素之间基本上是一条线；  
而到了二叉树，结构第一次真正变成了：

## 分层的、递归的、非线性的

所以学二叉树，不只是学一个新数据结构，  
更是在训练：

- 递归思维
- 层次结构理解
- DFS / BFS 的基础
- 后续 BST、堆、平衡树等内容的地基

---

# 一、二叉树的基本概念

## 1. 什么是二叉树

二叉树（Binary Tree）是每个结点最多只有两个孩子的树结构。

这两个孩子分别称为：

- 左孩子（left child）
- 右孩子（right child）

---

## 2. 常见术语

### 根节点（root）
整棵树最顶上的结点。

### 叶子节点（leaf）
没有孩子的结点。

### 父节点（parent）
某个结点的上一层结点。

### 子节点（child）
某个结点的左孩子或右孩子。

### 兄弟节点（sibling）
同一个父节点的两个孩子互为兄弟节点。

### 结点的度
一个结点孩子的个数。
- 度为 0：叶子结点
- 度为 1：只有一个孩子
- 度为 2：有两个孩子

---

## 3. 深度、高度、层数

### 结点的深度（depth）
从根到该结点经过的边数。

### 结点的层数
通常定义为“深度 + 1”。
- 根节点在第 1 层

### 结点的高度（height）
从该结点到最远叶子结点的最长路径上的边数。

### 树的高度
根节点的高度，也就是整棵树的高度。

---

# 二、特殊二叉树

## 1. 满二叉树
如果一棵二叉树中每个结点要么有 2 个孩子，要么有 0 个孩子，那么它是满二叉树。

特点：
- 不存在只有一个孩子的结点

---

## 2. 完全二叉树
除了最后一层外，其余各层都满；  
最后一层的结点从左到右连续排列。

特点：
- 非常适合数组存储
- 堆就是典型的完全二叉树

---

## 3. 二叉搜索树（BST）
满足以下性质的二叉树：

- 左子树所有结点值都小于根
- 右子树所有结点值都大于根
- 左右子树也分别是 BST

这是后面专门要重点学的一类树。

---

# 三、二叉树的存储方式

## 1. 链式存储（最常见）

每个结点一般定义成：

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};
```
这是最标准的二叉树表示方式。

------

## 2. 顺序存储（数组存储）

适用于完全二叉树。

如果根下标是 `1`，那么：

- 左孩子：`2 * i`
- 右孩子：`2 * i + 1`
- 父节点：`i / 2`

这种表示方式在：

- 堆
- 完全二叉树相关题

里很常见。

------

# 四、二叉树最核心的内容：遍历

遍历就是按某种顺序访问整棵树中的所有结点。

------

## 1. 前序遍历（Preorder）

顺序：

## 中 -> 左 -> 右

### 递归模板

```cpp
void preorder(TreeNode* root) {
    if (root == nullptr) return;
    cout << root->val << ' ';
    preorder(root->left);
    preorder(root->right);
}
```

------

## 2. 中序遍历（Inorder）

顺序：

## 左 -> 中 -> 右

### 递归模板

```cpp
void inorder(TreeNode* root) {
    if (root == nullptr) return;
    inorder(root->left);
    cout << root->val << ' ';
    inorder(root->right);
}
```

特别重要：

## BST 的中序遍历结果是有序的

------

## 3. 后序遍历（Postorder）

顺序：

## 左 -> 右 -> 中

### 递归模板

```cpp
void postorder(TreeNode* root) {
    if (root == nullptr) return;
    postorder(root->left);
    postorder(root->right);
    cout << root->val << ' ';
}
```

------

# 五、前中后序遍历的非递归实现

非递归本质上就是：

## 用栈来模拟递归调用过程

------

## 1. 前序非递归

思路：

- 根入栈
- 每次弹出栈顶，立刻访问
- 先压右，再压左

### 模板代码

```cpp
vector<int> preorderTraversal(TreeNode* root) {
    vector<int> ans;
    if (root == nullptr) return ans;

    stack<TreeNode*> st;
    st.push(root);

    while (!st.empty()) {
        TreeNode* cur = st.top();
        st.pop();

        ans.push_back(cur->val);

        if (cur->right) st.push(cur->right);
        if (cur->left) st.push(cur->left);
    }

    return ans;
}
```

------

## 2. 中序非递归

思路：

- 一路向左压栈
- 走到空后弹栈访问
- 再去右子树

### 模板代码

```cpp
vector<int> inorderTraversal(TreeNode* root) {
    vector<int> ans;
    stack<TreeNode*> st;
    TreeNode* cur = root;

    while (cur != nullptr || !st.empty()) {
        if (cur != nullptr) {
            st.push(cur);
            cur = cur->left;
        } else {
            cur = st.top();
            st.pop();

            ans.push_back(cur->val);
            cur = cur->right;
        }
    }

    return ans;
}
```

------

## 3. 后序非递归

最常用的简单方法：

## 根 -> 右 -> 左，再整体 reverse

### 模板代码

```cpp
vector<int> postorderTraversal(TreeNode* root) {
    vector<int> ans;
    if (root == nullptr) return ans;

    stack<TreeNode*> st;
    st.push(root);

    while (!st.empty()) {
        TreeNode* cur = st.top();
        st.pop();

        ans.push_back(cur->val);

        if (cur->left) st.push(cur->left);
        if (cur->right) st.push(cur->right);
    }

    reverse(ans.begin(), ans.end());
    return ans;
}
```

------

# 六、层序遍历（Level Order Traversal）

层序遍历就是：

## 从上到下、一层一层访问结点

它本质上就是：

## 树上的 BFS

所以天然使用：

## 队列

### 层序遍历模板

```cpp
vector<vector<int>> levelOrder(TreeNode* root) {
    vector<vector<int>> ans;
    if (root == nullptr) return ans;

    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        int sz = q.size();
        vector<int> level;

        for (int i = 0; i < sz; i++) {
            TreeNode* cur = q.front();
            q.pop();

            level.push_back(cur->val);

            if (cur->left) q.push(cur->left);
            if (cur->right) q.push(cur->right);
        }

        ans.push_back(level);
    }

    return ans;
}
```

# 九、二叉搜索树（BST）知识点

BST 是二叉树里最重要的特殊结构之一。

------

## 1. BST 的性质

对于任意结点 `x`：

- 左子树全部小于 `x.val`
- 右子树全部大于 `x.val`
- 左右子树也都是 BST

------

## 2. BST 最重要的性质

## 中序遍历结果严格有序

所以很多题都围绕这条性质展开。

------

## 3. BST 搜索

如果目标值 `target`：

- 小于当前节点值，就去左子树
- 大于当前节点值，就去右子树
- 等于就找到

### 模板代码

```cpp
TreeNode* searchBST(TreeNode* root, int val) {
    while (root != nullptr) {
        if (root->val == val) return root;
        if (val < root->val) root = root->left;
        else root = root->right;
    }
    return nullptr;
}
```

------

## 4. BST 插入

思路：

- 像搜索一样往下找
- 找到空位置
- 挂进去

------

## 5. BST 删除

删除分三种情况：

### 情况 1：没有左孩子

右子树顶上来

### 情况 2：没有右孩子

左子树顶上来

### 情况 3：左右孩子都有

找后继（右子树最小值）来顶替

这是 BST 最经典也是最难的操作之一。

------

## 6. BST 前驱和后继

### 后继

比当前结点大的最小结点

### 前驱

比当前结点小的最大结点

特别是：

- 有右子树：后继 = 右子树最小值
- 有左子树：前驱 = 左子树最大值

------

```cpp
bool check(TreeNode* a, TreeNode* b) {
    if (a == nullptr && b == nullptr) return true;
    if (a == nullptr || b == nullptr) return false;
    if (a->val != b->val) return false;

    return check(a->left, b->right) &&
           check(a->right, b->left);
}

    if (root == nullptr) return true;

}
```

## 6. 判断平衡二叉树

直观写法是：

对于每个结点，都分别求出左子树高度和右子树高度。  
如果高度差大于 1，则不是平衡二叉树；  
否则继续递归判断左右子树。

这种写法思路很自然，但会重复计算很多子树高度。

优化思路是：

写一个函数 `height(root)`：

- 如果以 `root` 为根的子树是平衡的，就返回它的高度
- 如果不平衡，就返回 `-1`

这样就把“求高度”和“判断是否平衡”合并到了一次递归里。

代码如下：

```cpp
int height(TreeNode* root) {
    if (!root) return 0;

    int l = height(root->left);
    if (l == -1) return -1; // 左子树不平衡，那么整个树肯定不平衡，右子树同理

    int r = height(root->right);
    if (r == -1) return -1;
	//此时左右子树都平衡，需要判断高度差是否超过1
    if (abs(l - r) > 1) return -1;
	//此时说明都满足，所以直接返回高度即可
    return max(l, r) + 1;
}

bool isBalanced(TreeNode* root) {
    return height(root) != -1; //直接对根节点调用height即可
}
```

## 7. 路径总和

这道题要求判断：

是否存在一条从根节点到叶子节点的路径，使得这条路径上所有节点值之和等于 `targetSum`。

一开始最直观的想法是：

- 先 DFS 找出所有根到叶子的路径和
- 把这些路径和存起来
- 最后再检查其中是否存在等于 `targetSum` 的值

这种方法是可以做出来的，但会额外存下所有路径和，其实没有必要。

因为题目只问：

**是否存在**

所以更自然的优化方式是：

## 在 DFS 过程中直接判断，一旦找到就返回 `true`

---

### 核心递归思路

设计函数：

```cpp
bool dfs(TreeNode* root, int aim)
```

它表示：

**从当前节点 `root` 出发，是否存在一条根到叶子的路径，使得当前还需要凑出的和为 `aim`。**

进入当前节点后，先让：

```cpp
aim -= root->val;
```

表示当前节点值已经被使用掉了。

------

### 递归边界

如果当前节点为空：

```cpp
if (!root) return false;
```

说明这条路走不通。

------

### 叶子节点时的判断

这题最关键的一点是：

**必须是根到叶子节点的路径。**

所以不能在中途只要发现 `aim == 0` 就直接返回 `true`，
必须等到当前节点是叶子节点时，才能检查：

```cpp
if (!root->left && !root->right) {
    return aim == 0;
}
```

也就是说：

- 当前节点是叶子
- 并且减去当前节点值之后剩余目标刚好为 0

这才说明找到了一条合法路径。

------

### 递归转移

如果当前节点不是叶子，就继续递归左右子树：

```cpp
return dfs(root->left, aim) || dfs(root->right, aim);
```

只要左右子树中有一边能找到合法路径，就返回 `true`。

------

### 完整代码

```cpp
class Solution {
public:
    bool dfs(TreeNode* root, int aim) {
        if (!root) return false;

        aim -= root->val;

        if (!root->left && !root->right) {
            return aim == 0;
        }

        return dfs(root->left, aim) || dfs(root->right, aim);
    }

    bool hasPathSum(TreeNode* root, int targetSum) {
        return dfs(root, targetSum);
    }
};
```

------

## 这道题最值得吸收的营养

### 1. 题目只问“是否存在”，就不要多做无用工作

如果题目只要 `true / false`，
那就不一定要把所有答案都收集出来。

一旦找到答案，就可以立刻返回。

这类优化思想在递归题里非常常见。

------

### 2. 递归函数的定义很重要

这题里最关键的不是代码本身，而是你怎么定义递归函数。

这里把 `dfs(root, aim)` 定义成：

**从当前节点出发，是否能找到一条路径，使得剩余和为 `aim`。**

有了这个定义以后，代码会变得特别自然。

这再次说明：

**递归题的本质往往在于“函数语义设计”。**

------

### 3. 必须抓住“根到叶子”这个限制

这题非常容易错在这里：

有些人会写成只要中途 `aim == 0` 就返回 `true`，
但这是不对的，因为题目要求的是：

**必须走到叶子节点，这条路径才算完整路径。**

所以判断成功的时机必须是：

- 当前节点是叶子
- 剩余目标刚好为 0

------

### 4. 这是“路径类树题”的第一个典型模型

这道题的本质是：

## 在递归过程中维护状态

这里维护的状态就是：

- 当前还剩多少目标值没凑够

后面像：

- 路径总和 II
- 二叉树的所有路径
- 其他路径记录类题目

都会在这个模型上继续扩展。

所以这题非常值得吃透。

这道题的关键在于：

**把递归函数定义成“剩余目标值是否还能在当前子树中被凑出来”，并且只在叶子节点处判断是否成功。**

## 8.路径总和II

这次在路径总和的基础上要记录满足条件的路径。这其实就是``dfs``的回溯

代码如下

```cpp
class Solution {
public:
    vector<vector<int>> ans; // 存所有满足条件的路径
    vector<int> path;        // 存当前正在递归的这条路径

    /**
     * @brief DFS + 回溯
     * @param root 当前访问到的节点
     * @param aim  从当前节点开始，还需要凑出的目标和
     *
     * 递归含义：
     * 从当前节点 root 出发，继续往下走，寻找所有“根到叶子”的路径，
     * 使得这条路径的节点和恰好等于 aim。
     */
    void dfs(TreeNode* root, int aim) {
        // 空节点直接返回
        if (!root) return;

        // 1. 进入当前节点：把当前节点加入路径
        path.push_back(root->val);

        // 2. 当前节点值已经用掉，所以目标和要减去当前节点值
        aim -= root->val;

        // 3. 如果当前节点是叶子节点，说明这是一条完整路径
        if (!root->left && !root->right) {
            // 若此时 aim == 0，说明当前路径和恰好等于目标值
            if (aim == 0) {
                ans.push_back(path); // 把当前路径加入答案
            }

            // 4. 回溯：离开当前节点前，把当前节点从路径中弹出
            path.pop_back();
            return;
        }

        // 5. 继续递归左右子树
        dfs(root->left, aim);
        dfs(root->right, aim);

        // 6. 回溯：当前节点的左右子树都处理完后，恢复现场
        path.pop_back();
    }

    vector<vector<int>> pathSum(TreeNode* root, int targetSum) {
        // 清空，保证多次调用时状态干净
        ans.clear();
        path.clear();

        dfs(root, targetSum);
        return ans;
    }
};
```

## 9.验证BST

给你一个二叉树的根节点 `root` ，判断其是否是一个有效的二叉搜索树。

**有效** 二叉搜索树定义如下：

- 节点的左子树只包含 **严格小于** 当前节点的数。
- 节点的右子树只包含 **严格大于** 当前节点的数。
- 所有左子树和右子树自身必须也是二叉搜索树。

ac代码：

```cpp
class Solution {
public:
    //重点关注参数的定义，还需要判断每个节点应该在的范围，才能保证**整个**左子树
    //的值小于当前节点，右子树同理，状态定义很重要
    bool dfs(TreeNode* root, long long low, long long high) {
        if (!root)
            return true;
        if (root->val <= low || root->val >= high) {
            return false;
        }

        return dfs(root->left, low, root->val) &&
               dfs(root->right, root->val, high);
    }

    bool isValidBST(TreeNode* root) {
        return dfs(root,LONG_LONG_MIN,LONG_LONG_MAX);
    }
};
```

进阶版，根据中序遍历的性质，边中序边判断，依旧特别注意返回函数的意义以及递归的练习。

```cpp

/**
 * 二叉树题的训练重点，不只是树本身，
 * 更重要的是系统练习递归：
 * - 定义递归函数语义
 * - 理清返回值含义
 * - 明确当前节点处理时机
 * - 学会利用返回值与回溯
 */
class Solution {
public:
    TreeNode* prev = nullptr;//直接初始化为空指针
    //注意递归函数的定义：判断中序遍历中是否有非严格递增元素
    bool inorder(TreeNode* root) {
        if(!root) return true;
        if(!inorder(root->left)) return false;//要利用函数返回值
        if(prev != nullptr && prev->val>=root->val)
        {
            return false;
        }
        prev = root;//直接记录，再不用prev-val =root->val
        if(!inorder(root->right)) return false;
        return true;
    }

    bool isValidBST(TreeNode* root) {
        return inorder(root);
    }
};
```

总结：左二叉树的题真就是要把递归的语义和细节好好理解通透

# 十一、二叉树最容易错的地方

## 1. 空节点处理不统一

很多错都来自：

- 忘记判空
- 递归出口写错

## 2. 把“深度”和“高度”混掉

做题时如果只是算答案问题不大，但写笔记时最好概念分清。

## 3. 非递归遍历 while 条件写错

特别是中序非递归：

```cpp
while (cur != nullptr || !st.empty())
```

不能写成 `&&`。

## 4. 路径类题漏回溯

如果维护路径数组或字符串，离开当前节点前要记得恢复现场。

## 5. BST 判断只看“父子关系”

这是最常见错误之一。必须保证：

## 整棵左子树都小于根，整棵右子树都大于根
