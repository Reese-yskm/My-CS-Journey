# 堆 / 优先队列专题学习笔记

堆（Heap）是一个非常经典、非常高频的数据结构。

它最核心的作用不是“排序全部元素”，而是：

## 快速维护当前集合中的最值

也就是说，当你遇到这种需求时，就要立刻想到堆：

- 动态维护最大值 / 最小值
- 不断取当前最优元素
- 维护 Top K
- 多路归并
- 贪心 + 数据结构

---

# 一、堆到底是什么

堆本质上是一棵**完全二叉树**，并且满足堆序性质。

---

## 1. 大顶堆

对于任意节点：

- 父节点值 >= 子节点值

所以堆顶就是：

## 当前最大值

---

## 2. 小顶堆

对于任意节点：

- 父节点值 <= 子节点值

所以堆顶就是：

## 当前最小值

---

# 二、为什么堆高频有用

因为很多题并不要求你把所有元素都排好序，  
它只是要求你：

- 不断拿当前最小的
- 不断拿当前最大的
- 维护前 k 大
- 每次选一个最优候选

这时如果你用排序：

- 每次都排序太贵
- 每次找一遍最值也很慢

而堆可以做到：

- 插入：`O(log n)`
- 删除堆顶：`O(log n)`
- 查看堆顶：`O(1)`

---

# 三、堆的底层直觉

堆不是“整体有序”，而是：

## 只保证堆顶最优

比如小顶堆：

- 堆顶最小
- 但下面元素之间不保证严格排序

所以：

## 堆适合“维护最值”
## 不适合“按顺序遍历所有元素”

这点很重要。

---

# 四、数组下标中的堆

因为堆是完全二叉树，所以常用数组存。

如果下标从 `1` 开始：

- 左孩子：`2 * i`
- 右孩子：`2 * i + 1`
- 父节点：`i / 2`

如果下标从 `0` 开始：

- 左孩子：`2 * i + 1`
- 右孩子：`2 * i + 2`
- 父节点：`(i - 1) / 2`

在 LeetCode / OJ 中，如果你自己手写堆，多半用 0 下标。

---

# 五、堆的两个核心操作

---

## 1. 上浮（sift up / heapify up）

场景：

## 新元素插入到堆尾后

此时它可能破坏堆序性质，就不断和父节点比较：

- 如果该上去，就交换
- 直到合法为止

比如小顶堆里：

- 新元素比父节点还小
- 就往上冒

---

## 2. 下沉（sift down / heapify down）

场景：

## 堆顶删除后，把堆尾元素放到堆顶

此时堆顶可能不合法，于是不断和左右孩子中更优的那个比较：

- 如果该下去，就交换
- 直到合法为止

比如小顶堆里：

- 当前元素比某个孩子还大
- 就要往下沉

---

# 六、C++ 里的 priority_queue

C++ 里最常用的是 STL 的 `priority_queue`。

---

## 1. 默认是大顶堆

```cpp
priority_queue<int> pq;
```

堆顶是最大值。

### 常用操作

```cpp
pq.push(x);      // 插入
pq.pop();        // 删除堆顶
pq.top();        // 查看堆顶
pq.empty();      // 判空
pq.size();       // 元素个数
```

------

## 2. 小顶堆写法

```cpp
priority_queue<int, vector<int>, greater<int>> pq;
```

堆顶是最小值。

这个写法要熟。

------

## 3. 存 pair

很多题会把：

- 值
- 下标
- 权重
- 时间

一起塞进堆里。

例如：

```cpp
priority_queue<pair<int,int>> pq;
```

默认比较规则是：

## 先比 first，再比 second

而且默认仍然是大顶堆。

------

## 4. 自定义比较（了解即可）

如果以后题更复杂，可以自定义比较器。

但你当前阶段先把：

- 大顶堆
- 小顶堆
- pair 入堆

吃熟就很好了。

------

# 七、什么时候应该想到堆

这部分很关键。

你做题时一旦看到下面这些味道，就要警觉：

------

## 1. “动态维护最值”

比如：

- 数据不断加入
- 每次都要快速知道当前最大 / 最小

这是堆最直接的使用场景。

------

## 2. “每次选一个当前最优候选”

比如：

- 每次取最小代价
- 每次取最大收益
- 每次弹出当前最短距离点

这类题经常是：

## 贪心 + 堆

------

## 3. “Top K”

比如：

- 前 k 大元素
- 第 k 大元素
- 最频繁的前 k 个元素

这类题堆特别常见。

------

## 4. “多路归并”

比如：

- 合并 k 个有序链表
- 合并多个有序数组
- 每次从若干候选头部选最小值

这类题经典地用小顶堆。

------

## 5. “边插入边查询中位数”

这时常常是：

## 双堆

你前面已经做过动态中位数了。

------

# 八、堆和排序的区别

很多人一开始容易混。

------

## 排序

结果是：

## 所有元素整体有序

代价通常较大，而且插入新元素后又要重新处理。

------

## 堆

结果是：

## 只有堆顶最优

所以：

- 想取最值：堆很强
- 想完整有序：堆不如排序直接

------

# 九、几个经典模型

------

## 模型 1：固定大小的小顶堆维护前 k 大

假设你想维护前 k 大元素。

思路：

- 用一个大小最多为 k 的小顶堆
- 堆里始终放当前前 k 大
- 堆顶就是这 k 个里最小的那个，也就是“门槛”

新元素 `x` 来了：

- 如果堆没满，直接放
- 如果堆满且 `x > 堆顶`，说明它能进入前 k 大
  - 弹出堆顶
  - 插入 `x`

这样最终堆里就是前 k 大。

------

## 模型 2：多路归并

比如合并 k 个有序链表。

思路：

- 每个链表当前头节点放入小顶堆
- 每次弹出最小的那个
- 再把它后继放进去

这样不断取最小值，就完成归并。

------

## 模型 3：双堆维护中位数

你前面刚做过。

思路：

- 左边大顶堆：维护较小一半
- 右边小顶堆：维护较大一半
- 保持大小平衡
- 中位数由两个堆顶决定

这是堆专题里非常经典的一题。

------

# 十一、堆专题最容易错的点

------

## 1. 大顶堆 / 小顶堆写反

尤其是：

```cpp
priority_queue<int, vector<int>, greater<int>> pq;
```

这个是小顶堆，不要混。

------

## 2. 忘记堆不是整体有序

不能指望从堆里随便遍历出来就是有序序列。

只有反复 `pop()` 才能得到有序弹出顺序。

------

## 3. pair 比较规则没搞清

默认是先比 `first`，再比 `second`。

------

## 4. Top K 的堆方向搞反

比如维护前 k 大，要用：

## 小顶堆

因为堆顶是“门槛”。

维护前 k 小则通常用大顶堆。

------

## 5. 双堆再平衡条件写错

这个你前面已经亲身感受过了：

- 左右大小关系
- 再平衡时机
- 搬哪个堆顶

都要很清楚。

------

# 十五、经典题型笔记

## 1.数组中第k个最大元素

给定整数数组 `nums` 和整数 `k`，请返回数组中第 `k` 个最大的元素。

请注意，你需要找的是数组排序后的第 `k` 个最大的元素，而不是第 `k` 个不同的元素。

```cpp
class Solution {
public:
    int findKthLargest(vector<int>& nums, int k) {
        priority_queue<int, vector<int>, greater<int>> pq; //使用小顶堆，维护当前前k大元素，所以堆顶最小的就是第k大
        for (auto& x : nums) {
            //如果大小没有达到k，直接入堆
            if (pq.size() < k) {
                pq.push(x);
            }
            //注意是else if,如果当前元素大于堆顶，出堆然后当前元素入堆
            else if (pq.top() < x) {
                pq.pop();
                pq.push(x);
            }
        }
        return pq.top();	//最后堆顶即为答案
    }
};
```

## 2.数据流中的中位数

**中位数**是有序整数列表中的中间值。如果列表的大小是偶数，则没有中间值，中位数是两个中间值的平均值。

- 例如 `arr = [2,3,4]` 的中位数是 `3` 。
- 例如 `arr = [2,3]` 的中位数是 `(2 + 3) / 2 = 2.5` 。

实现 MedianFinder 类:

- `MedianFinder()` 初始化 `MedianFinder` 对象。
- `void addNum(int num)` 将数据流中的整数 `num` 添加到数据结构中。
- `double findMedian()` 返回到目前为止所有元素的中位数。与实际答案相差 `10-5` 以内的答案将被接受。

```cpp
class MedianFinder
{
public:
    // 左边大顶堆：维护较小的一半数字
    // 堆顶是左半边的最大值
    priority_queue<int> l;

    // 右边小顶堆：维护较大的一半数字
    // 堆顶是右半边的最小值
    priority_queue<int, vector<int>, greater<int>> r;

    // 构造函数：初始化对象
    MedianFinder()
    {
    }

    void addNum(int num)
    {
        // -----------------------------
        // 第一步：先把新数字放进合适的堆
        // -----------------------------
        // 维护原则：
        // - 左边放较小的一半
        // - 右边放较大的一半
        //
        // 如果左堆为空，直接放左边
        if (l.empty())
        {
            l.push(num);
        }
        // 如果新数比左堆堆顶大，说明它更适合进入右边
        else if (num > l.top())
        {
            r.push(num);
        }
        // 否则放左边
        else
        {
            l.push(num);
        }

        // -----------------------------
        // 第二步：调整两个堆的大小，使之平衡
        // -----------------------------
        // 我们始终保持：
        // 1. l.size() == r.size()
        // 或
        // 2. l.size() == r.size() + 1
        //
        // 也就是说：
        // - 左边可以和右边一样多
        // - 或者左边只比右边多 1 个
        //
        // 这样做的好处是：
        // - 总数为奇数时，中位数就是左堆堆顶
        // - 总数为偶数时，中位数是两个堆顶的平均值

        // 如果左边太多了，就把左堆堆顶移到右边
        if (l.size() > r.size() + 1) //注意写法，尽量避免l.size()-r.size()这种减法运算写法
        {
            r.push(l.top());
            l.pop();
        }
        // 如果右边比左边多，就把右堆堆顶移到左边
        else if (r.size() > l.size())
        {
            l.push(r.top());
            r.pop();
        }
    }

    double findMedian()
    {
        // 当前总元素个数
        int sum = l.size() + r.size();

        // 如果总数是奇数
        // 根据维护规则，左边一定比右边多 1 个
        // 所以中位数就是左堆堆顶
        if (sum % 2 == 1)
        {
            return (double)l.top();
        }
        else
        {
            // 如果总数是偶数
            // 中位数是左半边最大值和右半边最小值的平均值
            return (l.top() + r.top()) / 2.0;
        }
    }
};
```
