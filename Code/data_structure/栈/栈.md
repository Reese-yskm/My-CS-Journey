# 栈 Stack 全套模板笔记

## 0. 栈的核心思想

栈是一种**后进先出**的数据结构。

英文叫：

```text
LIFO = Last In First Out
```

也就是：

> 后进去的元素，先出来。

------

## 1. 栈的直观理解

可以把栈想象成一摞盘子：

```text
栈顶
 ↓
[5]
[4]
[3]
[2]
[1]
```

如果你放入一个新元素 `6`：

```text
[6]  ← 新来的在最上面
[5]
[4]
[3]
[2]
[1]
```

如果你要拿一个元素，只能先拿最上面的 `6`。

所以栈的特点是：

```text
只能从栈顶进，也只能从栈顶出
```

------

# 1️⃣ C++ 栈基础模板

## 基础用法

```cpp
#include <stack>
using namespace std;

stack<int> st;

st.push(10);     // 入栈
st.push(20);

int x = st.top(); // 查看栈顶元素，x = 20

st.pop();        // 弹出栈顶元素

bool ok = st.empty(); // 判断是否为空
```

------

## 常用操作总结

```cpp
stack<int> st;

st.push(x);      // 把 x 放入栈顶
st.pop();        // 删除栈顶元素，不返回值
st.top();        // 返回栈顶元素
st.empty();      // 判断栈是否为空
st.size();       // 返回栈中元素个数
```

------

## 重要注意点

### 1. `pop()` 不返回元素

错误写法：

```cpp
int x = st.pop(); // 错误
```

正确写法：

```cpp
int x = st.top();
st.pop();
```

------

### 2. 调用 `top()` 和 `pop()` 前最好先判断是否为空

错误写法：

```cpp
int x = st.top(); // 如果栈空，会出问题
```

安全写法：

```cpp
if (!st.empty()) {
    int x = st.top();
    st.pop();
}
```

------

# 2️⃣ 用 vector 模拟栈模板

有时候我们不用 `stack<int>`，而是用 `vector<int>` 来模拟栈。

## 模板

```cpp
vector<int> st;

st.push_back(x);     // 入栈
st.pop_back();       // 出栈
int x = st.back();   // 栈顶元素
bool ok = st.empty();
```

------

## 为什么有时用 vector 模拟栈？

因为 `stack` 只能访问栈顶，不能遍历内部元素。

而 `vector` 更灵活：

```cpp
for (int x : st) {
    cout << x << " ";
}
```

所以在一些题里，尤其是需要调试或者需要最后输出栈内内容时，用 `vector` 会更方便。

# 3️⃣ 栈的第一个核心模型：括号匹配

## 题型特征

看到这些东西，要想到栈：

```text
() [] {}
括号是否合法
表达式是否合法
成对匹配
最近的未匹配元素
```

------

## 核心思想

遇到左括号：

```text
入栈
```

遇到右括号：

```text
看栈顶是否是对应的左括号
```

------

## 图示

字符串：

```text
([{}])
```

扫描过程：

```text
遇到 (    栈: (
遇到 [    栈: ( [
遇到 {    栈: ( [ {
遇到 }    栈顶是 {，匹配，弹出
遇到 ]    栈顶是 [，匹配，弹出
遇到 )    栈顶是 (，匹配，弹出
最后栈空，合法
```

------

## 括号匹配模板

```cpp
bool isValid(string s)
{
    stack<char> st;

    for (char c : s)
    {
        if (c == '(' || c == '[' || c == '{')
        {
            st.push(c);
        }
        else
        {
            if (st.empty())
            {
                return false;
            }

            char top = st.top();
            st.pop();

            if (c == ')' && top != '(')
            {
                return false;
            }
            if (c == ']' && top != '[')
            {
                return false;
            }
            if (c == '}' && top != '{')
            {
                return false;
            }
        }
    }

    return st.empty();
}
```

------

## 关键解释

为什么最后要：

```cpp
return st.empty();
```

因为可能出现这种情况：

```text
(((
```

扫描过程中没有遇到非法右括号，但左括号没被匹配完。

所以最后必须检查栈是否为空。

------

## 括号匹配口诀

```text
左括号入栈，右括号看栈顶。
栈空不合法，类型不匹配不合法。
最后栈空才合法。
```

# 4️⃣ 栈的第二个核心模型：消除问题

## 题型特征

看到这些描述，要想到栈：

```text
相邻相同就消除
遇到某种模式就抵消
前一个元素和当前元素发生反应
```

比如：

```text
abbaca
```

如果相邻相同就消除：

```text
abbaca
  bb 消除
aaca
aa 消除
ca
```

结果是：

```text
ca
```

------

## 核心思想

用栈保存“还没有被消掉的元素”。

每来一个新元素：

- 如果和栈顶能消除，就弹出栈顶
- 否则入栈

------

## 模板

```cpp
string removeDuplicates(string s)
{
    string st;

    for (char c : s)
    {
        if (!st.empty() && st.back() == c)
        {
            st.pop_back();
        }
        else
        {
            st.push_back(c);
        }
    }

    return st;
}
```

------

## 为什么这里用 string 当栈？

因为 `string` 也支持：

```cpp
push_back()
pop_back()
back()
empty()
```

所以它可以像栈一样使用。

------

## 图示

输入：

```text
abbaca
```

过程：

```text
读 a: 栈 a
读 b: 栈 ab
读 b: b 和栈顶 b 相同，消除，栈 a
读 a: a 和栈顶 a 相同，消除，栈 空
读 c: 栈 c
读 a: 栈 ca
```

结果：

```text
ca
```

# 5️⃣ 栈的第三个核心模型：单调栈

单调栈是栈里最重要、也最需要认真掌握的部分。

它解决的是：

> 找某个元素左边或右边，第一个比它大/小的元素。

------

## 题型特征

看到这些词，要想到单调栈：

```text
下一个更大元素
下一个更小元素
左边第一个更大
右边第一个更小
每日温度
柱状图
区间内第一个挡住它的元素
```

------

## 单调栈的本质

普通栈只是后进先出。

单调栈是在栈的基础上，额外维护一个性质：

```text
栈内元素保持单调递增或单调递减
```

------

## 单调栈常用写法

栈里通常不直接存值，而是存**下标**。

```cpp
stack<int> st; // 存下标
```

为什么存下标？

因为下标能同时拿到：

```cpp
nums[i]       // 当前值
nums[st.top()] // 栈顶对应的值
```

并且可以更新答案数组：

```cpp
res[st.top()] = ...
```

------

# 6️⃣ 单调栈模板一：找右边第一个更大元素

## 问题

给数组： 

```text
nums = [2, 1, 3]
```

对每个元素，找它右边第一个比它大的元素。

答案：

```text
2 的右边第一个更大是 3
1 的右边第一个更大是 3
3 的右边没有更大，是 -1
```

所以：

```text
res = [3, 3, -1]
```

------

## 模板代码

```cpp
vector<int> nextGreaterElement(vector<int>& nums)
{
    int n = nums.size();
    vector<int> res(n, -1);
    stack<int> st; // 存下标

    for (int i = 0; i < n; i++)
    {
        while (!st.empty() && nums[i] > nums[st.top()])
        {
            res[st.top()] = nums[i];
            st.pop();
        }

        st.push(i);
    }

    return res;
}
```

------

## 图示理解

数组：

```text
nums = [2, 1, 3]
下标:    0  1  2
```

### i = 0，nums[i] = 2

栈空，入栈：

```text
st: [0]
```

表示：下标 0 这个元素 `2` 还没找到右边更大的。

------

### i = 1，nums[i] = 1

`1 > 2` 不成立，入栈：

```text
st: [0, 1]
```

表示：

```text
2 还没找到右边更大
1 也还没找到右边更大
```

------

### i = 2，nums[i] = 3

现在 `3 > nums[1] = 1`

所以：

```cpp
res[1] = 3;
pop 1;
```

继续看栈顶：

```text
3 > nums[0] = 2
```

所以：

```cpp
res[0] = 3;
pop 0;
```

最后入栈 2：

```text
st: [2]
```

下标 2 的元素 3 右边没有更大，所以保持 -1。

------

## 关键解释

这句是核心：

```cpp
while (!st.empty() && nums[i] > nums[st.top()])
```

含义是：

> 当前元素 nums[i] 来了，它可以作为栈顶元素的“右边第一个更大”。

为什么是“第一个”？

因为我们是从左往右扫描的，当前这个 `i` 是栈顶元素右边第一个能把它弹出去的更大元素。

------

## 口诀

```text
找右边第一个更大：
从左往右扫，维护递减栈。
当前元素比栈顶大，就说明找到了栈顶的答案。
```

------

# 7️⃣ 单调栈模板二：找右边第一个更小元素

## 模板代码

```cpp
vector<int> nextSmallerElement(vector<int>& nums)
{
    int n = nums.size();
    vector<int> res(n, -1);
    stack<int> st;

    for (int i = 0; i < n; i++)
    {
        while (!st.empty() && nums[i] < nums[st.top()])
        {
            res[st.top()] = nums[i];
            st.pop();
        }

        st.push(i);
    }

    return res;
}
```

------

## 对比

找右边更大：

```cpp
nums[i] > nums[st.top()]
```

找右边更小：

```cpp
nums[i] < nums[st.top()]
```

------

## 口诀

```text
找右边更大，用递减栈。
找右边更小，用递增栈。
```

# 8️⃣ 单调栈模板三：每日温度

这是 LeetCode 739，也是单调栈最经典的题。

## 题意

给你每天温度：

```text
temperatures = [73,74,75,71,69,72,76,73]
```

求每一天还要等几天才会有更高温度。

答案：

```text
[1,1,4,2,1,1,0,0]
```

------

## 思路

这本质上是：

```text
找右边第一个更大的元素，并且返回距离
```

------

## 模板代码

```cpp
vector<int> dailyTemperatures(vector<int>& temperatures)
{
    int n = temperatures.size();
    vector<int> res(n, 0);
    stack<int> st; // 存下标

    for (int i = 0; i < n; i++)
    {
        while (!st.empty() && temperatures[i] > temperatures[st.top()])
        {
            int idx = st.top();
            st.pop();
            res[idx] = i - idx;
        }

        st.push(i);
    }

    return res;
}
```

------

## 关键解释

```cpp
res[idx] = i - idx;
```

因为：

- `idx` 是之前某一天
- `i` 是第一次出现更高温度的那一天
- 等待天数就是 `i - idx`

------

## 图示

```text
温度: 73 74 75 71 69 72 76 73
下标: 0  1  2  3  4  5  6  7
```

比如第 3 天温度 71：

```text
右边第一个比 71 高的是第 5 天 72
所以答案是 5 - 3 = 2
```

# 9️⃣ 单调栈模板四：从右往左扫描

有些题也可以从右往左做。

## 找右边第一个更大元素：右往左版本

```cpp
vector<int> nextGreaterRightToLeft(vector<int>& nums)
{
    int n = nums.size();
    vector<int> res(n, -1);
    stack<int> st; // 存值也可以

    for (int i = n - 1; i >= 0; i--)
    {
        while (!st.empty() && st.top() <= nums[i])
        {
            st.pop();
        }

        if (!st.empty())
        {
            res[i] = st.top();
        }

        st.push(nums[i]);
    }

    return res;
}
```

------

## 这个版本怎么理解？

从右往左扫时，栈里保存的是：

```text
当前位置右边可能成为答案的元素
```

如果栈顶比当前元素小或等于当前元素，它就不可能成为答案，弹掉。

------

## 图示

```text
nums = [2, 1, 3]
```

从右往左：

### i = 2，nums[i] = 3

右边没人：

```text
res[2] = -1
st = [3]
```

### i = 1，nums[i] = 1

右边第一个更大是 3：

```text
res[1] = 3
st = [3, 1]
```

### i = 0，nums[i] = 2

栈顶 1 <= 2，弹掉。

剩下 3：

```text
res[0] = 3
```

------

## 两种写法怎么选？

### 从左往右

适合：

```text
当前元素帮助前面的元素确定答案
```

### 从右往左

适合：

```text
当前元素直接从右侧候选中找答案
```

你刚开始建议先熟练**从左往右版本**，因为每日温度、下一个更大元素这类题最常用。

# 1️⃣1️⃣ 栈的第五个核心模型：用两个栈实现队列

## 队列特点

队列是：

```text
先进先出 FIFO
```

栈是：

```text
后进先出 LIFO
```

两个栈可以实现队列。

------

## 思路

准备两个栈：

```cpp
stack<int> in;
stack<int> out;
```

- 入队时，放到 `in`
- 出队时，如果 `out` 为空，就把 `in` 全部倒到 `out`
- 然后从 `out` 弹出

------

## 图示

入队：

```text
push 1, 2, 3

in 栈:
3
2
1
```

如果要出队，把 `in` 倒入 `out`：

```text
out 栈:
1
2
3
```

这时候 `out.top()` 就是最早入队的 1。

------

## 模板代码

```cpp
class MyQueue {
private:
    stack<int> inStack;
    stack<int> outStack;

    void move()
    {
        if (outStack.empty())
        {
            while (!inStack.empty())
            {
                outStack.push(inStack.top());
                inStack.pop();
            }
        }
    }

public:
    void push(int x)
    {
        inStack.push(x);
    }

    int pop()
    {
        move();
        int x = outStack.top();
        outStack.pop();
        return x;
    }

    int peek()
    {
        move();
        return outStack.top();
    }

    bool empty()
    {
        return inStack.empty() && outStack.empty();
    }
};
```

------

## 核心理解

```text
inStack 负责收集新元素
outStack 负责按队列顺序弹出
```

# 1️⃣2️⃣ 栈的第六个核心模型：用栈实现递归/回溯思想

这个你现在不用深挖，但要知道：

> 函数递归本质上使用了系统调用栈。

比如：

```cpp
void dfs(int x)
{
    if (x == 0) return;
    dfs(x - 1);
}
```

调用过程：

```text
dfs(3)
  dfs(2)
    dfs(1)
      dfs(0)
```

这些函数会一层一层压入系统栈。

------

## 图示

```text
栈顶
dfs(0)
dfs(1)
dfs(2)
dfs(3)
```

返回时从上往下弹出。

这就是为什么递归天然有“后进先出”的特点。

# 1️⃣3️⃣ 表达式求值基础模板

这个属于栈的进阶内容，先建立认知即可。

## 题型特征

```text
计算器
逆波兰表达式
括号表达式
加减乘除
```

------

## 逆波兰表达式模板

比如表达式：

```text
["2", "1", "+", "3", "*"]
```

含义是：

```text
(2 + 1) * 3 = 9
```

------

## 思路

- 遇到数字：入栈
- 遇到运算符：弹出两个数计算，再把结果入栈

------

## 模板代码

```cpp
int evalRPN(vector<string>& tokens)
{
    stack<int> st;

    for (string s : tokens)
    {
        if (s == "+" || s == "-" || s == "*" || s == "/")
        {
            int b = st.top();
            st.pop();

            int a = st.top();
            st.pop();

            if (s == "+") st.push(a + b);
            else if (s == "-") st.push(a - b);
            else if (s == "*") st.push(a * b);
            else st.push(a / b);
        }
        else
        {
            st.push(stoi(s));
        }
    }

    return st.top();
}
```

------

## 关键注意

弹出顺序很重要：

```cpp
int b = st.top(); st.pop();
int a = st.top(); st.pop();
```

因为对于减法和除法：

```text
a - b
a / b
```

不能写反。

# 1️⃣5️⃣ 栈题做题思维

看到一道题，先问自己下面几个问题。

------

## 1. 是否有“最近的未匹配元素”？

如果有，考虑栈。

典型：

```text
括号匹配
路径简化
HTML 标签匹配
```

------

## 2. 是否涉及“后进先出”？

如果有，考虑栈。

典型：


# 1️⃣9️⃣ 栈的核心口诀

```text
栈：处理“最近的、还没解决的东西”。
```

再具体一点：

```text
括号匹配：保存最近的左括号。
相邻消除：保存还没被消掉的字符。
单调栈：保存还没找到答案的元素。
两个栈实现队列：用一次倒栈改变顺序。
```

# 2️⃣0️⃣ 栈最终总结

栈不是难在语法，而是难在识别题型。

你要掌握的核心模型是：

```text
1. 普通栈：push / pop / top
2. 括号匹配：左入右出
3. 相邻消除：能消就弹，不能消就入
4. 单调栈：维护一批“还没找到答案”的元素
5. 双栈队列：用两个 LIFO 模拟 FIFO
6. 表达式求值：数字入栈，符号计算
```
