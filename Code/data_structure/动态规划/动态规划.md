# 动态规划

# DP 入门先抓这五件事

以后看到一道 DP 题，你不要急着写转移，先按这五步问自己：

## 1. 状态表示什么？

这是最核心的一步。

比如：

```cpp
dp[i]
```

到底表示：

- 前 `i` 个元素的答案？
- 以第 `i` 个元素结尾的答案？
- 到达第 `i` 个位置的方案数？
- 容量为 `i` 时的最优值？

DP 很多题错，不是转移错，而是 `dp[i]` 的含义一开始就没定义清楚。

------

## 2. 最后一步是什么？

DP 常常从“最后一步”倒推。

比如爬楼梯：

要到第 `i` 阶，最后一步可能来自：

- 第 `i - 1` 阶
- 第 `i - 2` 阶

所以：

```cpp
dp[i] = dp[i - 1] + dp[i - 2];
```

这个不是背公式，而是由“最后一步”推出来的。

------

## 3. 状态转移从哪里来？

你要能说清楚：

```cpp
dp[i] = ...
```

为什么这样转移。

比如最大子数组和：

```cpp
dp[i] = max(nums[i], dp[i - 1] + nums[i]);
```

含义是：

以 `i` 结尾的最大子数组，要么：

- 自己重新开始
- 接在前一个最大子数组后面

------

## 4. 初始条件是什么？

比如：

```cpp
dp[0] = ?
dp[1] = ?
```

初始条件不是随便写的，必须和状态定义一致。

如果 `dp[i]` 表示“到达第 i 阶的方法数”，那 `dp[0]` 通常表示“站在起点的方法数”，是 `1`。

------

## 5. 遍历顺序是什么？

有些题从前往后。

有些二维 DP 要先行后列。

背包题还要区分：

- 01 背包：倒序遍历容量
- 完全背包：正序遍历容量

------

# 经典题目及其分析

## 爬楼梯

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。

每次你可以爬 `1` 或 `2` 个台阶。你有多少种不同的方法可以爬到楼顶呢？

```cpp
class Solution
{
public:
    int climbStairs(int n)
    {
        /*
            dp[i] 表示：爬到第 i 阶楼梯的方法数

            例如：
            dp[1] = 1
            只能走：1

            dp[2] = 2
            可以走：
            1 + 1
            2
        */

        int dp[46];

        /*
            初始状态

            dp[0] = 1 的含义：
            “站在第 0 阶”本身算作一种起点状态。
            这样后面转移时会比较统一。

            dp[1] = 1：
            爬到第 1 阶只有一种方法：走 1 阶。
        */
        dp[0] = 1;
        dp[1] = 1;

        /*
            状态转移

            想到达第 i 阶，最后一步只有两种可能：

            1. 从第 i - 1 阶走 1 阶上来
               方法数是 dp[i - 1]

            2. 从第 i - 2 阶走 2 阶上来
               方法数是 dp[i - 2]

            所以：
            dp[i] = dp[i - 1] + dp[i - 2]
        */
        for (int i = 2; i <= n; i++)
        {
            dp[i] = dp[i - 1] + dp[i - 2];
        }

        // 返回爬到第 n 阶楼梯的方法数
        return dp[n];
    }
};
```

## 最长公共子序列

```cpp
#include <bits/stdc++.h>
using namespace std;

/*
    最长公共子序列 LCS 经典做法

    给定两个字符串 x 和 y，求它们的最长公共子序列长度，
    并恢复出其中一个最长公共子序列。

    **状态定义**：
        c[i][j] 表示 x 的前 i 个字符和 y 的前 j 个字符的 LCS 长度。

    注意：
        x 的第 i 个字符是 x[i - 1]
        y 的第 j 个字符是 y[j - 1]

    转移：
        1. 如果 x[i - 1] == y[j - 1]
           那么这个字符可以作为 LCS 的最后一个字符：
           c[i][j] = c[i - 1][j - 1] + 1

        2. 如果 x[i - 1] != y[j - 1]
           那么不能同时选这两个字符，只能尝试丢掉其中一个：
           c[i][j] = max(c[i - 1][j], c[i][j - 1])

    为了恢复方案，使用 b[i][j] 记录转移方向：
        1 表示来自左上角，即 x[i - 1] == y[j - 1]，当前字符被选入 LCS
        2 表示来自上方，即 c[i][j] = c[i - 1][j]
        3 表示来自左方，即 c[i][j] = c[i][j - 1]
*/

string x, y;

// c[i][j] 表示 x[0..i-1] 和 y[0..j-1] 的 LCS 长度
vector<vector<int>> c;

// b[i][j] 用于恢复路径
// 1: 左上方 ↖
// 2: 上方 ↑
// 3: 左方 ←
vector<vector<int>> b;

void lcs_length()
{
    int m = x.size();
    int n = y.size();

    // 多开一行一列，用来处理空串边界
    c.assign(m + 1, vector<int>(n + 1, 0));
    b.assign(m + 1, vector<int>(n + 1, 0));

    // c[i][0] = 0，表示 x 的前 i 个字符和空串的 LCS 长度为 0
    for (int i = 0; i <= m; i++)
    {
        c[i][0] = 0;
    }

    // c[0][j] = 0，表示空串和 y 的前 j 个字符的 LCS 长度为 0
    for (int j = 0; j <= n; j++)
    {
        c[0][j] = 0;
    }

    // 从短前缀开始，逐步填表
    for (int i = 1; i <= m; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            // 情况 1：两个前缀的最后一个字符相同
            // 那么这个字符可以加入 LCS
            if (x[i - 1] == y[j - 1])
            {
                c[i][j] = c[i - 1][j - 1] + 1;
                b[i][j] = 1; // 来自左上方，说明 x[i - 1] 被选入 LCS
            }
            else
            {
                // 情况 2：最后一个字符不同
                // 比较“丢掉 x[i - 1]”和“丢掉 y[j - 1]”哪个更优
                if (c[i - 1][j] >= c[i][j - 1])
                {
                    c[i][j] = c[i - 1][j];
                    b[i][j] = 2; // 来自上方，表示丢掉 x[i - 1]
                }
                else
                {
                    c[i][j] = c[i][j - 1];
                    b[i][j] = 3; // 来自左方，表示丢掉 y[j - 1]
                }
            }
        }
    }
}

// 根据 b 表递归恢复一个 LCS
void print_lcs(int i, int j)
{
    // 如果某个字符串已经为空，就没有公共子序列了
    if (i == 0 || j == 0)
    {
        return;
    }

    // 来自左上方，说明 x[i - 1] == y[j - 1]
    // 当前字符属于 LCS
    if (b[i][j] == 1)
    {
        print_lcs(i - 1, j - 1);

        // 注意：递归回来后再打印
        // 因为我们是从右下角往左上角追踪的，直接打印会倒序
        cout << x[i - 1];
    }
    // 来自上方，继续往上追踪
    else if (b[i][j] == 2)
    {
        print_lcs(i - 1, j);
    }
    // 来自左方，继续往左追踪
    else
    {
        print_lcs(i, j - 1);
    }
}

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> x >> y;

    lcs_length();

    int m = x.size();
    int n = y.size();

    cout << "LCS length: " << c[m][n] << "\n";

    cout << "One LCS: ";
    print_lcs(m, n);
    cout << "\n";

    return 0;
}
```

**如果两个字符串长度相差很大，这时采用如下dp方法**

```cpp
#include <bits/stdc++.h>
using namespace std;

/*
    最长公共子序列 LCS 优化版

    题目特点：
        |s| <= 1e6
        |t| <= 1e3

    普通 LCS 做法：
        dp[i][j] 表示 s 的前 i 个字符和 t 的前 j 个字符的 LCS 长度
        时间复杂度 O(|s| * |t|)
        最坏为 1e9，容易超时。

    本题优化思路：
        因为 t 很短，所以我们主要围绕短串 t 来做 DP。

    核心思想：
        对于相同长度的公共子序列，我们希望它在长串 s 中的结尾位置尽可能靠前。

        为什么？
        因为结尾越靠前，后面剩余可匹配的位置越多，
        这个方案越有机会继续扩展成更长的公共子序列。

    状态定义：
        dp[len] 表示：
            当前已经处理过 t 的某个前缀后，
            若能匹配出长度为 len 的公共子序列，
            那么这个公共子序列在 s 中“最早结束”的位置是多少。

        如果 dp[len] = INF，
        表示当前还无法匹配出长度为 len 的公共子序列。

    例如：
        dp[3] = 10 表示：
            目前能匹配出长度为 3 的公共子序列，
            并且在所有长度为 3 的方案中，
            最优方案最早可以结束在 s 的第 10 个位置。

    初始化：
        dp[0] = 0
        表示长度为 0 的公共子序列不需要匹配任何字符，
        可以认为它结束在 s 的第 0 个位置。

    转移：
        依次处理短串 t 中的每个字符 ch。

        假设当前已经能匹配出长度 len，
        并且这个序列最早结束在 s 的位置 dp[len]。

        那么如果想把 ch 接到后面，
        就要在 s 中找到：
            位置 > dp[len] 的第一个 ch。

        如果找到了这个位置 nxt，
        就可以更新：
            dp[len + 1] = min(dp[len + 1], nxt) 
        保证始终是s中靠前，所以用min

    如何快速找 nxt？
        预处理长串 s 中每个字符出现的位置数组 pos[c]。
        然后用 upper_bound 找到第一个大于 dp[len] 的位置。

    为什么 len 要倒序枚举？
        因为 t 中每个字符只能使用一次。

        如果正序枚举 len，
        当前这个 t 字符可能先更新 dp[len + 1]，
        然后又被用来更新 dp[len + 2]，
        这就相当于同一个字符被重复使用了。

        所以和 01 背包一样，要倒序枚举。
*/

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s, t;
    cin >> s >> t;

    int n = s.size();
    int m = t.size();

    /*
        pos[c] 存储字符 c 在长串 s 中出现的所有位置。

        这里使用 1-based 位置：
            s[0] 的位置记为 1
            s[1] 的位置记为 2
            ...

        这样 dp[0] = 0 就很自然：
            长度为 0 的公共子序列结束在位置 0，
            后续要找的位置必须 > 0。
    */
    vector<int> pos[26];

    for (int i = 0; i < n; i++)
    {
        int c = s[i] - 'a';
        pos[c].push_back(i + 1);
    }

    /*
        INF 表示无法匹配。

        因为 s 的最大合法位置是 n，
        所以 n + 1 可以作为无穷大。
    */
    const int INF = n + 1;

    /*
        dp[len] 表示：
            匹配出长度为 len 的公共子序列时，
            它在 s 中最早的结束位置。

        len 的最大值不会超过 m，因为公共子序列长度不可能超过短串 t 的长度。
    */
    vector<int> dp(m + 1, INF);

    // 长度为 0 的公共子序列，认为结束在 s 的第 0 个位置
    dp[0] = 0;

    /*
        ans 表示当前已经能够匹配出的最大公共子序列长度。

        它不是必须的，但可以让内层循环只枚举到 ans，
        不必每次都枚举到 m。
    */
    int ans = 0;

    /*
        依次处理短串 t 中的每个字符。

        当前处理到的字符 ch，可以选择：
            1. 不用它；
            2. 把它接到某个已有公共子序列后面，使长度增加 1。
    */
    for (char ch : t)
    {
        int c = ch - 'a';

        /*
            倒序枚举当前已经能匹配出的长度 len。

            如果 dp[len] 有效，说明我们已经能匹配出长度 len，
            并且最早结束位置是 dp[len]。

            现在尝试把当前字符 ch 接到它后面。
        */
        for (int len = ans; len >= 0; len--)
        {
            // 如果当前长度 len 还无法匹配出来，跳过
            if (dp[len] == INF)
            {
                continue;
            }

            /*
                在 s 中找到第一个位置 nxt，满足：
                    nxt > dp[len]
                    s[nxt] == ch

                由于 pos[c] 是有序数组，可以用 upper_bound。
            */
            auto it = upper_bound(pos[c].begin(), pos[c].end(), dp[len]);

            // 如果找到了这样的字符位置
            if (it != pos[c].end())
            {
                int nxt = *it;

                /*
                    用当前字符 ch 把长度 len 的公共子序列扩展成长度 len + 1。

                    dp[len + 1] 要取最小位置，
                    因为同样长度下，结尾越靠前越好。
                */
                if (nxt < dp[len + 1])
                {
                    dp[len + 1] = nxt;

                    // 更新当前能达到的最大长度
                    ans = max(ans, len + 1);
                }
            }
        }
    }

    /*
        ans 就是最长公共子序列长度。

        因为 dp[len] 有效表示存在长度为 len 的公共子序列，
        我们一直维护最大有效 len。
    */
    cout << ans << "\n";

    return 0;
}
```



## 01背包

```cpp
#include <bits/stdc++.h>
using namespace std;

/*
    经典 01 背包问题

    有 n 件物品，一个容量为 V 的背包。
    第 i 件物品的体积为 v[i]，价值为 w[i]。
    每件物品最多只能选一次。

    目标：
        在总体积不超过 V 的前提下，使总价值最大。

    状态定义：
        dp[j] 表示背包容量为 j 时，能够获得的最大价值。

    转移：
        对于第 i 件物品，有两种选择：

        1. 不选第 i 件物品：
           dp[j] 不变。

        2. 选第 i 件物品：
           前提是 j >= v[i]。
           那么剩余容量是 j - v[i]，
           新价值是 dp[j - v[i]] + w[i]。

        所以：
           dp[j] = max(dp[j], dp[j - v[i]] + w[i])

    为什么容量 j 要倒序枚举？

        因为每件物品只能选一次。

        如果正序枚举：
            dp[j - v[i]] 可能已经在本轮被第 i 件物品更新过，
            那么再用它更新 dp[j]，就等于第 i 件物品被用了多次。

        倒序枚举可以保证：
            dp[j - v[i]] 还是上一轮的结果，
            也就是还没有使用第 i 件物品时的结果。

        所以 01 背包必须倒序枚举容量。
*/

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, V;
    cin >> n >> V;

    vector<int> v(n + 1); // v[i] 表示第 i 件物品的体积
    vector<int> w(n + 1); // w[i] 表示第 i 件物品的价值

    for (int i = 1; i <= n; i++)
    {
        cin >> v[i] >> w[i];
    }

    // dp[j] 表示容量为 j 时的最大价值
    vector<int> dp(V + 1, 0);

    // 外层枚举每一件物品，它只有选或不选两种可能
    for (int i = 1; i <= n; i++)
    {
        // 01 背包容量必须倒序枚举
        // 从 V 枚举到 v[i]，保证第 i 件物品最多只被选一次
        for (int j = V; j >= v[i]; j--)
        {
            // 不选第 i 件物品：dp[j]
            // 选第 i 件物品：dp[j - v[i]] + w[i]
            dp[j] = max(dp[j], dp[j - v[i]] + w[i]);
        }
    }

    // dp[V] 就是容量不超过 V 时能获得的最大价值
    cout << dp[V] << "\n";

    return 0;
}
```

另一个版本，dp总价值

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

/*
    01 背包优化版：按价值做 DP

    题目特点：
        n <= 500
        V <= 1e9
        v[i] <= 1e9
        sum(w[i]) <= 1e6

    经典 01 背包通常定义：
        dp[容量] = 最大价值

    但这题容量 V 最大为 1e9，不能开 dp[V + 1]。

    所以反过来定义：
        dp[价值] = 达到该价值所需的最小体积

    最后找最大的价值 value，使得：
        dp[value] <= V

    这就是“容量太大，价值较小”时常用的背包优化思路。
*/

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    ll V;

    cin >> n >> V;

    // v[i] 表示第 i 件物品的体积
    // 体积可能很大，并且会累加，所以用 long long
    vector<ll> v(n + 1);

    // w[i] 表示第 i 件物品的价值
    // 题目保证总价值 <= 1e6，所以 int 足够
    vector<int> w(n + 1);

    // sumw 表示所有物品价值之和
    // 后面 dp 数组的大小就是 sumw + 1
    int sumw = 0;

    for (int i = 1; i <= n; i++)
    {
        cin >> v[i] >> w[i];
        sumw += w[i];
    }

    // INF 表示一个很大的体积
    // 因为最大总体积可能达到 500 * 1e9 = 5e11
    // 所以 4e18 足够大
    const ll INF = 4e18;

    /*
        dp[j] 表示：
            凑出总价值 j 所需要的最小总体积。

        如果 dp[j] = INF，表示当前还无法凑出价值 j。
    */
    vector<ll> dp(sumw + 1, INF);

    // 凑出价值 0 不需要选任何物品，体积为 0
    dp[0] = 0;

    /*
        枚举每一件物品。

        因为每件物品只能选一次，所以这是 01 背包。

        注意：
            这里倒序枚举的是“价值 j”，不是容量。
            倒序的目的仍然是防止同一件物品被重复使用。
    */
    for (int i = 1; i <= n; i++)
    {
        /*
            j 表示当前想凑出的总价值。

            如果选第 i 件物品，它贡献价值 w[i]，
            那么之前需要凑出价值 j - w[i]。

            所以转移为：
                dp[j] = min(dp[j], dp[j - w[i]] + v[i])
        */
        for (int j = sumw; j >= w[i]; j--)
        {
            dp[j] = min(dp[j], dp[j - w[i]] + v[i]);
        }
    }

    /*
        现在 dp[j] 表示达到价值 j 的最小体积。

        题目要求总体积不超过 V，且价值最大。

        所以从大到小枚举价值 j，
        找到第一个满足 dp[j] <= V 的 j。
    */
    int ans = 0;

    for (int j = sumw; j >= 0; j--)
    {
        if (dp[j] <= V)
        {
            ans = j;
            break;
        }
    }

    cout << ans << "\n";

    return 0;
}
```

**总结：**这两个版本的`dp`对象不同，具体应用看数据，那个小`dp`哪个

## 完全背包

```cpp
class Solution {
public:
    int change(int amount, vector<int>& coins) {
        /*
            题目：518. 零钱兑换 II

            给定：
            - 一个总金额 amount
            - 若干种硬币 coins
            - 每种硬币可以使用无限次

            问：
            凑成 amount 的组合数有多少？

            注意：
            这里统计的是“组合数”，不是“排列数”。

            例如 coins = [1, 2]，amount = 3

            组合：
            1 + 1 + 1
            1 + 2

            其中 1 + 2 和 2 + 1 算同一种组合。
        */

        /*
            dp[j] 表示：
            使用已经遍历过的硬币，凑出金额 j 的组合数。

            例如：
            dp[5] = 3 表示：
            在当前已经考虑过的硬币种类中，
            凑出金额 5 有 3 种组合方式。
        */
        vector<unsigned long long> dp(amount + 1, 0);

        /*
            初始化：

            dp[0] = 1

            含义：
            凑出金额 0 有 1 种方式：什么都不选。

            这是计数型背包的基础状态。
            如果没有这个起点，后面的 dp[j] 都无法被转移出来。
        */
        dp[0] = 1;

        /*
            完全背包与 01 背包的核心区别：

            --------------------------------------------------
            01 背包：
            每个物品只能选一次。

            写法：
                for (int x : nums) {
                    for (int j = target; j >= x; j--) {
                        dp[j] += dp[j - x];
                    }
                }

            为什么倒序？
            因为倒序可以保证 dp[j - x] 还是“没有使用当前物品 x 之前”的状态，
            从而避免当前物品在同一轮中被重复使用。

            --------------------------------------------------
            完全背包：
            每个物品可以选无限次。

            写法：
                for (int coin : coins) {
                    for (int j = coin; j <= amount; j++) {
                        dp[j] += dp[j - coin];
                    }
                }

            为什么正序？
            因为正序时，dp[j - coin] 可能已经在本轮被当前 coin 更新过，
            这正好表示当前 coin 可以被重复使用。

            --------------------------------------------------
            本题是完全背包：
            因为每种硬币可以使用无限次。
        */

        /*
            外层遍历硬币，内层遍历金额。

            这样统计的是“组合数”。

            为什么？

            因为硬币是一种一种加入的。
            比如先处理 coin = 1，再处理 coin = 2。
            那么组合 1 + 2 只会在处理 coin = 2 时形成一次，
            不会再把 2 + 1 当成另一种方案重复统计。

            如果外层遍历金额、内层遍历硬币，
            那更容易统计成“排列数”，顺序不同也可能被重复计算。
        */
        for (int coin : coins) {
            /*
                完全背包：容量正序遍历。

                j 从 coin 开始，
                因为只有当当前金额 j >= coin 时，
                才能使用这个 coin。
            */
            for (int j = coin; j <= amount; j++) {
                /*
                    转移：

                    dp[j] += dp[j - coin]

                    含义：
                    如果要凑出金额 j，并且最后使用了一个 coin，
                    那么在使用这个 coin 之前，需要先凑出 j - coin。

                    凑出 j - coin 的所有组合，
                    每一种后面加上当前 coin，
                    都可以形成一种凑出 j 的组合。
                */
                dp[j] += dp[j - coin];
            }
        }

        /*
            说明：

            这里使用 unsigned long long，
            是因为力扣某些测试点中间状态可能非常大，
            用 int 或 long long 可能触发溢出报错。

            虽然题目最终答案保证可以放进 int，
            但中间 dp[j] 可能远大于最终 dp[amount]。
        */
        return (int)dp[amount];
    }
};
```

## 背包+最小值

```cpp
class Solution {
public:
    int coinChange(vector<int>& coins, int amount) {
        /*
            题目：322. 零钱兑换

            给定：
            - 若干种硬币 coins
            - 一个目标金额 amount
            - 每种硬币可以使用无限次

            要求：
            凑出 amount 所需要的最少硬币数。
            如果凑不出来，返回 -1。

            这是一道典型的：

                完全背包 + 最小值问题

            和 518 零钱兑换 II 的区别是：

            518 问：凑出 amount 有多少种组合
            322 问：凑出 amount 最少需要多少枚硬币
        */

        /*
            dp[j] 表示：

                凑出金额 j 所需要的最少硬币数。

            例如：
            dp[11] = 3 表示：
            凑出金额 11 最少需要 3 枚硬币。
        */

        /*
            初始化是这题最容易错的地方。

            对于金额 0：
                凑出 0 元不需要任何硬币，所以 dp[0] = 0。

            对于其他金额：
                一开始我们还不知道能不能凑出来，
                所以要先设置成一个很大的数 INF，
                表示“暂时不可达”。

            不能初始化成 0。

            如果 dp[j] 初始化成 0，
            那它会被误解为：
                凑出金额 j 最少需要 0 枚硬币。

            这显然不对。
        */
        int INF = amount + 1;
        vector<int> dp(amount + 1, INF);

        // 凑出金额 0 最少需要 0 枚硬币
        dp[0] = 0;

        /*
            完全背包写法：

            外层遍历硬币，
            内层遍历金额。

            因为每种硬币可以使用无限次，
            所以内层金额要正序遍历。

            对比 01 背包：

            01 背包：
                每个物品只能使用一次
                所以容量倒序遍历

                for (int j = target; j >= x; j--)

            完全背包：
                每个物品可以使用多次
                所以容量正序遍历

                for (int j = coin; j <= amount; j++)

            正序遍历的意义是：
            dp[j - coin] 可以是本轮刚刚更新过的状态，
            这就允许当前 coin 被重复使用。
        */
        for (int coin : coins) {
            for (int j = coin; j <= amount; j++) {
                /*
                    状态转移：

                    如果最后使用了一枚 coin，
                    那么在使用这枚 coin 之前，
                    需要先凑出金额 j - coin。

                    凑出 j - coin 的最少硬币数是：
                        dp[j - coin]

                    加上当前这枚 coin，
                    硬币数就是：
                        dp[j - coin] + 1

                    所以：
                        dp[j] = min(dp[j], dp[j - coin] + 1)

                    含义：
                    原来凑出 j 的方案，和
                    “先凑出 j - coin，再加一枚 coin” 的方案，
                    取硬币数更少的那个。
                */
                dp[j] = min(dp[j], dp[j - coin] + 1);
            }
        }

        /*
            如何判断凑不出来？

            如果所有硬币都处理完后，
            dp[amount] 仍然是 INF，
            说明没有任何组合能凑出 amount。

            此时返回 -1。
        */
        if (dp[amount] == INF) {
            return -1;
        }

        return dp[amount];
    }
};
```

## 背包总结

```cpp
/*
    背包 DP 常见三类问题总结

    背包题不只是“选不选物品”，更关键的是：
    dp[j] 存的到底是什么？

    常见有三类：

    1. 可行性问题：能不能凑出 j
    2. 计数问题：凑出 j 有多少种方法
    3. 最值问题：凑出 j 的最大/最小代价是多少

    不同类型的 dp 初始化和转移完全不同。
*/


/* =========================================================
   一、可行性问题：能不能凑出 j
   =========================================================

   dp[j] 表示：
       能不能从已经处理过的物品中，选出若干个，使它们的和为 j。

   典型题：
       416. 分割等和子集

   初始化：
       dp[j] = false;
       dp[0] = true;

   含义：
       一开始所有正容量都凑不出来，所以是 false。
       什么都不选，可以凑出 0，所以 dp[0] = true。

   转移：
       dp[j] = dp[j] || dp[j - x];

   含义：
       能凑出 j，有两种可能：
       1. 原来就能凑出 j
       2. 原来能凑出 j - x，现在选上 x
*/

vector<bool> dp(target + 1, false);
dp[0] = true;

for (int x : nums) {
    // 01 背包：每个 x 只能用一次，所以倒序
    for (int j = target; j >= x; j--) {
        dp[j] = dp[j] || dp[j - x];
    }
}


/* =========================================================
   二、计数问题：凑出 j 有多少种方法
   =========================================================

   dp[j] 表示：
       从已经处理过的物品中，选出若干个，使它们的和为 j 的方案数。

   典型题：
       494. 目标和
       518. 零钱兑换 II

   初始化：
       dp[j] = 0;
       dp[0] = 1;

   含义：
       一开始凑出正容量的方法数是 0。
       什么都不选，凑出 0，有 1 种方法。

   转移：
       dp[j] += dp[j - x];

   含义：
       如果要凑出 j，并且选了当前物品 x，
       那么之前需要凑出 j - x。
       所以凑出 j 的方案数增加 dp[j - x]。
*/


/* ---------- 计数型 01 背包 ----------
   每个物品只能用一次。

   典型题：
       494. 目标和

   容量倒序：
       防止同一个物品在一轮中被重复使用。
*/

vector<int> dp(bag + 1, 0);
dp[0] = 1;

for (int x : nums) {
    for (int j = bag; j >= x; j--) {
        dp[j] += dp[j - x];
    }
}


/* ---------- 计数型完全背包 ----------
   每个物品可以用无限次。

   典型题：
       518. 零钱兑换 II

   容量正序：
       允许当前物品在本轮中被重复使用。
*/

vector<unsigned long long> dp(amount + 1, 0);
dp[0] = 1;

for (int coin : coins) {
    for (int j = coin; j <= amount; j++) {
        dp[j] += dp[j - coin];
    }
}


/* =========================================================
   三、最小值问题：凑出 j 的最小代价
   =========================================================

   dp[j] 表示：
       凑出状态 j 所需要的最小代价。

   典型题：
       322. 零钱兑换

   初始化：
       dp[j] = INF;
       dp[0] = 0;

   含义：
       一开始正容量都视为“不可达”，所以设为 INF。
       凑出 0 不需要任何物品，所以 dp[0] = 0。

   转移：
       dp[j] = min(dp[j], dp[j - x] + cost);

   对于 322：
       cost = 1，表示多用一枚硬币。
*/

const int INF = amount + 1;
vector<int> dp(amount + 1, INF);
dp[0] = 0;

for (int coin : coins) {
    // 完全背包：硬币可以重复使用，所以正序
    for (int j = coin; j <= amount; j++) {
        dp[j] = min(dp[j], dp[j - coin] + 1);
    }
}

return dp[amount] == INF ? -1 : dp[amount];


/* =========================================================
   四、最大值问题：容量 j 下的最大价值
   =========================================================

   dp[j] 表示：
       容量不超过 j 时，能够获得的最大价值。

   典型：
       经典 01 背包

   初始化：
       dp[j] = 0;

   含义：
       什么都不选，价值是 0。
       在“容量不超过 j”的定义下，0 是合法初始值。

   转移：
       dp[j] = max(dp[j], dp[j - weight] + value);
*/

vector<int> dp(capacity + 1, 0);

for (int i = 0; i < n; i++) {
    int w = weight[i];
    int v = value[i];

    // 01 背包：每个物品只能选一次，所以倒序
    for (int j = capacity; j >= w; j--) {
        dp[j] = max(dp[j], dp[j - w] + v);
    }
}


/* =========================================================
   五、以价值为状态的最小代价 DP
   =========================================================

   有时候也会反过来定义：

       dp[v] 表示：
           达到价值 v 所需要的最小重量。

   这种时候，它也是“最小值问题”。

   初始化：
       dp[v] = INF;
       dp[0] = 0;

   含义：
       一开始只有价值 0 可以用重量 0 达成。
       其他价值暂时都不可达。

   转移：
       dp[v] = min(dp[v], dp[v - value[i]] + weight[i]);
*/

const int INF = 1e9;
vector<int> dp(maxValue + 1, INF);
dp[0] = 0;

for (int i = 0; i < n; i++) {
    int w = weight[i];
    int v = value[i];

    // 01 背包，物品只能用一次，所以价值倒序
    for (int val = maxValue; val >= v; val--) {
        dp[val] = min(dp[val], dp[val - v] + w);
    }
}


/* =========================================================
   六、01 背包 vs 完全背包：遍历方向
   =========================================================

   01 背包：
       每个物品只能使用一次。
       容量必须倒序。

       for (int j = target; j >= x; j--)

   原因：
       防止当前物品在同一轮中被重复使用。

   ---------------------------------------------------------

   完全背包：
       每个物品可以使用无限次。
       容量必须正序。

       for (int j = x; j <= target; j++)

   原因：
       允许 dp[j - x] 使用本轮刚更新过的状态，
       也就是允许当前物品重复使用。
*/


/* =========================================================
   七、背包题快速判断模板
   =========================================================

   看到背包题，先问 4 个问题：

   1. 每个物品能用几次？
      - 只能用一次：01 背包，容量倒序
      - 可以无限用：完全背包，容量正序

   2. dp[j] 表示什么？
      - 能不能凑出 j：bool
      - 凑出 j 有几种方法：int / long long
      - 凑出 j 的最小代价：INF 初始化
      - 容量 j 下最大价值：0 初始化

   3. dp[0] 应该是什么？
      - 可行性：dp[0] = true
      - 计数：dp[0] = 1
      - 最小值：dp[0] = 0
      - 最大值：dp[0] = 0

   4. 其他状态怎么初始化？
      - 可行性：false
      - 计数：0
      - 最小值：INF
      - 最大值：通常 0，具体看状态含义
*/


/* =========================================================
   八、一句话总结
   =========================================================

   背包 DP 的初始化不是死记模板，
   而是由 dp[j] 的语义决定的：

   - dp[j] 表示“能不能”：初始 false，dp[0] = true
   - dp[j] 表示“有几种”：初始 0，dp[0] = 1
   - dp[j] 表示“最小代价”：初始 INF，dp[0] = 0
   - dp[j] 表示“最大收益”：通常初始 0

   然后再根据物品能不能重复使用，
   决定容量是倒序还是正序。
*/
```

## 最长递增子序列

给你一个整数数组 `nums` ，找到其中最长严格递增子序列的长度。

**子序列** 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，`[3,6,2,7]` 是数组 `[0,3,1,6,2,2,7]` 的子序列。

```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        int n = nums.size();

        /*
            dp[i] 表示：
                以 nums[i] 结尾的最长递增子序列长度。

            注意关键词是“以 nums[i] 结尾”。

            例如：
                nums = [10, 9, 2, 5, 3, 7]

            如果 i 指向 7，那么 dp[i] 表示：
                必须以 7 作为最后一个元素时，
                能得到的最长递增子序列长度。

            可能是：
                [2, 5, 7]
            或者：
                [2, 3, 7]
            长度都是 3。
        */
        vector<int> dp(n, 1);

        /*
            pre[i] 表示：
                在以 nums[i] 结尾的某个最优 LIS 中，
                nums[i] 前一个元素的下标。

            如果 pre[i] = -1，
            表示 nums[i] 前面没有元素，
            即 nums[i] 自己就是这个递增子序列的开头。
        */
        vector<int> pre(n, -1);

        /*
            ans 记录目前找到的 LIS 最大长度。

            pos 记录这个最长递增子序列的最后一个元素下标。
            后面恢复路径时，就从 pos 开始沿着 pre 往前找。
        */
        int ans = 1;
        int pos = 0;

        /*
            枚举每一个位置 i。

            我们要计算：
                如果 LIS 必须以 nums[i] 结尾，
                那么它的前一个元素可以是谁？
        */
        for (int i = 0; i < n; i++) {

            /*
                枚举 i 前面的所有位置 j。

                如果 nums[j] < nums[i]，
                说明 nums[i] 可以接在 nums[j] 后面，
                从而形成一个更长的递增子序列。
            */
            for (int j = 0; j < i; j++) {

                /*
                    只有 nums[j] < nums[i] 时，
                    nums[i] 才能接到以 nums[j] 结尾的 LIS 后面。
                */
                if (nums[j] < nums[i]) {

                    /*
                        如果接在 nums[j] 后面能得到更长的序列，
                        就更新 dp[i]。

                        原来：
                            dp[i] 表示当前已知的、以 nums[i] 结尾的最长长度。

                        新方案：
                            以 nums[j] 结尾的 LIS 长度为 dp[j]，
                            接上 nums[i] 后长度变为 dp[j] + 1。
                    */
                    if (dp[j] + 1 > dp[i]) {
                        dp[i] = dp[j] + 1;

                        /*
                            记录前驱。

                            这句话表示：
                                当前最优的、以 nums[i] 结尾的 LIS，
                                是从 nums[j] 接过来的。
                        */
                        pre[i] = j;
                    }
                }
            }

            /*
                LIS 不一定以最后一个元素结尾，
                所以每算完一个 dp[i]，都要更新全局答案。
            */
            if (dp[i] > ans) {
                ans = dp[i];
                pos = i;
            }
        }

        /*
            恢复一个 LIS。

            从最长序列的最后一个位置 pos 出发，
            不断沿着 pre[pos] 往前跳。

            由于是从后往前找，所以恢复出来的 res 是反的。
        */
        vector<int> res;

        while (pos != -1) {
            res.push_back(nums[pos]);
            pos = pre[pos];
        }

        /*
            反转后，res 就是从前到后的一个 LIS。
        */
        reverse(res.begin(), res.end());

        /*
            LeetCode 300 只要求返回 LIS 长度，
            所以这里返回 ans。

            如果你在算法导论练习里需要输出具体序列，
            res 里面就是恢复出来的一个最长递增子序列。
        */
        return ans;
    }
};
```

**优化版**

```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        /*
            tail[len - 1] 表示：

                长度为 len 的递增子序列中，所有可能方案的“最小结尾值”。

            例如：
                tail[0] 表示长度为 1 的递增子序列的最小结尾值；
                tail[1] 表示长度为 2 的递增子序列的最小结尾值；
                tail[2] 表示长度为 3 的递增子序列的最小结尾值。

            为什么要保留“最小结尾值”？

                对于相同长度的递增子序列，结尾越小越好。
                因为结尾越小，后面越容易接上更大的数。

            注意：
                tail 本身不一定是某个真实的 LIS，
                它只是维护每种长度下最有扩展潜力的结尾值。
        */
        vector<int> tail;

        /*
            依次扫描 nums 中的每个数 x。

            我们要判断 x 可以放到 tail 的哪个位置。
        */
        for (int x : nums) {
            /*
                lower_bound 找到 tail 中第一个 >= x 的位置。

                因为题目要求“严格递增”，所以相等的数不能接在后面。

                如果 tail 中存在某个数 >= x，
                就用 x 替换它。

                这表示：
                    对于这个长度的递增子序列，
                    我们找到了一个更小的结尾 x，
                    后续扩展能力更强。
            */
            auto it = lower_bound(tail.begin(), tail.end(), x);

            /*
                如果没有找到 >= x 的位置，
                说明 x 比 tail 中所有结尾值都大。

                那么 x 可以接在当前最长递增子序列后面，
                使长度增加 1。
            */
            if (it == tail.end()) {
                tail.push_back(x);
            }
            /*
                否则，用 x 替换第一个 >= x 的元素。

                注意：
                    这里不是改变 LIS 的长度，
                    而是优化某个长度下的最小结尾值。
            */
            else {
                *it = x;
            }
        }

        /*
            tail 的长度就是 LIS 的长度。

            因为 tail 中每多一个元素，
            就说明我们成功构造出了更长一位的递增子序列。
        */
        return tail.size();
    }
};
```

## 打家劫舍

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，**如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警**。

给定一个代表每个房屋存放金额的非负整数数组，计算你 **不触动警报装置的情况下** ，一夜之内能够偷窃到的最高金额。

```cpp
class Solution {
public:
    int rob(vector<int>& nums) {
        int n = nums.size();

        /*
            dp[i] 表示：
                考虑前 i 间房子时，能够偷到的最大金额。

            注意：
                这里的“前 i 间房子”对应 nums[0] 到 nums[i - 1]。

            所以：
                dp[0] 表示没有房子，最大金额为 0。
                dp[1] 表示只考虑第 0 间房子，最大金额为 nums[0]。
        */
        vector<int> dp(n + 1, 0);

        dp[0] = 0;
        dp[1] = nums[0];

        /*
            从第 2 间房子开始考虑。

            当我们计算 dp[i] 时，实际上是在考虑 nums[i - 1] 这间房子。
        */
        for (int i = 2; i <= n; i++) {
            /*
                对于第 i 间房子 nums[i - 1]，有两种选择：

                1. 不偷它：
                   那么最大金额就是考虑前 i - 1 间房子的结果：
                   dp[i - 1]

                2. 偷它：
                   因为相邻房子不能同时偷，
                   所以前一间 nums[i - 2] 不能偷。
                   因此只能加上考虑前 i - 2 间房子的最大金额：
                   dp[i - 2] + nums[i - 1]

                二者取最大值。
            */
            dp[i] = max(dp[i - 1], dp[i - 2] + nums[i - 1]);
        }

        /*
            dp[n] 表示考虑全部 n 间房子时能偷到的最大金额。
        */
        return dp[n];
    }
};
```

## 不同路径II

二维 DP 写出状态转移后，必须检查边界位置是否也满足这个转移。很多错不是主思路错，而是边界状态没有单独处理。

```cpp
class Solution {
public:
    int uniquePathsWithObstacles(vector<vector<int>>& obstacleGrid) {
        /*
            题目：不同路径 II

            机器人从左上角 (0, 0) 出发，
            每次只能向右或向下走，
            要到达右下角 (m - 1, n - 1)。

            obstacleGrid[i][j] == 1 表示障碍物，不能经过；
            obstacleGrid[i][j] == 0 表示空地，可以经过。

            要求：从左上角到右下角共有多少条不同路径。
        */

        int m = obstacleGrid.size();
        int n = obstacleGrid[0].size();

        /*
            dp[i][j] 表示：
            从起点 (0, 0) 走到位置 (i, j) 的不同路径数。

            这类二维路径 DP 的核心：
            走到当前位置 (i, j)，最后一步只有两种可能：

            1. 从上方 (i - 1, j) 走下来
            2. 从左方 (i, j - 1) 走过来

            所以如果当前位置不是障碍物：
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1]

            如果当前位置是障碍物：
            dp[i][j] = 0
        */
        vector<vector<int>> dp(m, vector<int>(n, 0)); //全部初始化为0

        /*
            如果起点就是障碍物，那么机器人一开始就无法出发，
            直接返回 0。
        */
        if (obstacleGrid[0][0] == 1) {
            return 0;
        }

        /*
            起点不是障碍物时，
            从起点到起点本身，算作 1 种路径。
        */
        dp[0][0] = 1;

        /*
            初始化第一列。

            第一列的位置只能从上方一路往下走到达，
            没有“从左边来”的可能。

            所以：
            - 如果当前格子不是障碍物，并且上方能到达，那么它也能到达
            - 一旦遇到障碍物，后面的格子都无法到达

            因此遇到障碍物后可以直接 break。
        */
        for (int i = 1; i < m; i++) {
            if (obstacleGrid[i][0] != 1) {
                dp[i][0] = 1;
            } else {
                break;
            }
        }

        /*
            初始化第一行。

            第一行的位置只能从左边一路往右走到达，
            没有“从上方来”的可能。

            同理：
            - 没遇到障碍物之前，路径数都是 1
            - 一旦遇到障碍物，右边的格子都无法到达

            因此遇到障碍物后可以直接 break。
        */
        for (int j = 1; j < n; j++) {
            if (obstacleGrid[0][j] != 1) {
                dp[0][j] = 1;
            } else {
                break;
            }
        }

        /*
            从 (1, 1) 开始填表。

            对于普通位置 (i, j)：
            如果它不是障碍物，那么路径数来自：
            - 上方 dp[i - 1][j]
            - 左方 dp[i][j - 1]

            如果它是障碍物，那么保持 0。
        */
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (obstacleGrid[i][j] != 1) {
                    dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
                }
            }
        }

        /*
            右下角 dp[m - 1][n - 1]
            就是从左上角到右下角的不同路径数。
        */
        return dp[m - 1][n - 1];
    }
};
```

## 分割等和子集

```cpp
class Solution {
public:
    bool canPartition(vector<int>& nums) {
        /*
            题目：416. 分割等和子集

            要判断能不能把数组分成两个子集，
            使得两个子集的元素和相等。

            如果总和是 sum，那么两个子集的和都必须是 sum / 2。
            所以问题可以转化为：

                能不能从 nums 中选出一些数，
                使它们的和恰好等于 sum / 2？

            这就是一个典型的 01 背包可行性问题。
        */

        int sum = 0;

        // 先计算数组总和
        for (int x : nums) {
            sum += x;
        }

        /*
            如果总和是奇数，不可能平均分成两个整数和相等的子集。
            例如 sum = 11，就不可能分成 5.5 和 5.5。
        */
        if (sum % 2 == 1) {
            return false;
        }

        // 目标：判断能不能凑出 target
        int target = sum / 2;

        /*
            dp[j] 表示：
            在已经处理过的数字中，能不能选出一些数，使它们的和恰好为 j。

            例如：
            dp[5] == true 表示：
            当前已经看过的这些数里，可以凑出和为 5 的子集。
        */
        vector<bool> dp(target + 1, false);

        /*
            dp[0] = true

            含义：
            什么数都不选，就可以凑出和 0。

            这是背包可行性问题的起点。
        */
        dp[0] = true;

        /*
            依次考虑每一个数字 x。

            对于每个 x，有两种选择：
            1. 不选 x
               那么 dp[j] 保持原来的状态。

            2. 选 x
               如果之前可以凑出 j - x，
               那么现在加上 x，就可以凑出 j。

            所以转移是：

                dp[j] = dp[j] || dp[j - x]

            含义：
            能凑出 j，要么原来就能凑出 j，
            要么原来能凑出 j - x，现在选上 x 后凑出 j。
        */
        for (int x : nums) {
            /*
                注意：这里必须倒序遍历。

                因为每个数字只能使用一次，这是 01 背包。

                如果正序遍历，可能会在同一轮中重复使用当前这个 x。

                比如 x = 3：
                正序时可能先用 x 更新 dp[3]，
                然后又用刚刚更新出来的 dp[3] 去更新 dp[6]，
                这就相当于同一个 3 被用了两次。

                倒序遍历可以保证：
                dp[j - x] 还是上一轮的状态，
                也就是没有使用当前 x 之前的状态。
            */
            for (int j = target; j >= x; j--) {
                dp[j] = dp[j] || dp[j - x];
            }
        }

        /*
            最后看能不能凑出 target。

            如果 dp[target] == true，
            说明可以选出一部分数和为 sum / 2，
            那剩下的数自然也是 sum / 2，
            所以可以分割成两个等和子集。
        */
        return dp[target];
    }
};
```

## 高楼抛鸡蛋

```cpp
#include <bits/stdc++.h>
using namespace std;

/**
 * @brief 高楼抛鸡蛋
 *
 * 【题目理解】
 *
 * 有 k 枚鸡蛋，n 层楼。
 *
 * 存在一个临界楼层 f：
 *
 * - 从高于 f 的楼层扔鸡蛋，鸡蛋一定会碎；
 * - 从 f 或低于 f 的楼层扔鸡蛋，鸡蛋一定不会碎。
 *
 * 目标：
 *
 *     在最坏情况下，至少需要多少次操作，
 *     才能确定 f 的确切值。
 *
 * --------------------------------------------------
 * 【为什么不能简单二分】
 *
 * 如果鸡蛋无限多，可以二分。
 *
 * 但鸡蛋数量有限。
 *
 * 例如只有 1 枚鸡蛋时，如果从很高的楼层扔碎了，
 * 就没有鸡蛋继续测试下面楼层了。
 *
 * 所以鸡蛋少的时候，必须更加保守。
 *
 * 这道题的本质是：
 *
 *     在有限鸡蛋数下，设计一种最坏情况也能确定答案的策略。
 *
 * --------------------------------------------------
 * 【DP 思路转换】
 *
 * 直接想：
 *
 *     dp[k][n] = k 个鸡蛋、n 层楼最少需要几次
 *
 * 会比较麻烦，因为第一次从哪层扔需要枚举。
 *
 * 更巧妙的思路是反过来想：
 *
 *     给定鸡蛋数和操作次数，最多能确定多少层楼？
 *
 * --------------------------------------------------
 * 【状态定义】
 *
 * dp[egg] 表示：
 *
 *     当前已经允许操作 t 次时，
 *     使用 egg 个鸡蛋，最多可以确定多少层楼。
 *
 * 注意：
 *
 * 这里的 t 没有写进数组维度里，
 * 因为我们会一轮一轮增加操作次数 t，
 * 用一维数组滚动更新。
 *
 * --------------------------------------------------
 * 【二维形式的状态其实是】
 *
 * 如果写成二维，就是：
 *
 *     f[t][egg]
 *
 * 表示：
 *
 *     有 egg 个鸡蛋，最多操作 t 次，
 *     最多能确定多少层楼。
 *
 * 转移是：
 *
 *     f[t][egg] = f[t - 1][egg - 1] + 1 + f[t - 1][egg]
 *
 * --------------------------------------------------
 * 【转移为什么成立】
 *
 * 假设现在有 egg 个鸡蛋，最多还能操作 t 次。
 *
 * 我们选择某一层扔一次鸡蛋。
 *
 * 这一次扔之后有两种情况：
 *
 * 1. 鸡蛋碎了
 *
 *    说明临界楼层在当前层下面。
 *
 *    此时：
 *    - 鸡蛋少 1 个
 *    - 操作次数少 1 次
 *
 *    下面最多能确定：
 *
 *        f[t - 1][egg - 1]
 *
 *    层。
 *
 * 2. 鸡蛋没碎
 *
 *    说明临界楼层在当前层或上面。
 *
 *    此时：
 *    - 鸡蛋数量不变
 *    - 操作次数少 1 次
 *
 *    上面最多能确定：
 *
 *        f[t - 1][egg]
 *
 *    层。
 *
 * 再加上当前测试的这一层本身：
 *
 *        + 1
 *
 * 所以：
 *
 *     f[t][egg] = f[t - 1][egg - 1] + 1 + f[t - 1][egg]
 *
 * --------------------------------------------------
 * 【为什么可以用一维数组】
 *
 * 计算第 t 轮时，只依赖第 t - 1 轮的状态。
 *
 * 所以可以用一维 dp[egg] 表示上一轮结果，
 * 然后原地更新。
 *
 * 但是更新时必须倒序遍历 egg：
 *
 *     for (int egg = k; egg >= 1; egg--)
 *
 * 因为：
 *
 *     dp[egg]     表示上一轮的 f[t - 1][egg]
 *     dp[egg - 1] 表示上一轮的 f[t - 1][egg - 1]
 *
 * 如果正序更新，dp[egg - 1] 会先被本轮更新掉，
 * 这样就会污染转移。
 *
 * --------------------------------------------------
 * 【答案怎么得到】
 *
 * 我们从 t = 0 开始，不断增加操作次数。
 *
 * 每增加一次操作次数，就更新一遍 dp。
 *
 * 当：
 *
 *     dp[k] >= n
 *
 * 说明：
 *
 *     用 k 个鸡蛋、t 次操作，
 *     已经可以确定至少 n 层楼。
 *
 * 此时这个 t 就是最小操作次数。
 *
 * --------------------------------------------------
 * 【复杂度】
 *
 * k <= 100
 * n <= 10000
 *
 * 每增加一次操作次数，更新 k 个状态。
 *
 * 时间复杂度：
 *
 *     O(ans * k)
 *
 * 空间复杂度：
 *
 *     O(k)
 *
 * --------------------------------------------------
 * 【一句话总结】
 *
 * 不直接问“n 层楼最少几次”，
 * 而是反过来问“t 次操作、k 个鸡蛋最多能确定多少层楼”。
 * 每次操作把楼层分成“碎了的下面部分”和“没碎的上面部分”，
 * 所以转移为：
 *
 *     dp[egg] = dp[egg] + dp[egg - 1] + 1
 */

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int k, n;
    cin >> k >> n;

    /*
        dp[egg] 表示：

        在当前操作次数 t 下，
        使用 egg 个鸡蛋，最多可以确定多少层楼。

        初始时 t = 0，
        没有操作机会，无论有多少鸡蛋，都无法确定任何楼层，
        所以 dp 全部初始化为 0。
    */
    vector<int> dp(k + 1, 0);

    /*
        t 表示当前允许的操作次数。

        我们不断增加 t，
        直到用 k 个鸡蛋、t 次操作可以覆盖 n 层楼。
    */
    int t = 0;

    /*
        当 dp[k] < n 时，
        说明当前操作次数 t 还不够，
        还需要增加一次操作机会。
    */
    while (dp[k] < n)
    {
        t++;

        /*
            倒序更新 egg。

            转移公式：
                dp[egg] = dp[egg] + dp[egg - 1] + 1

            其中：
            - 原来的 dp[egg] 表示上一轮 f[t - 1][egg]
              即鸡蛋没碎时，上面还能确定的楼层数。

            - 原来的 dp[egg - 1] 表示上一轮 f[t - 1][egg - 1]
              即鸡蛋碎了时，下面还能确定的楼层数。

            - +1 表示当前测试的这一层。
        */
        for (int egg = k; egg >= 1; egg--)
        {
            dp[egg] = dp[egg] + dp[egg - 1] + 1;
        }
    }

    /*
        循环结束时，dp[k] >= n。

        说明用 k 个鸡蛋、t 次操作，
        已经足以确定 n 层楼的临界楼层。

        并且 t 是从小到大第一次满足条件的，
        所以 t 就是最少操作次数。
    */
    cout << t << "\n";

    return 0;
}
```

## 目标和

当 DP 状态下标不好表示，尤其是会出现负数时，先别硬写，**尝试转化状态或转化问题**。

```cpp
class Solution {
public:
    int findTargetSumWays(vector<int>& nums, int target) {
        /*
            题目：494. 目标和

            给定一个非负整数数组 nums。
            对每个 nums[i]，可以选择在它前面添加 '+' 或 '-'。
            问最终表达式结果等于 target 的方法数。

            例如：
            nums = [1,1,1,1,1], target = 3

            一种表达式：
            +1 +1 +1 +1 -1 = 3

            要求统计这样的表达式一共有多少种。
        */

        int sum = 0;

        // 计算数组所有元素之和
        for (int x : nums) {
            sum += x;
        }

        /*
            先做范围判断。

            nums 中所有数加起来最大也只能得到 sum；
            全部取负最小只能得到 -sum。

            所以如果 target 超出 [-sum, sum]，
            那一定不可能构造出来。
        */
        if (abs(target) > sum) {
            return 0;
        }

        /*
            核心转化：

            假设被添加 '+' 的数之和为 P，
            被添加 '-' 的数之和为 N。

            那么有：

                P - N = target

            同时，所有数都被使用了，所以：

                P + N = sum

            两式相加：

                2P = sum + target

            所以：

                P = (sum + target) / 2

            也就是说，原问题可以转化为：

                从 nums 中选出一些数，
                使它们的和等于 (sum + target) / 2，
                问有多少种选法。
        */

        /*
            如果 sum + target 不是偶数，
            那么 P 就不是整数，不可能存在合法方案。
        */
        if ((sum + target) % 2 != 0) {
            return 0;
        }

        // 背包容量：需要凑出的加号集合之和
        int bag = (sum + target) / 2;

        /*
            dp[j] 表示：

            在已经处理过的数字中，
            选出若干个数，使它们的和恰好为 j 的方法数。

            注意：
            这题不是判断“能不能凑出 j”，
            而是统计“有多少种方法凑出 j”。

            所以 dp 数组存的是方案数，而不是 bool。
        */
        vector<int> dp(bag + 1, 0);

        /*
            初始状态：

            dp[0] = 1

            含义：
            什么数都不选，可以凑出和 0。
            这种选法算 1 种。

            这是计数型背包问题最重要的初始化。
            如果不写 dp[0] = 1，后面所有 dp[j] 都会一直是 0。
        */
        dp[0] = 1;

        /*
            遍历每一个数字 x。

            每个数字只能使用一次：
            - 要么选进加号集合 P
            - 要么不选进加号集合 P

            所以这是 01 背包。
        */
        for (int x : nums) {
            /*
                01 背包必须倒序遍历容量。

                为什么？

                因为每个 x 只能用一次。

                如果正序遍历：
                    dp[x] 可能刚刚由 dp[0] 更新出来，
                    然后 dp[2x] 又用刚更新的 dp[x] 更新，
                    这就相当于同一个 x 被用了多次。

                倒序遍历可以保证：
                    dp[j - x] 还是处理当前 x 之前的旧状态。
            */
            for (int j = bag; j >= x; j--) {
                /*
                    转移：

                    要凑出 j，有两类方法：

                    1. 不选当前数字 x
                       方法数仍然是原来的 dp[j]

                    2. 选当前数字 x
                       那么之前需要凑出 j - x
                       方法数是 dp[j - x]

                    所以总方法数为：

                        dp[j] = dp[j] + dp[j - x]
                */
                dp[j] += dp[j - x];
            }
        }

        /*
            dp[bag] 就表示：
            从 nums 中选出若干个数，使它们的和为 bag 的方法数。

            这也就是原题中添加 +/- 后得到 target 的方法数。
        */
        return dp[bag];
    }
};
```

## 分段 DP：从 Printing neatly 看动态规划建模

### 1. 这道题为什么不容易独立想出来

Printing neatly 这道题表面上是在“排版”，但真正的抽象是：

把一个单词序列切成若干行，每一行是一段连续的单词，并且每一行都有一个代价。

也就是说，它不是简单地处理某一个单词，而是在决定：

第几到第几个单词放在同一行。

这类题的难点不在公式计算，而在于先把现实问题抽象成：

“把序列分成若干连续段”。

只要能看出这一点，DP 状态和转移就会自然很多。

------

### 2. 破题关键：每一行是一段连续单词

输入是一串单词：

$$
1,2,\dots,n
$$

排版之后，每一行一定由若干个连续单词组成。

例如一种排版可能是：

$$
1\sim 3
$$

$$
4\sim 7
$$

$$
8\sim n
$$

所以整个问题其实是在问：

怎么把序列切成若干段，使总代价最小。

这就是典型的“分段 DP”。

------

### 3. 分段 DP 的通用模板

如果一个问题满足：

一个序列要被分成若干连续段；

每一段有一个代价；

总目标是让所有段的代价之和最小或最大；

那么常见状态是：

$$
dp[j]=\text{处理前 }j\text{ 个元素的最优值}
$$

为了求 $dp[j]$，枚举最后一段从哪里开始。

假设最后一段是：

$$
i,i+1,\dots,j
$$

那么前面部分是：

$$
1,2,\dots,i-1
$$

所以转移为：

$$
dp[j]=\min_{1\le i\le j}{dp[i-1]+cost(i,j)}
$$

其中：

$$
cost(i,j)
$$

表示把第 $i$ 到第 $j$ 个元素放成一段的代价。

这个模板要记住：

$$
dp[j]=\min_{1\le i\le j}{dp[i-1]+cost(i,j)}
$$

核心语言是：

枚举最后一段从哪里开始。

------

### 4. 套到 Printing neatly

这道题中：

$$
dp[j]=\text{整齐打印前 }j\text{ 个单词的最小总代价}
$$

为了求 $dp[j]$，枚举最后一行从第 $i$ 个单词开始。

也就是最后一行包含：

$$
i,i+1,\dots,j
$$

那么前 $i-1$ 个单词的最优排版代价是：

$$
dp[i-1]
$$

最后一行的代价是：

$$
cost(i,j)
$$

所以：

$$
dp[j]=\min_{1\le i\le j}{dp[i-1]+cost(i,j)}
$$

------

### 5. 如何计算一行的剩余空格

如果第 $i$ 到第 $j$ 个单词放在同一行，那么单词长度总和是：

$$
\sum_{k=i}^{j}l_k
$$

这些单词之间有：

$$
j-i
$$

个空格。

所以这一行实际占用字符数为：

$$
\sum_{k=i}^{j}l_k+(j-i)
$$

每行最多容纳 $M$ 个字符，因此剩余空格数是：

$$
extra(i,j)=M-\sum_{k=i}^{j}l_k-(j-i)
$$

如果：

$$
extra(i,j)<0
$$

说明第 $i$ 到第 $j$ 个单词放不进同一行。

------

### 6. 一行的代价

题目规定：除最后一行外，每一行的代价是剩余空格数的三次方。

所以：

$$
cost(i,j)=
\begin{cases}
\infty, & extra(i,j)<0,\
0, & j=n\text{ 且 }extra(i,j)\ge 0,\
extra(i,j)^3, & j<n\text{ 且 }extra(i,j)\ge 0.
\end{cases}
$$

注意最后一行不计代价，这是这题很容易漏掉的条件。

------

### 7. 完整递推式

初始化：

$$
dp[0]=0
$$

表示没有打印任何单词时，总代价为 0。

对于：

$$
j=1,2,\dots,n
$$

有：

$$
dp[j]=\min_{1\le i\le j}{dp[i-1]+cost(i,j)}
$$

其中 $cost(i,j)$ 按上一节定义。

为了恢复具体断行方案，可以记录：

$$
line[j]=\text{最优方案中，前 }j\text{ 个单词的最后一行起点}
$$

当某个 $i$ 使得：

$$
dp[i-1]+cost(i,j)
$$

更小时，就令：

$$
line[j]=i
$$

------

### 8. 恢复断行方案

如果已经算出了 $line[j]$，那么可以从 $j=n$ 开始恢复。

如果：

$$
line[j]=i
$$

说明前 $j$ 个单词的最优排版中，最后一行是：

$$
i,i+1,\dots,j
$$

然后继续恢复前：

$$
i-1
$$

个单词。

递归地看：

先恢复前 $i-1$ 个单词，再输出第 $i$ 到第 $j$ 个单词作为一行。

------

### 9. 这题和最近学过模型的联系

这道题和钢条切割有点像。

钢条切割是：

枚举最后一段或者第一段切多长。

Printing neatly 是：

枚举最后一行从哪里开始。

两者都属于：

“前面部分最优 + 最后一段单独计算代价”。

它也和矩阵链乘法、最优 BST 有联系，但不完全一样。

矩阵链乘法和最优 BST 是典型区间 DP：

$$
dp[i][j]
$$

表示区间 $[i,j]$ 的最优值，并且要枚举中间分割点或根。

而 Printing neatly 虽然也出现了 $cost(i,j)$，但核心状态仍然是一维：

$$
dp[j]
$$

因为它处理的是前缀最优。

所以它更准确地说是：

分段 DP / 前缀 DP。

------

### 10. 如何识别这类题

以后如果看到题目描述中有这些特征：

一个序列；

要把序列分成若干连续部分；

每个部分有一个代价；

总代价要求最小或最大；

那么可以优先想到分段 DP。

常见思考路径是：

第一步：问“最后一段是什么？”

第二步：假设最后一段是 $i\sim j$。

第三步：前面 $1\sim i-1$ 是否可以用一个最优子问题表示？

第四步：写成：

$$
dp[j]=\min_{i}{dp[i-1]+cost(i,j)}
$$

这就是这类题的标准破题方式。

------

### 11. 复杂度

如果直接预处理所有：

$$
cost(i,j)
$$

需要：

$$
O(n^2)
$$

因为一共有约：

$$
\frac{n(n+1)}{2}
$$

个可能的行区间。

计算 DP 时，外层枚举 $j$，内层枚举最后一行起点 $i$，也是：

$$
O(n^2)
$$

所以总时间复杂度是：

$$
O(n^2)
$$

如果保存 $extra$ 和 $cost$ 两张二维表，空间复杂度是：

$$
O(n^2)
$$

如果边算边转移，也可以把空间进一步优化，但算法设计题写 $O(n^2)$ 空间通常已经足够清楚。

------

### 12. 一句话总结

Printing neatly 的本质不是排版细节，而是分段 DP。

关键是看出：

每一行是一段连续单词，整个排版就是把单词序列切成若干连续段。

因此定义：

$$
dp[j]=\text{打印前 }j\text{ 个单词的最小代价}
$$

然后枚举最后一行从哪里开始：

$$
dp[j]=\min_{1\le i\le j}{dp[i-1]+cost(i,j)}
$$

这道题暂时不能独立想出来很正常。真正要吸收的是这个模板：

看到“序列分成若干连续段，每段有代价”，就尝试用“枚举最后一段”的分段 DP。

## 编辑距离
```cpp
class Solution {
public:
    int minDistance(string word1, string word2) {
        /*
            题目：72. 编辑距离

            给定两个字符串 word1 和 word2，
            每次可以对 word1 做三种操作之一：

            1. 插入一个字符
            2. 删除一个字符
            3. 替换一个字符

            问：最少需要多少次操作，可以把 word1 变成 word2。

            --------------------------------------------------
            【核心思想】

            这题是典型的字符串二维 DP。

            和 LCS 很像，它们都喜欢用：

                “前 i 个字符”
                “前 j 个字符”

            来定义状态。

            注意：
            dp[i][j] 里的 i、j 不是字符串下标，
            而是表示“前 i 个字符”和“前 j 个字符”。

            所以：
                word1 的前 i 个字符是 word1[0 ... i - 1]
                word2 的前 j 个字符是 word2[0 ... j - 1]
        */

        int m = word1.size();
        int n = word2.size();

        /*
            dp[i][j] 表示：

                把 word1 的前 i 个字符，
                变成 word2 的前 j 个字符，
                所需要的最少操作次数。

            最终答案就是：

                dp[m][n]

            也就是：
                把整个 word1 变成整个 word2 的最少操作次数。
        */
        vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));

        /*
            初始化第一行：

            dp[0][j] 表示：
                把空字符串 "" 变成 word2 的前 j 个字符。

            只能不断插入字符。

            例如：
                "" -> "abc"

            需要插入 3 次。

            所以：
                dp[0][j] = j
        */
        for (int j = 0; j <= n; j++) {
            dp[0][j] = j;
        }

        /*
            初始化第一列：

            dp[i][0] 表示：
                把 word1 的前 i 个字符变成空字符串 ""。

            只能不断删除字符。

            例如：
                "abc" -> ""

            需要删除 3 次。

            所以：
                dp[i][0] = i
        */
        for (int i = 0; i <= m; i++) {
            dp[i][0] = i;
        }

        /*
            开始填表。

            因为 dp[0][j] 和 dp[i][0] 已经初始化好了，
            所以 i 和 j 都从 1 开始。
        */
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                /*
                    当前比较的是：

                        word1[i - 1]
                        word2[j - 1]

                    因为 dp[i][j] 表示“前 i 个”和“前 j 个”，
                    所以它们的最后一个字符下标分别是 i - 1 和 j - 1。
                */

                if (word1[i - 1] == word2[j - 1]) {
                    /*
                        如果最后一个字符相等，
                        那么这个字符不需要额外操作。

                        问题就退化为：

                            把 word1 的前 i - 1 个字符
                            变成 word2 的前 j - 1 个字符

                        所以：
                            dp[i][j] = dp[i - 1][j - 1]
                    */
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    /*
                        如果最后一个字符不相等，
                        就有三种操作可以选择。

                        --------------------------------------------------
                        1. 删除 word1[i - 1]

                        当前是：
                            word1 前 i 个字符 -> word2 前 j 个字符

                        如果删除 word1 的最后一个字符，
                        那么问题变成：

                            word1 前 i - 1 个字符 -> word2 前 j 个字符

                        代价：
                            dp[i - 1][j] + 1

                        --------------------------------------------------
                        2. 插入 word2[j - 1]

                        可以理解为：
                        为了让 word1 匹配 word2 的最后一个字符，
                        我往 word1 后面插入 word2[j - 1]。

                        插入之后，word2 的最后一个字符已经匹配好了，
                        剩下问题是：

                            word1 前 i 个字符 -> word2 前 j - 1 个字符

                        代价：
                            dp[i][j - 1] + 1

                        --------------------------------------------------
                        3. 替换 word1[i - 1] 为 word2[j - 1]

                        替换后，最后一个字符匹配好了，
                        剩下问题是：

                            word1 前 i - 1 个字符 -> word2 前 j - 1 个字符

                        代价：
                            dp[i - 1][j - 1] + 1

                        --------------------------------------------------
                        三种操作取最小值。
                    */
                    dp[i][j] = 1 + min(
                        min(dp[i - 1][j],     // 删除
                            dp[i][j - 1]),    // 插入
                        dp[i - 1][j - 1]      // 替换
                    );
                }
            }
        }

        /*
            dp[m][n] 表示：
                把整个 word1 变成整个 word2 的最少操作次数。
        */
        return dp[m][n];
    }
};
```

## 手环

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

const ll MOD = 1000000007;

/*
    题意转化：

    每颗珠子只有两种属性：

        木：0
        金：1

    相邻两颗珠子之间的灵径为金色，当且仅当：
        两颗珠子中至少有一颗是金属性。

    也就是说，非法情况只有一种：

        木 木
        0  0

    所以题目等价于：

        求长度为 n 的环形 0/1 序列数量，
        要求任意相邻两个位置不能同时为 0。

    注意：
        因为是手环，所以第 n 颗和第 1 颗也相邻。
        题目说“不考虑旋转同构”，说明旋转后相同的方案仍然算不同方案。
        因此可以把它看成有编号的位置 1, 2, ..., n。
*/

/*
    你这题一开始容易漏掉的点：

    不能只定义：

        dp[i] 表示长度为 i 的合法方案数

    因为当你要往后加一颗珠子时，
    能不能加“木”，取决于前一颗珠子是什么。

    所以普通线性 DP 需要区分结尾：

        dp[i][0]：长度为 i，最后一颗是木的方案数
        dp[i][1]：长度为 i，最后一颗是金的方案数

    转移：

        如果第 i 颗是木：
            前一颗必须是金
            dp[i][0] = dp[i - 1][1]

        如果第 i 颗是金：
            前一颗是木或金都可以
            dp[i][1] = dp[i - 1][0] + dp[i - 1][1]

    写成矩阵就是：

        [ dp[i][0] ]   [ 0 1 ] [ dp[i - 1][0] ]
        [ dp[i][1] ] = [ 1 1 ] [ dp[i - 1][1] ]

    转移矩阵 A：

        A = [ 0 1
              1 1 ]

    其中：
        A[x][y] = 1 表示属性 x 后面可以接属性 y。

    也就是：

        木 -> 木 不合法，所以是 0
        木 -> 金 合法
        金 -> 木 合法
        金 -> 金 合法
*/

/*
    为什么环形答案是 trace(A^n)？

    环形序列有 n 条相邻边：

        第1颗 -> 第2颗
        第2颗 -> 第3颗
        ...
        第n颗 -> 第1颗

    如果从某种属性出发，经过 n 次合法转移后又回到自己，
    就形成了一个合法的环。

    A^n[x][x] 表示：
        从属性 x 出发，经过 n 次合法转移，最后回到属性 x 的方案数。

    所以总方案数为：

        A^n[0][0] + A^n[1][1]

    也就是矩阵 A^n 的迹 trace。
*/

/*
    矩阵乘法：计算 2x2 矩阵 a * b
*/
vector<vector<ll>> mul(const vector<vector<ll>>& a, const vector<vector<ll>>& b)
{
    vector<vector<ll>> res(2, vector<ll>(2, 0));

    res[0][0] = (a[0][0] * b[0][0] + a[0][1] * b[1][0]) % MOD;
    res[0][1] = (a[0][0] * b[0][1] + a[0][1] * b[1][1]) % MOD;
    res[1][0] = (a[1][0] * b[0][0] + a[1][1] * b[1][0]) % MOD;
    res[1][1] = (a[1][0] * b[0][1] + a[1][1] * b[1][1]) % MOD;

    return res;
}

/*
    矩阵快速幂：计算 a^n

    快速幂模板：

        如果当前 n 是奇数：
            res = res * a

        然后：
            a = a * a
            n >>= 1

    注意：
        平方的是底数 a，不是 res。
*/
vector<vector<ll>> quick_pow(vector<vector<ll>> a, long long n)
{
    /*
        单位矩阵，相当于矩阵乘法中的 1。

        任意矩阵 A：
            I * A = A
            A * I = A
    */
    vector<vector<ll>> res(2, vector<ll>(2, 0));
    res[0][0] = 1;
    res[1][1] = 1;

    while (n > 0)
    {
        if (n & 1)
        {
            res = mul(res, a);
        }

        a = mul(a, a);
        n >>= 1;
    }

    return res;
}

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    /*
        转移矩阵：

            木 = 0
            金 = 1

        A[0][0] = 0：木后面不能接木
        A[0][1] = 1：木后面可以接金
        A[1][0] = 1：金后面可以接木
        A[1][1] = 1：金后面可以接金
    */
    vector<vector<ll>> A(2, vector<ll>(2, 0));
    A[0][0] = 0;
    A[0][1] = 1;
    A[1][0] = 1;
    A[1][1] = 1;

    int T;
    cin >> T;

    while (T--)
    {
        long long n;
        cin >> n;

        /*
            求 A^n。
            因为环上有 n 条边，所以是 n 次转移。
        */
        vector<vector<ll>> ans = quick_pow(A, n);

        /*
            环形合法方案数：

                A^n[0][0] + A^n[1][1]

            即从木出发回到木的方案数
            加上从金出发回到金的方案数。
        */
        cout << (ans[0][0] + ans[1][1]) % MOD << "\n";
    }

    return 0;
}
```

## 最长回文子序列

```cpp
class Solution {
public:
    int longestPalindromeSubseq(string s) {
        /*
            题目：516. 最长回文子序列

            给定一个字符串 s，返回其中最长回文子序列的长度。

            注意：
                子序列：可以不连续，但相对顺序不能改变。
                子串：必须连续。

            例如：
                s = "bbbab"

            一个最长回文子序列是：
                "bbbb"

            长度为 4。

            --------------------------------------------------
            这题有两种常见做法：

            方法一：
                把 s 反转成 rev，
                求 s 和 rev 的最长公共子序列 LCS。

            方法二：
                原生区间 DP。

            这里写的是方法二：区间 DP。
        */

        int n = s.size();

        /*
            dp[i][j] 表示：

                字符串 s 在区间 [i, j] 内，
                最长回文子序列的长度。

            也就是：

                s[i...j] 这一段里的最长回文子序列长度。

            注意：
                这里 i 和 j 是真实下标，不是“前 i 个字符”。

            这和 LCS / 编辑距离那种 dp[i][j] 表示前缀不同。
        */
        vector<vector<int>> dp(n, vector<int>(n, 0));

        /*
            初始化：

            当区间长度为 1 时，也就是只有一个字符：

                s[i...i]

            单个字符本身就是回文子序列。

            所以：
                dp[i][i] = 1
        */
        for (int i = 0; i < n; i++) {
            dp[i][i] = 1;
        }

        /*
            区间 DP 的关键是循环顺序。

            dp[i][j] 会依赖：

                dp[i + 1][j - 1]
                dp[i + 1][j]
                dp[i][j - 1]

            也就是说：
                它依赖更短的区间。

            所以我们必须先算小区间，再算大区间。

            这里采用一种常见写法：

                i 从后往前枚举
                j 从 i + 1 往后枚举

            为什么 i 要倒着来？

            因为 dp[i][j] 需要用到 dp[i + 1][j]，
            所以 i + 1 这一行必须已经算过。

            当 i 从 n - 1 倒着到 0 时，
            dp[i + 1][j] 就已经在之前算好了。
        */
        for (int i = n - 1; i >= 0; i--) {
            for (int j = i + 1; j < n; j++) {
                /*
                    当前考虑区间：

                        s[i...j]

                    关键看两端字符：

                        s[i]
                        s[j]
                */

                if (s[i] == s[j]) {
                    /*
                        情况一：两端字符相同

                        如果 s[i] == s[j]，
                        那么这两个字符可以一起放进回文子序列的两端。

                        中间部分就是：

                            s[i + 1 ... j - 1]

                        所以：

                            dp[i][j] = dp[i + 1][j - 1] + 2

                        直观理解：

                            s[i] + 中间最长回文子序列 + s[j]

                        例如：

                            s = "bbbab"

                        如果某个区间两端都是 'b'，
                        那么这两个 'b' 可以作为回文的左右两端。
                    */
                    dp[i][j] = dp[i + 1][j - 1] + 2;
                } else {
                    /*
                        情况二：两端字符不同

                        如果 s[i] != s[j]，
                        那么 s[i] 和 s[j] 不可能同时作为同一个回文子序列的两端。

                        所以只能考虑：

                        1. 不要左端 s[i]
                           看 s[i + 1 ... j]

                           dp[i + 1][j]

                        2. 不要右端 s[j]
                           看 s[i ... j - 1]

                           dp[i][j - 1]

                        取两者最大值。
                    */
                    dp[i][j] = max(dp[i + 1][j], dp[i][j - 1]);
                }
            }
        }

        /*
            最终答案：

                整个字符串 s[0...n-1] 的最长回文子序列长度。

            所以返回：

                dp[0][n - 1]
        */
        return dp[0][n - 1];
    }
};
```
