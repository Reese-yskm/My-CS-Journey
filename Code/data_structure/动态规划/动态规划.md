# 动态规划

# DP 入门先抓这五件事

以后看到一道 DP 题，你不要急着写转移，先按这五步问自己：

## 1. 状态表示什么？

这是最核心的一步。

比如：

```cpp
dp[i]
```

到底表示：

- 前 `i` 个元素的答案？
- 以第 `i` 个元素结尾的答案？
- 到达第 `i` 个位置的方案数？
- 容量为 `i` 时的最优值？

DP 很多题错，不是转移错，而是 `dp[i]` 的含义一开始就没定义清楚。

------

## 2. 最后一步是什么？

DP 常常从“最后一步”倒推。

比如爬楼梯：

要到第 `i` 阶，最后一步可能来自：

- 第 `i - 1` 阶
- 第 `i - 2` 阶

所以：

```cpp
dp[i] = dp[i - 1] + dp[i - 2];
```

这个不是背公式，而是由“最后一步”推出来的。

------

## 3. 状态转移从哪里来？

你要能说清楚：

```cpp
dp[i] = ...
```

为什么这样转移。

比如最大子数组和：

```cpp
dp[i] = max(nums[i], dp[i - 1] + nums[i]);
```

含义是：

以 `i` 结尾的最大子数组，要么：

- 自己重新开始
- 接在前一个最大子数组后面

------

## 4. 初始条件是什么？

比如：

```cpp
dp[0] = ?
dp[1] = ?
```

初始条件不是随便写的，必须和状态定义一致。

如果 `dp[i]` 表示“到达第 i 阶的方法数”，那 `dp[0]` 通常表示“站在起点的方法数”，是 `1`。

------

## 5. 遍历顺序是什么？

有些题从前往后。

有些二维 DP 要先行后列。

背包题还要区分：

- 01 背包：倒序遍历容量
- 完全背包：正序遍历容量

------

# 经典题目及其分析

## 1.爬楼梯

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。

每次你可以爬 `1` 或 `2` 个台阶。你有多少种不同的方法可以爬到楼顶呢？

```cpp
class Solution
{
public:
    int climbStairs(int n)
    {
        /*
            dp[i] 表示：爬到第 i 阶楼梯的方法数

            例如：
            dp[1] = 1
            只能走：1

            dp[2] = 2
            可以走：
            1 + 1
            2
        */

        int dp[46];

        /*
            初始状态

            dp[0] = 1 的含义：
            “站在第 0 阶”本身算作一种起点状态。
            这样后面转移时会比较统一。

            dp[1] = 1：
            爬到第 1 阶只有一种方法：走 1 阶。
        */
        dp[0] = 1;
        dp[1] = 1;

        /*
            状态转移

            想到达第 i 阶，最后一步只有两种可能：

            1. 从第 i - 1 阶走 1 阶上来
               方法数是 dp[i - 1]

            2. 从第 i - 2 阶走 2 阶上来
               方法数是 dp[i - 2]

            所以：
            dp[i] = dp[i - 1] + dp[i - 2]
        */
        for (int i = 2; i <= n; i++)
        {
            dp[i] = dp[i - 1] + dp[i - 2];
        }

        // 返回爬到第 n 阶楼梯的方法数
        return dp[n];
    }
};
```

## 2.最长公共子序列

```cpp
#include <bits/stdc++.h>
using namespace std;

/*
    最长公共子序列 LCS 经典做法

    给定两个字符串 x 和 y，求它们的最长公共子序列长度，
    并恢复出其中一个最长公共子序列。

    **状态定义**：
        c[i][j] 表示 x 的前 i 个字符和 y 的前 j 个字符的 LCS 长度。

    注意：
        x 的第 i 个字符是 x[i - 1]
        y 的第 j 个字符是 y[j - 1]

    转移：
        1. 如果 x[i - 1] == y[j - 1]
           那么这个字符可以作为 LCS 的最后一个字符：
           c[i][j] = c[i - 1][j - 1] + 1

        2. 如果 x[i - 1] != y[j - 1]
           那么不能同时选这两个字符，只能尝试丢掉其中一个：
           c[i][j] = max(c[i - 1][j], c[i][j - 1])

    为了恢复方案，使用 b[i][j] 记录转移方向：
        1 表示来自左上角，即 x[i - 1] == y[j - 1]，当前字符被选入 LCS
        2 表示来自上方，即 c[i][j] = c[i - 1][j]
        3 表示来自左方，即 c[i][j] = c[i][j - 1]
*/

string x, y;

// c[i][j] 表示 x[0..i-1] 和 y[0..j-1] 的 LCS 长度
vector<vector<int>> c;

// b[i][j] 用于恢复路径
// 1: 左上方 ↖
// 2: 上方 ↑
// 3: 左方 ←
vector<vector<int>> b;

void lcs_length()
{
    int m = x.size();
    int n = y.size();

    // 多开一行一列，用来处理空串边界
    c.assign(m + 1, vector<int>(n + 1, 0));
    b.assign(m + 1, vector<int>(n + 1, 0));

    // c[i][0] = 0，表示 x 的前 i 个字符和空串的 LCS 长度为 0
    for (int i = 0; i <= m; i++)
    {
        c[i][0] = 0;
    }

    // c[0][j] = 0，表示空串和 y 的前 j 个字符的 LCS 长度为 0
    for (int j = 0; j <= n; j++)
    {
        c[0][j] = 0;
    }

    // 从短前缀开始，逐步填表
    for (int i = 1; i <= m; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            // 情况 1：两个前缀的最后一个字符相同
            // 那么这个字符可以加入 LCS
            if (x[i - 1] == y[j - 1])
            {
                c[i][j] = c[i - 1][j - 1] + 1;
                b[i][j] = 1; // 来自左上方，说明 x[i - 1] 被选入 LCS
            }
            else
            {
                // 情况 2：最后一个字符不同
                // 比较“丢掉 x[i - 1]”和“丢掉 y[j - 1]”哪个更优
                if (c[i - 1][j] >= c[i][j - 1])
                {
                    c[i][j] = c[i - 1][j];
                    b[i][j] = 2; // 来自上方，表示丢掉 x[i - 1]
                }
                else
                {
                    c[i][j] = c[i][j - 1];
                    b[i][j] = 3; // 来自左方，表示丢掉 y[j - 1]
                }
            }
        }
    }
}

// 根据 b 表递归恢复一个 LCS
void print_lcs(int i, int j)
{
    // 如果某个字符串已经为空，就没有公共子序列了
    if (i == 0 || j == 0)
    {
        return;
    }

    // 来自左上方，说明 x[i - 1] == y[j - 1]
    // 当前字符属于 LCS
    if (b[i][j] == 1)
    {
        print_lcs(i - 1, j - 1);

        // 注意：递归回来后再打印
        // 因为我们是从右下角往左上角追踪的，直接打印会倒序
        cout << x[i - 1];
    }
    // 来自上方，继续往上追踪
    else if (b[i][j] == 2)
    {
        print_lcs(i - 1, j);
    }
    // 来自左方，继续往左追踪
    else
    {
        print_lcs(i, j - 1);
    }
}

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> x >> y;

    lcs_length();

    int m = x.size();
    int n = y.size();

    cout << "LCS length: " << c[m][n] << "\n";

    cout << "One LCS: ";
    print_lcs(m, n);
    cout << "\n";

    return 0;
}
```

**如果两个字符串长度相差很大，这时采用如下dp方法**

```cpp
#include <bits/stdc++.h>
using namespace std;

/*
    最长公共子序列 LCS 优化版

    题目特点：
        |s| <= 1e6
        |t| <= 1e3

    普通 LCS 做法：
        dp[i][j] 表示 s 的前 i 个字符和 t 的前 j 个字符的 LCS 长度
        时间复杂度 O(|s| * |t|)
        最坏为 1e9，容易超时。

    本题优化思路：
        因为 t 很短，所以我们主要围绕短串 t 来做 DP。

    核心思想：
        对于相同长度的公共子序列，我们希望它在长串 s 中的结尾位置尽可能靠前。

        为什么？
        因为结尾越靠前，后面剩余可匹配的位置越多，
        这个方案越有机会继续扩展成更长的公共子序列。

    状态定义：
        dp[len] 表示：
            当前已经处理过 t 的某个前缀后，
            若能匹配出长度为 len 的公共子序列，
            那么这个公共子序列在 s 中“最早结束”的位置是多少。

        如果 dp[len] = INF，
        表示当前还无法匹配出长度为 len 的公共子序列。

    例如：
        dp[3] = 10 表示：
            目前能匹配出长度为 3 的公共子序列，
            并且在所有长度为 3 的方案中，
            最优方案最早可以结束在 s 的第 10 个位置。

    初始化：
        dp[0] = 0
        表示长度为 0 的公共子序列不需要匹配任何字符，
        可以认为它结束在 s 的第 0 个位置。

    转移：
        依次处理短串 t 中的每个字符 ch。

        假设当前已经能匹配出长度 len，
        并且这个序列最早结束在 s 的位置 dp[len]。

        那么如果想把 ch 接到后面，
        就要在 s 中找到：
            位置 > dp[len] 的第一个 ch。

        如果找到了这个位置 nxt，
        就可以更新：
            dp[len + 1] = min(dp[len + 1], nxt) 
        保证始终是s中靠前，所以用min

    如何快速找 nxt？
        预处理长串 s 中每个字符出现的位置数组 pos[c]。
        然后用 upper_bound 找到第一个大于 dp[len] 的位置。

    为什么 len 要倒序枚举？
        因为 t 中每个字符只能使用一次。

        如果正序枚举 len，
        当前这个 t 字符可能先更新 dp[len + 1]，
        然后又被用来更新 dp[len + 2]，
        这就相当于同一个字符被重复使用了。

        所以和 01 背包一样，要倒序枚举。
*/

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s, t;
    cin >> s >> t;

    int n = s.size();
    int m = t.size();

    /*
        pos[c] 存储字符 c 在长串 s 中出现的所有位置。

        这里使用 1-based 位置：
            s[0] 的位置记为 1
            s[1] 的位置记为 2
            ...

        这样 dp[0] = 0 就很自然：
            长度为 0 的公共子序列结束在位置 0，
            后续要找的位置必须 > 0。
    */
    vector<int> pos[26];

    for (int i = 0; i < n; i++)
    {
        int c = s[i] - 'a';
        pos[c].push_back(i + 1);
    }

    /*
        INF 表示无法匹配。

        因为 s 的最大合法位置是 n，
        所以 n + 1 可以作为无穷大。
    */
    const int INF = n + 1;

    /*
        dp[len] 表示：
            匹配出长度为 len 的公共子序列时，
            它在 s 中最早的结束位置。

        len 的最大值不会超过 m，因为公共子序列长度不可能超过短串 t 的长度。
    */
    vector<int> dp(m + 1, INF);

    // 长度为 0 的公共子序列，认为结束在 s 的第 0 个位置
    dp[0] = 0;

    /*
        ans 表示当前已经能够匹配出的最大公共子序列长度。

        它不是必须的，但可以让内层循环只枚举到 ans，
        不必每次都枚举到 m。
    */
    int ans = 0;

    /*
        依次处理短串 t 中的每个字符。

        当前处理到的字符 ch，可以选择：
            1. 不用它；
            2. 把它接到某个已有公共子序列后面，使长度增加 1。
    */
    for (char ch : t)
    {
        int c = ch - 'a';

        /*
            倒序枚举当前已经能匹配出的长度 len。

            如果 dp[len] 有效，说明我们已经能匹配出长度 len，
            并且最早结束位置是 dp[len]。

            现在尝试把当前字符 ch 接到它后面。
        */
        for (int len = ans; len >= 0; len--)
        {
            // 如果当前长度 len 还无法匹配出来，跳过
            if (dp[len] == INF)
            {
                continue;
            }

            /*
                在 s 中找到第一个位置 nxt，满足：
                    nxt > dp[len]
                    s[nxt] == ch

                由于 pos[c] 是有序数组，可以用 upper_bound。
            */
            auto it = upper_bound(pos[c].begin(), pos[c].end(), dp[len]);

            // 如果找到了这样的字符位置
            if (it != pos[c].end())
            {
                int nxt = *it;

                /*
                    用当前字符 ch 把长度 len 的公共子序列扩展成长度 len + 1。

                    dp[len + 1] 要取最小位置，
                    因为同样长度下，结尾越靠前越好。
                */
                if (nxt < dp[len + 1])
                {
                    dp[len + 1] = nxt;

                    // 更新当前能达到的最大长度
                    ans = max(ans, len + 1);
                }
            }
        }
    }

    /*
        ans 就是最长公共子序列长度。

        因为 dp[len] 有效表示存在长度为 len 的公共子序列，
        我们一直维护最大有效 len。
    */
    cout << ans << "\n";

    return 0;
}
```



## 3.01背包

```cpp
#include <bits/stdc++.h>
using namespace std;

/*
    经典 01 背包问题

    有 n 件物品，一个容量为 V 的背包。
    第 i 件物品的体积为 v[i]，价值为 w[i]。
    每件物品最多只能选一次。

    目标：
        在总体积不超过 V 的前提下，使总价值最大。

    状态定义：
        dp[j] 表示背包容量为 j 时，能够获得的最大价值。

    转移：
        对于第 i 件物品，有两种选择：

        1. 不选第 i 件物品：
           dp[j] 不变。

        2. 选第 i 件物品：
           前提是 j >= v[i]。
           那么剩余容量是 j - v[i]，
           新价值是 dp[j - v[i]] + w[i]。

        所以：
           dp[j] = max(dp[j], dp[j - v[i]] + w[i])

    为什么容量 j 要倒序枚举？

        因为每件物品只能选一次。

        如果正序枚举：
            dp[j - v[i]] 可能已经在本轮被第 i 件物品更新过，
            那么再用它更新 dp[j]，就等于第 i 件物品被用了多次。

        倒序枚举可以保证：
            dp[j - v[i]] 还是上一轮的结果，
            也就是还没有使用第 i 件物品时的结果。

        所以 01 背包必须倒序枚举容量。
*/

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, V;
    cin >> n >> V;

    vector<int> v(n + 1); // v[i] 表示第 i 件物品的体积
    vector<int> w(n + 1); // w[i] 表示第 i 件物品的价值

    for (int i = 1; i <= n; i++)
    {
        cin >> v[i] >> w[i];
    }

    // dp[j] 表示容量为 j 时的最大价值
    vector<int> dp(V + 1, 0);

    // 外层枚举每一件物品，它只有选或不选两种可能
    for (int i = 1; i <= n; i++)
    {
        // 01 背包容量必须倒序枚举
        // 从 V 枚举到 v[i]，保证第 i 件物品最多只被选一次
        for (int j = V; j >= v[i]; j--)
        {
            // 不选第 i 件物品：dp[j]
            // 选第 i 件物品：dp[j - v[i]] + w[i]
            dp[j] = max(dp[j], dp[j - v[i]] + w[i]);
        }
    }

    // dp[V] 就是容量不超过 V 时能获得的最大价值
    cout << dp[V] << "\n";

    return 0;
}
```

另一个版本，dp总价值

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

/*
    01 背包优化版：按价值做 DP

    题目特点：
        n <= 500
        V <= 1e9
        v[i] <= 1e9
        sum(w[i]) <= 1e6

    经典 01 背包通常定义：
        dp[容量] = 最大价值

    但这题容量 V 最大为 1e9，不能开 dp[V + 1]。

    所以反过来定义：
        dp[价值] = 达到该价值所需的最小体积

    最后找最大的价值 value，使得：
        dp[value] <= V

    这就是“容量太大，价值较小”时常用的背包优化思路。
*/

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    ll V;

    cin >> n >> V;

    // v[i] 表示第 i 件物品的体积
    // 体积可能很大，并且会累加，所以用 long long
    vector<ll> v(n + 1);

    // w[i] 表示第 i 件物品的价值
    // 题目保证总价值 <= 1e6，所以 int 足够
    vector<int> w(n + 1);

    // sumw 表示所有物品价值之和
    // 后面 dp 数组的大小就是 sumw + 1
    int sumw = 0;

    for (int i = 1; i <= n; i++)
    {
        cin >> v[i] >> w[i];
        sumw += w[i];
    }

    // INF 表示一个很大的体积
    // 因为最大总体积可能达到 500 * 1e9 = 5e11
    // 所以 4e18 足够大
    const ll INF = 4e18;

    /*
        dp[j] 表示：
            凑出总价值 j 所需要的最小总体积。

        如果 dp[j] = INF，表示当前还无法凑出价值 j。
    */
    vector<ll> dp(sumw + 1, INF);

    // 凑出价值 0 不需要选任何物品，体积为 0
    dp[0] = 0;

    /*
        枚举每一件物品。

        因为每件物品只能选一次，所以这是 01 背包。

        注意：
            这里倒序枚举的是“价值 j”，不是容量。
            倒序的目的仍然是防止同一件物品被重复使用。
    */
    for (int i = 1; i <= n; i++)
    {
        /*
            j 表示当前想凑出的总价值。

            如果选第 i 件物品，它贡献价值 w[i]，
            那么之前需要凑出价值 j - w[i]。

            所以转移为：
                dp[j] = min(dp[j], dp[j - w[i]] + v[i])
        */
        for (int j = sumw; j >= w[i]; j--)
        {
            dp[j] = min(dp[j], dp[j - w[i]] + v[i]);
        }
    }

    /*
        现在 dp[j] 表示达到价值 j 的最小体积。

        题目要求总体积不超过 V，且价值最大。

        所以从大到小枚举价值 j，
        找到第一个满足 dp[j] <= V 的 j。
    */
    int ans = 0;

    for (int j = sumw; j >= 0; j--)
    {
        if (dp[j] <= V)
        {
            ans = j;
            break;
        }
    }

    cout << ans << "\n";

    return 0;
}
```

**总结：**这两个版本的dp对象不同，具体应用看数据，那个小dp哪个

## 4.最长递增子序列

给你一个整数数组 `nums` ，找到其中最长严格递增子序列的长度。

**子序列** 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，`[3,6,2,7]` 是数组 `[0,3,1,6,2,2,7]` 的子序列。

```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        int n = nums.size();

        /*
            dp[i] 表示：
                以 nums[i] 结尾的最长递增子序列长度。

            注意关键词是“以 nums[i] 结尾”。

            例如：
                nums = [10, 9, 2, 5, 3, 7]

            如果 i 指向 7，那么 dp[i] 表示：
                必须以 7 作为最后一个元素时，
                能得到的最长递增子序列长度。

            可能是：
                [2, 5, 7]
            或者：
                [2, 3, 7]
            长度都是 3。
        */
        vector<int> dp(n, 1);

        /*
            pre[i] 表示：
                在以 nums[i] 结尾的某个最优 LIS 中，
                nums[i] 前一个元素的下标。

            如果 pre[i] = -1，
            表示 nums[i] 前面没有元素，
            即 nums[i] 自己就是这个递增子序列的开头。
        */
        vector<int> pre(n, -1);

        /*
            ans 记录目前找到的 LIS 最大长度。

            pos 记录这个最长递增子序列的最后一个元素下标。
            后面恢复路径时，就从 pos 开始沿着 pre 往前找。
        */
        int ans = 1;
        int pos = 0;

        /*
            枚举每一个位置 i。

            我们要计算：
                如果 LIS 必须以 nums[i] 结尾，
                那么它的前一个元素可以是谁？
        */
        for (int i = 0; i < n; i++) {

            /*
                枚举 i 前面的所有位置 j。

                如果 nums[j] < nums[i]，
                说明 nums[i] 可以接在 nums[j] 后面，
                从而形成一个更长的递增子序列。
            */
            for (int j = 0; j < i; j++) {

                /*
                    只有 nums[j] < nums[i] 时，
                    nums[i] 才能接到以 nums[j] 结尾的 LIS 后面。
                */
                if (nums[j] < nums[i]) {

                    /*
                        如果接在 nums[j] 后面能得到更长的序列，
                        就更新 dp[i]。

                        原来：
                            dp[i] 表示当前已知的、以 nums[i] 结尾的最长长度。

                        新方案：
                            以 nums[j] 结尾的 LIS 长度为 dp[j]，
                            接上 nums[i] 后长度变为 dp[j] + 1。
                    */
                    if (dp[j] + 1 > dp[i]) {
                        dp[i] = dp[j] + 1;

                        /*
                            记录前驱。

                            这句话表示：
                                当前最优的、以 nums[i] 结尾的 LIS，
                                是从 nums[j] 接过来的。
                        */
                        pre[i] = j;
                    }
                }
            }

            /*
                LIS 不一定以最后一个元素结尾，
                所以每算完一个 dp[i]，都要更新全局答案。
            */
            if (dp[i] > ans) {
                ans = dp[i];
                pos = i;
            }
        }

        /*
            恢复一个 LIS。

            从最长序列的最后一个位置 pos 出发，
            不断沿着 pre[pos] 往前跳。

            由于是从后往前找，所以恢复出来的 res 是反的。
        */
        vector<int> res;

        while (pos != -1) {
            res.push_back(nums[pos]);
            pos = pre[pos];
        }

        /*
            反转后，res 就是从前到后的一个 LIS。
        */
        reverse(res.begin(), res.end());

        /*
            LeetCode 300 只要求返回 LIS 长度，
            所以这里返回 ans。

            如果你在算法导论练习里需要输出具体序列，
            res 里面就是恢复出来的一个最长递增子序列。
        */
        return ans;
    }
};
```

**优化版**

```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        /*
            tail[len - 1] 表示：

                长度为 len 的递增子序列中，所有可能方案的“最小结尾值”。

            例如：
                tail[0] 表示长度为 1 的递增子序列的最小结尾值；
                tail[1] 表示长度为 2 的递增子序列的最小结尾值；
                tail[2] 表示长度为 3 的递增子序列的最小结尾值。

            为什么要保留“最小结尾值”？

                对于相同长度的递增子序列，结尾越小越好。
                因为结尾越小，后面越容易接上更大的数。

            注意：
                tail 本身不一定是某个真实的 LIS，
                它只是维护每种长度下最有扩展潜力的结尾值。
        */
        vector<int> tail;

        /*
            依次扫描 nums 中的每个数 x。

            我们要判断 x 可以放到 tail 的哪个位置。
        */
        for (int x : nums) {
            /*
                lower_bound 找到 tail 中第一个 >= x 的位置。

                因为题目要求“严格递增”，所以相等的数不能接在后面。

                如果 tail 中存在某个数 >= x，
                就用 x 替换它。

                这表示：
                    对于这个长度的递增子序列，
                    我们找到了一个更小的结尾 x，
                    后续扩展能力更强。
            */
            auto it = lower_bound(tail.begin(), tail.end(), x);

            /*
                如果没有找到 >= x 的位置，
                说明 x 比 tail 中所有结尾值都大。

                那么 x 可以接在当前最长递增子序列后面，
                使长度增加 1。
            */
            if (it == tail.end()) {
                tail.push_back(x);
            }
            /*
                否则，用 x 替换第一个 >= x 的元素。

                注意：
                    这里不是改变 LIS 的长度，
                    而是优化某个长度下的最小结尾值。
            */
            else {
                *it = x;
            }
        }

        /*
            tail 的长度就是 LIS 的长度。

            因为 tail 中每多一个元素，
            就说明我们成功构造出了更长一位的递增子序列。
        */
        return tail.size();
    }
};
```

## 5.打家劫舍

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，**如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警**。

给定一个代表每个房屋存放金额的非负整数数组，计算你 **不触动警报装置的情况下** ，一夜之内能够偷窃到的最高金额。

```cpp
class Solution {
public:
    int rob(vector<int>& nums) {
        int n = nums.size();

        /*
            dp[i] 表示：
                考虑前 i 间房子时，能够偷到的最大金额。

            注意：
                这里的“前 i 间房子”对应 nums[0] 到 nums[i - 1]。

            所以：
                dp[0] 表示没有房子，最大金额为 0。
                dp[1] 表示只考虑第 0 间房子，最大金额为 nums[0]。
        */
        vector<int> dp(n + 1, 0);

        dp[0] = 0;
        dp[1] = nums[0];

        /*
            从第 2 间房子开始考虑。

            当我们计算 dp[i] 时，实际上是在考虑 nums[i - 1] 这间房子。
        */
        for (int i = 2; i <= n; i++) {
            /*
                对于第 i 间房子 nums[i - 1]，有两种选择：

                1. 不偷它：
                   那么最大金额就是考虑前 i - 1 间房子的结果：
                   dp[i - 1]

                2. 偷它：
                   因为相邻房子不能同时偷，
                   所以前一间 nums[i - 2] 不能偷。
                   因此只能加上考虑前 i - 2 间房子的最大金额：
                   dp[i - 2] + nums[i - 1]

                二者取最大值。
            */
            dp[i] = max(dp[i - 1], dp[i - 2] + nums[i - 1]);
        }

        /*
            dp[n] 表示考虑全部 n 间房子时能偷到的最大金额。
        */
        return dp[n];
    }
};
```

## 6. 不同路径II

二维 DP 写出状态转移后，必须检查边界位置是否也满足这个转移**。很多错不是主思路错，而是边界状态没有单独处理。

```cpp
class Solution {
public:
    int uniquePathsWithObstacles(vector<vector<int>>& obstacleGrid) {
        /*
            题目：不同路径 II

            机器人从左上角 (0, 0) 出发，
            每次只能向右或向下走，
            要到达右下角 (m - 1, n - 1)。

            obstacleGrid[i][j] == 1 表示障碍物，不能经过；
            obstacleGrid[i][j] == 0 表示空地，可以经过。

            要求：从左上角到右下角共有多少条不同路径。
        */

        int m = obstacleGrid.size();
        int n = obstacleGrid[0].size();

        /*
            dp[i][j] 表示：
            从起点 (0, 0) 走到位置 (i, j) 的不同路径数。

            这类二维路径 DP 的核心：
            走到当前位置 (i, j)，最后一步只有两种可能：

            1. 从上方 (i - 1, j) 走下来
            2. 从左方 (i, j - 1) 走过来

            所以如果当前位置不是障碍物：
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1]

            如果当前位置是障碍物：
            dp[i][j] = 0
        */
        vector<vector<int>> dp(m, vector<int>(n, 0)); //全部初始化为0

        /*
            如果起点就是障碍物，那么机器人一开始就无法出发，
            直接返回 0。
        */
        if (obstacleGrid[0][0] == 1) {
            return 0;
        }

        /*
            起点不是障碍物时，
            从起点到起点本身，算作 1 种路径。
        */
        dp[0][0] = 1;

        /*
            初始化第一列。

            第一列的位置只能从上方一路往下走到达，
            没有“从左边来”的可能。

            所以：
            - 如果当前格子不是障碍物，并且上方能到达，那么它也能到达
            - 一旦遇到障碍物，后面的格子都无法到达

            因此遇到障碍物后可以直接 break。
        */
        for (int i = 1; i < m; i++) {
            if (obstacleGrid[i][0] != 1) {
                dp[i][0] = 1;
            } else {
                break;
            }
        }

        /*
            初始化第一行。

            第一行的位置只能从左边一路往右走到达，
            没有“从上方来”的可能。

            同理：
            - 没遇到障碍物之前，路径数都是 1
            - 一旦遇到障碍物，右边的格子都无法到达

            因此遇到障碍物后可以直接 break。
        */
        for (int j = 1; j < n; j++) {
            if (obstacleGrid[0][j] != 1) {
                dp[0][j] = 1;
            } else {
                break;
            }
        }

        /*
            从 (1, 1) 开始填表。

            对于普通位置 (i, j)：
            如果它不是障碍物，那么路径数来自：
            - 上方 dp[i - 1][j]
            - 左方 dp[i][j - 1]

            如果它是障碍物，那么保持 0。
        */
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (obstacleGrid[i][j] != 1) {
                    dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
                }
            }
        }

        /*
            右下角 dp[m - 1][n - 1]
            就是从左上角到右下角的不同路径数。
        */
        return dp[m - 1][n - 1];
    }
};
```

## 7.分割等和子集

```cpp
class Solution {
public:
    bool canPartition(vector<int>& nums) {
        /*
            题目：416. 分割等和子集

            要判断能不能把数组分成两个子集，
            使得两个子集的元素和相等。

            如果总和是 sum，那么两个子集的和都必须是 sum / 2。
            所以问题可以转化为：

                能不能从 nums 中选出一些数，
                使它们的和恰好等于 sum / 2？

            这就是一个典型的 01 背包可行性问题。
        */

        int sum = 0;

        // 先计算数组总和
        for (int x : nums) {
            sum += x;
        }

        /*
            如果总和是奇数，不可能平均分成两个整数和相等的子集。
            例如 sum = 11，就不可能分成 5.5 和 5.5。
        */
        if (sum % 2 == 1) {
            return false;
        }

        // 目标：判断能不能凑出 target
        int target = sum / 2;

        /*
            dp[j] 表示：
            在已经处理过的数字中，能不能选出一些数，使它们的和恰好为 j。

            例如：
            dp[5] == true 表示：
            当前已经看过的这些数里，可以凑出和为 5 的子集。
        */
        vector<bool> dp(target + 1, false);

        /*
            dp[0] = true

            含义：
            什么数都不选，就可以凑出和 0。

            这是背包可行性问题的起点。
        */
        dp[0] = true;

        /*
            依次考虑每一个数字 x。

            对于每个 x，有两种选择：
            1. 不选 x
               那么 dp[j] 保持原来的状态。

            2. 选 x
               如果之前可以凑出 j - x，
               那么现在加上 x，就可以凑出 j。

            所以转移是：

                dp[j] = dp[j] || dp[j - x]

            含义：
            能凑出 j，要么原来就能凑出 j，
            要么原来能凑出 j - x，现在选上 x 后凑出 j。
        */
        for (int x : nums) {
            /*
                注意：这里必须倒序遍历。

                因为每个数字只能使用一次，这是 01 背包。

                如果正序遍历，可能会在同一轮中重复使用当前这个 x。

                比如 x = 3：
                正序时可能先用 x 更新 dp[3]，
                然后又用刚刚更新出来的 dp[3] 去更新 dp[6]，
                这就相当于同一个 3 被用了两次。

                倒序遍历可以保证：
                dp[j - x] 还是上一轮的状态，
                也就是没有使用当前 x 之前的状态。
            */
            for (int j = target; j >= x; j--) {
                dp[j] = dp[j] || dp[j - x];
            }
        }

        /*
            最后看能不能凑出 target。

            如果 dp[target] == true，
            说明可以选出一部分数和为 sum / 2，
            那剩下的数自然也是 sum / 2，
            所以可以分割成两个等和子集。
        */
        return dp[target];
    }
};
```
