# 第2章 连通和遍历

## 2.1 连通和 DFS

这一节的主线是：

```text
连通：图中两个点之间能不能走到。
DFS：用深度优先的方式探索图。
```

假期预习时，这一节最重要的是把 DFS 的代码写熟。很多后续内容都会建立在 DFS 上，例如连通分量、割点、割边、环检测、拓扑排序、强连通分量等。

## 2.1.1 理论

### 路径

在图中，如果可以从顶点 `u` 沿着若干条边走到顶点 `v`，那么这条经过的顶点和边组成一条从 `u` 到 `v` 的路径。

例如：

```text
A ----- B ----- C
```

从 `A` 到 `C` 有路径：

```text
A -> B -> C
```

### 连通

在无向图中，如果顶点 `u` 和顶点 `v` 之间存在路径，就说 `u` 和 `v` 连通。

例如：

```text
A ----- B      C ----- D
```

其中：

```text
A 和 B 连通
C 和 D 连通
A 和 C 不连通
```

### 连通图

如果无向图中任意两个顶点之间都连通，那么这个图叫连通图。

例如：

```text
A ----- B
|       |
D ----- C
```

这是连通图，因为任意两个顶点之间都能找到路径。

而：

```text
A ----- B      C ----- D
```

不是连通图，因为从左边部分走不到右边部分。

### 连通分量

如果一个图不是连通图，它会分成若干个互相走不到的部分。每个“最大的连通子图”叫一个连通分量。

例如：

```text
A ----- B      C ----- D      E
```

它有三个连通分量：

```text
{A, B}
{C, D}
{E}
```

注意：孤立顶点自己也是一个连通分量。

这里“最大”很重要。`{A}` 当然也是连通的，但它不是连通分量，因为它还能扩展成 `{A, B}`。

## 2.1.2 DFS 算法

DFS 是 Depth-First Search，中文叫深度优先搜索。

它的直觉是：

```text
从一个顶点出发；
能往前走就一直往前走；
走不动了，就退回来换另一条路。
```

例如：

```text
A ----- B ----- D
|
C
```

从 `A` 开始 DFS，一种可能的访问顺序是：

```text
A -> B -> D -> C
```

注意：DFS 的访问顺序不唯一，取决于邻接表中邻居的排列顺序。

### DFS 的关键：visited

在图中 DFS 时，一定要记录已经访问过的顶点。

原因是无向图中一条边可以来回走。如果没有 `visited`，可能出现：

```text
A 访问 B
B 又访问 A
A 又访问 B
...
```

这样会无限递归。

因此，DFS 的核心结构是：

```text
访问当前点；
把当前点标记为已访问；
枚举它的邻居；
如果邻居没访问过，就继续 DFS。
```

### 递归版 DFS

```python
def dfs(graph, u, visited):
    visited.add(u)
    print(u)

    for v in graph[u]:
        if v not in visited:
            dfs(graph, v, visited)
```

示例：

```python
graph = {
    "A": ["B", "C"],
    "B": ["A", "D"],
    "C": ["A"],
    "D": ["B"],
}

visited = set()
dfs(graph, "A", visited)
```

可能输出：

```text
A
B
D
C
```

### 非递归版 DFS

递归版 DFS 好理解，但在 Python 中，如果图很大，递归层数太深可能触发递归深度限制。此时可以用栈手动模拟 DFS。

```python
def dfs_iterative(graph, start):
    visited = set()
    stack = [start]

    while stack:
        u = stack.pop()

        if u in visited:
            continue

        visited.add(u)
        print(u)

        for v in graph[u]:
            if v not in visited:
                stack.append(v)

    return visited
```

这里的 `stack` 是栈，满足：

```text
后进先出
```

这正好对应 DFS “先深入”的特点。

## 用 DFS 判断两个顶点是否连通

如果从 `start` 出发 DFS，最后访问到了 `target`，就说明两个顶点连通。

```python
def has_path(graph, start, target):
    visited = set()

    def dfs(u):
        if u == target:
            return True

        visited.add(u)

        for v in graph[u]:
            if v not in visited:
                if dfs(v):
                    return True

        return False

    return dfs(start)
```

示例：

```python
graph = {
    "A": ["B"],
    "B": ["A"],
    "C": ["D"],
    "D": ["C"],
}

print(has_path(graph, "A", "B"))  # True
print(has_path(graph, "A", "C"))  # False
```

## 用 DFS 判断图是否连通

如果从任意一个顶点出发 DFS，可以访问到所有顶点，那么这个图是连通图。

```python
def is_connected(graph):
    if not graph:
        return True

    start = next(iter(graph))
    visited = set()

    def dfs(u):
        visited.add(u)

        for v in graph[u]:
            if v not in visited:
                dfs(v)

    dfs(start)

    return len(visited) == len(graph)
```

示例：

```python
graph1 = {
    "A": ["B"],
    "B": ["A", "C"],
    "C": ["B"],
}

graph2 = {
    "A": ["B"],
    "B": ["A"],
    "C": [],
}

print(is_connected(graph1))  # True
print(is_connected(graph2))  # False
```

注意：如果存在孤立顶点，也要在邻接表中写出来：

```python
graph = {
    "A": ["B"],
    "B": ["A"],
    "C": [],
}
```

否则 Python 字典里没有这个顶点，算法会误以为图中没有它。

## 用 DFS 求连通分量

思路：

```text
遍历所有顶点；
如果某个顶点还没访问过，就从它开始 DFS；
这一次 DFS 访问到的所有顶点就是一个连通分量；
继续找下一个未访问顶点。
```

代码：

```python
def connected_components(graph):
    visited = set()
    components = []

    def dfs(u, component):
        visited.add(u)
        component.append(u)

        for v in graph[u]:
            if v not in visited:
                dfs(v, component)

    for u in graph:
        if u not in visited:
            component = []
            dfs(u, component)
            components.append(component)

    return components
```

示例：

```python
graph = {
    "A": ["B"],
    "B": ["A"],
    "C": ["D"],
    "D": ["C"],
    "E": [],
}

print(connected_components(graph))
```

输出：

```python
[['A', 'B'], ['C', 'D'], ['E']]
```

## 从边集建图后求连通分量

算法题中常给的是边集，而不是邻接表。因此常见流程是：

```text
边集 -> 邻接表 -> DFS
```

```python
def build_graph(n, edges):
    graph = {i: [] for i in range(n)}

    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)

    return graph


def connected_components_from_edges(n, edges):
    graph = build_graph(n, edges)
    return connected_components(graph)
```

示例：

```python
n = 5
edges = [
    [0, 1],
    [2, 3],
]

print(connected_components_from_edges(n, edges))
```

输出：

```python
[[0, 1], [2, 3], [4]]
```

## 复杂度

设图有：

```text
n 个顶点
m 条边
```

使用邻接表时，DFS 会访问每个顶点一次，每条边在无向图中最多被检查两次。

所以时间复杂度为：

```text
O(n + m)
```

空间复杂度主要来自：

- 邻接表
- visited 集合
- 递归调用栈或手动栈

通常记为：

```text
O(n + m)
```

如果只讨论 DFS 额外空间，不算存图，则是：

```text
O(n)
```

## 易错点

### 忘记 visited

无向图中如果不记录访问过的顶点，很容易在两点之间反复访问。

### 忘记孤立顶点

如果图有 `n` 个顶点，即使某个顶点没有边，也要在邻接表中保留它。

例如：

```python
graph = {i: [] for i in range(n)}
```

### 把 DFS 顺序当成唯一

DFS 的访问顺序取决于邻接表中邻居的顺序，不同顺序可能得到不同 DFS 序列。

### 递归深度问题

Python 默认递归深度有限。大图中递归 DFS 可能报错：

```text
RecursionError
```

此时可以使用非递归 DFS，或者在竞赛环境中适当调整递归深度。

## 本节小结

本节重点：

- 两个顶点之间存在路径，则它们连通。
- 任意两个顶点都连通的无向图叫连通图。
- 连通分量是最大的连通子图。
- DFS 是深度优先搜索，核心是 `visited`。
- DFS 可以用来判断两个点是否连通。
- DFS 可以用来判断整个图是否连通。
- DFS 可以用来求所有连通分量。
- 算法题常见流程是：边集建邻接表，再 DFS。

### 1. 1971. 寻找图中是否存在路径

链接：

```text
https://leetcode.cn/problems/find-if-path-exists-in-graph/
```

对应知识点：

```text
无向图、连通性、DFS、并查集
```

### 题面核心

题目给出：

```text
n 个点
edges 边集
source 起点
destination 终点
```

要判断 `source` 能不能走到 `destination`。

图论翻译：

```text
给定一个无向图，判断两个顶点是否在同一个连通分量中。
```

### 解法一：DFS

DFS 的思路是：

```text
1. 先用 edges 建无向邻接表。
2. 从 source 开始深度优先搜索。
3. 如果搜索过程中到达 destination，返回 True。
4. 如果所有能走到的点都搜完了仍没到达，返回 False。
```

正确代码：

```python
class Solution:
    def validPath(
        self, n: int, edges: List[List[int]], source: int, destination: int
    ) -> bool:
        if source == destination:
            return True

        graph = {i: [] for i in range(n)}

        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)

        visited = set()

        def dfs(cur):
            if cur == destination:
                return True

            visited.add(cur)

            for nxt in graph[cur]:
                if nxt not in visited:
                    if dfs(nxt):
                        return True

            return False

        return dfs(source)
```

### DFS 第一版错误复盘

错误一：

```python
if not edges:
    return True
```

这个判断不对。没有边不代表任意两点之间有路径。

例如：

```text
n = 3
edges = []
source = 0
destination = 2
```

这时应该返回 `False`。只有当：

```python
source == destination
```

时，才可以直接返回 `True`。

错误二：孤立顶点可能不在字典中。

如果这样建图：

```python
graph = {}
for u, v in edges:
    graph.setdefault(u, []).append(v)
    graph.setdefault(v, []).append(u)
```

那么没有出现在任何边里的孤立顶点不会进入 `graph`。更稳的写法是：

```python
graph = {i: [] for i in range(n)}
```

这样所有顶点都会被注册进图中，即使它的度为 `0`。

错误三：递归返回值没有层层传回。

错误写法：

```python
dfs(nxt)
```

如果深处找到了 `destination`，返回了 `True`，但外层没有接住这个返回值，最终仍可能返回 `False`。

应该写成：

```python
if dfs(nxt):
    return True
```

错误四：到达目标时要立刻判断。

DFS 函数开头应该有：

```python
if cur == destination:
    return True
```

否则即使已经走到了目标点，代码也不会意识到已经成功。

### 解法二：并查集

这题也可以用并查集。

核心想法：

```text
每条边 u-v 表示 u 和 v 属于同一个连通分量。
把所有边 union 完。
最后判断 source 和 destination 的根是否相同。
```

正确代码：

```python
class Solution:
    def validPath(
        self, n: int, edges: List[List[int]], source: int, destination: int
    ) -> bool:
        father = [i for i in range(n)]

        def find(x):
            if x == father[x]:
                return x

            father[x] = find(father[x])
            return father[x]

        def union(x, y):
            fx = find(x)
            fy = find(y)

            if fx != fy:
                father[fx] = fy

        for u, v in edges:
            union(u, v)

        return find(source) == find(destination)
```

### 并查集第一版错误复盘

错误一：路径压缩不能写在 `return` 里赋值。

错误写法：

```python
return father[x] = find(father[x])
```

Python 不允许在 `return` 后面做这种赋值。

应该拆成两步：

```python
father[x] = find(father[x])
return father[x]
```

错误二：`union` 时要修改 `father` 数组，而不是修改局部变量。

错误写法：

```python
fx = father[fy]
```

这只改变了局部变量 `fx`，没有改变并查集结构。

正确写法：

```python
father[fx] = fy
```

意思是把 `x` 所在集合的根节点挂到 `y` 所在集合的根节点下面。

### 本题心得

这题可以用两种角度理解：

```text
DFS：从 source 出发，看能不能搜到 destination。
并查集：把所有边连接的点合并，最后看两个点是否属于同一集合。
```

DFS 更贴合本节“连通和遍历”的主题；并查集更适合快速判断连通块归属。

这题暴露出的关键代码细节：

```text
1. 图中孤立点也要注册。
2. DFS 搜索答案时，返回值要一层一层传回。
3. DFS 到达目标点时，要立刻返回 True。
4. 并查集路径压缩要先赋值，再 return。
5. union 要改 father 数组，不能只改临时变量。
```

一句话总结：

```text
判断两个点是否连通，可以 DFS 搜，也可以并查集合并后查根。
```

### 2. 547. 省份数量

链接：

```text
https://leetcode.cn/problems/number-of-provinces/
```

对应知识点：

```text
连通分量、邻接矩阵、DFS
```

### 题面核心

题目给的是邻接矩阵 `isConnected`，其中：

```text
isConnected[i][j] = 1 表示城市 i 和城市 j 直接相连
```

题目要返回“省份”的数量。图论翻译就是：

```text
给定一个无向图的邻接矩阵，求连通分量个数。
```

一个省份就是一个连通分量。

### 最终通过代码：直接在邻接矩阵上 DFS

```python
class Solution:
    def findCircleNum(self, isConnected: List[List[int]]) -> int:
        n = len(isConnected)

        ans = 0
        visited = set()

        def dfs(u):
            visited.add(u)

            for v in range(n):
                if isConnected[u][v] == 1 and v not in visited:
                    dfs(v)

        for u in range(n):
            if u not in visited:
                ans += 1
                dfs(u)

        return ans
```

### 思路总结

这题可以直接使用“求连通分量”的模板：

```text
1. 遍历所有顶点。
2. 如果发现一个顶点还没访问过，说明发现了一个新的连通分量。
3. ans += 1。
4. 从这个顶点开始 DFS，把整个连通分量都标记为已访问。
5. 最后 ans 就是连通分量个数。
```

关键句：

```text
DFS 启动了几次，就有几个连通分量。
```

### 邻接矩阵也可以直接做 DFS

一开始的想法是：

```text
邻接矩阵 -> 邻接表 -> DFS
```

这个思路是对的，也能做出来。

但这题已经给了邻接矩阵，矩阵本身就能判断两个顶点是否相邻：

```python
isConnected[u][v] == 1
```

所以不一定要额外建邻接表，可以直接在矩阵上枚举 `u` 的所有可能邻居 `v`：

```python
for v in range(n):
    if isConnected[u][v] == 1 and v not in visited:
        dfs(v)
```

这也是一种 DFS，只是“找邻居”的方式不同：

```text
邻接表 DFS：for v in graph[u]
邻接矩阵 DFS：for v in range(n)，再判断 matrix[u][v] 是否为 1
```

### 第一版思路中的细节问题

第一版想先建邻接表，大方向正确，但要注意：

```python
n = len(isConnected)
```

题目没有单独给 `n`，需要从矩阵长度得到。

另外城市编号是：

```text
0, 1, 2, ..., n - 1
```

所以如果建图，范围应该是：

```python
range(n)
```

不是：

```python
range(n + 1)
```

### 本题心得

这题和 1971 的区别：

```text
1971：判断两个点是否在同一个连通分量。
547：统计整张图有几个连通分量。
```

这题最重要的收获：

```text
邻接矩阵本身就是图的表示方式，可以直接在邻接矩阵上 DFS。
```

一句话总结：

```text
看到“省份数量”，就把它翻译成“连通分量个数”；每从一个未访问点启动一次 DFS，答案就加一。
```

```text
房间 i 里有钥匙 j，就表示 i -> j
```

题目问能否从 `0` 号房间出发访问所有房间。

核心思路：

```text
1. 从房间 0 开始 DFS。
2. 拿到钥匙就访问对应房间。
3. 最后看 visited 的数量是否等于房间总数。
```

虽然有向图在后面章节才系统讲，但这题很直观，可以提前练“从一个点能到达哪些点”。

### 推荐刷题顺序

```text
1971 -> 547 -> 200 -> 695 -> 841
```

如果 323 能打开，可以插在 547 后面：

```text
1971 -> 547 -> 323 -> 200 -> 695 -> 841
```

### 做题时的固定检查清单

每道 DFS 题先问自己：

```text
1. 顶点是什么？
2. 边是什么？
3. 输入是边集、邻接表、邻接矩阵，还是二维网格？
4. 要不要先建图？
5. visited 用 set、list，还是直接修改 grid？
6. DFS 是判断能否到达，还是统计一个连通块？
```

### 这一节最该练出来的代码感觉

```text
看到“能不能到达” -> DFS
看到“分成几块” -> 遍历所有点，多次 DFS
看到“最大一块多大” -> DFS 返回当前连通块大小
看到“矩阵上下左右连通” -> 把格子当顶点
```
