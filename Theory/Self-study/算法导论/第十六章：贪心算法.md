# 《算法导论》第十六章：贪心算法 (Greedy Algorithms) 详细笔记

## 1. 贪心算法的核心哲学
*   **贪心选择性质 (Greedy Choice Property)**：我们可以通过做出**局部最优**（当前最好）的选择来构造全局最优解。
*   **与 DP 的区别**：
    *   **动态规划**：每一步的选择取决于子问题的解。因此通常是**自底向上**计算（先看子问题，再做当前决策）。
    *   **贪心算法**：在做当前选择时，不依赖子问题的解。它通常是**自顶向下**的，做一个选择，然后缩减问题规模。

---

## 2. 经典案例 1：活动选择问题 (Activity-Selection Problem)

这是贪心算法的“开场秀”，展示了如何通过简单的排序和筛选解决复杂的调度问题。

### 问题描述：
有一组活动 $S = \{a_1, a_2, \dots, a_n\}$，每个活动有开始时间 $s_i$ 和结束时间 $f_i$。目标是在同一间教室内安排**尽可能多**的不冲突活动。

### 天才的贪心策略：
**每次都选择结束时间 $f_i$ 最早的活动。**

*   **直觉**：结束得越早，留给后面活动的时间就越多。
*   **算法步骤**：
    1.  按结束时间 $f_i$ 对活动进行**升序排列**。
    2.  选取第一个活动 $a_1$。
    3.  遍历剩下的活动，如果某个活动的 $s_i \ge$ 上一个选中活动的 $f_{last}$，则选中它。

### 为什么它是正确的？
《算法导论》证明了：在所有兼容的活动中，结束时间最早的活动必然属于某个最优解。通过不断的“贪心选择+缩减问题”，最终得到全局最大数量。

---

## 3. 贪心算法的两大要素 (理论支柱)

要证明一个问题可以用贪心算法，必须证明以下两点：

1.  **贪心选择性质**：我们可以通过局部最优选择达到全局最优。
2.  **最优子结构**：一个问题的最优解包含其子问题的最优解。

### 🚨 警示：贪心算法的局限性
贪心算法**并不对所有问题都有效**。如果局部最优导致无法达到全局最优，就必须退回到动态规划。

**经典对比：背包问题 (The Knapsack Problem)**
*   **0-1 背包问题**（物品不能拆分）：**贪心法无效**。你拿了单位价值最高的，可能导致空间浪费，反而不如拿两个次优的。必须用 **DP**。
*   **分数背包问题**（物品可以拆分，如金粉）：**贪心法有效**。直接按单位价值（价值/重量）排序，从高往低拿即可。

## 4. 经典案例 2：赫夫曼编码 (Huffman Codes)

这是贪心算法在**数据压缩**领域的伟大应用，也是你之前学过的“二叉树”和“堆”的完美结合。

### 背景：
如何用最少的二进制位表示一段文本？
*   **定长编码**：每个字母都用 3 位（如 ASCII）。
*   **变长编码**：给频率高的字母短编码（如 'e' 用 `01`），给频率低的字母长编码（如 'z' 用 `1101`）。

### 天才的约束：前缀码 (Prefix Codes)
为了避免歧义（比如 `01` 是 'a'，`011` 是 'b'，那收到 `011` 时该怎么解？），赫夫曼采用了**前缀码**：没有任何一个编码是另一个编码的前缀。这在逻辑上表现为：**所有字母都在二叉树的叶子节点上。**

### 赫夫曼算法逻辑（贪心策略）：
1.  计算每个字符出现的频率。
2.  将所有字符看作单节点树，放入一个**最小优先队列（最小堆）**。
3.  **贪心步骤**：每次从堆中弹出**频率最低**的两个节点，合成为一个新节点，频率为两者之和。
4.  将新节点插回堆。
5.  重复直到堆中只剩一个根节点。

**复杂度**：$O(n \lg n)$，主要花在优先队列的操作上。

## 5. 拟阵理论 (Matroids) —— 贪心的数学根基（进阶内容）

《算法导论》在 16.4 节引入了**拟阵**。这部分数学味很浓，大一学生可以先理解结论：
*   拟阵是一个数学结构，如果一个问题能证明符合“拟阵”的公理，那么**贪心算法一定能在这个问题上找到最优解**。
*   例如：图的最小生成树（MST）问题符合拟阵性质，所以后期的 **Kruskal 算法** 和 **Prim 算法** 都是贪心算法。

## 6. C++ 实现：活动选择 (English Commented)

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

struct Activity {
    int id;
    int start;
    int finish;
};

// Sort activities by finish time (Greedy Choice Property)
bool compareFinish(Activity a, Activity b) {
    return a.finish < b.finish;
}

void activitySelection(vector<Activity>& acts) {
    // 1. Sort by finish time: O(n log n)
    sort(acts.begin(), acts.end(), compareFinish);

    cout << "--- Selected Activities ---" << endl;
    
    // 2. The first activity is always part of an optimal solution
    int n = acts.size();
    if (n == 0) return;

    cout << "Activity ID: " << acts[0].id << " (" << acts[0].start << ", " << acts[0].finish << ")" << endl;
    
    int lastFinish = acts[0].finish;

    // 3. Greedily pick the next compatible activity
    for (int i = 1; i < n; i++) {
        if (acts[i].start >= lastFinish) {
            cout << "Activity ID: " << acts[i].id << " (" << acts[i].start << ", " << acts[i].finish << ")" << endl;
            lastFinish = acts[i].finish;
        }
    }
}

int main() {
    vector<Activity> acts = {
        {1, 1, 4}, {2, 3, 5}, {3, 0, 6}, {4, 5, 7},
        {5, 3, 9}, {6, 5, 9}, {7, 6, 10}, {8, 8, 11},
        {9, 8, 12}, {10, 2, 14}, {11, 12, 16}
    };

    activitySelection(acts);
    return 0;
}
```

## 💡 助教的深度总结与感悟

1.  **贪心的快感**：
    学完 DP 再学贪心，你会有一种“如释重负”的感觉。DP 需要填一张复杂的表，而贪心只需要排个序，然后一个循环走到底。**这种降维打击的效率，是算法优化追求的极致。**

2.  **证明比实现更难**：
    写贪心算法代码只需 5 分钟，但证明这个贪心策略是对的可能需要 5 小时。大一自学时，建议多看书上的**替换证明法（Exchange Argument）**，这是理解贪心逻辑的精髓。

3.  **直觉与陷阱**：
    贪心算法非常依赖直觉，但直觉往往会骗人。例如：找零钱问题。
    *   如果是 [1, 5, 10, 25] 美分，贪心（每次给最大的）永远是对的。
    *   如果是 [1, 3, 4] 分，要找 6 分钱。贪心会给 [4, 1, 1]（三枚），但最优解是 [3, 3]（两枚）。
    *   **结论**：用贪心前，先问问自己：这个局部最优真的不会封死未来的路吗？

## 贪心证明：交换论证

### 1. 为什么贪心里经常出现交换论证

贪心算法的核心问题是：

局部最优选择为什么不会破坏全局最优？

也就是说，当我们说“这一步应该选某个看起来最好的对象”时，必须证明：

存在一个全局最优解，也包含这个贪心选择。

这个证明通常不是直接构造所有最优解，而是用交换论证：

先假设有一个最优解；

如果它已经包含贪心选择，那很好；

如果它不包含贪心选择，就把最优解中的某个部分换成贪心选择；

证明换完之后仍然可行，并且总代价不变差；

于是说明存在一个包含贪心选择的最优解。

这就是贪心选择安全性的常见证明套路。

------

### 2. 交换论证的通用模板

假设我们想证明某个贪心选择 (g) 是安全的。

常见证明结构是：

1. 取一个最优解 (OPT)。
2. 如果 (OPT) 已经包含 (g)，则结论成立。
3. 如果 (OPT) 不包含 (g)，就在 (OPT) 中找到一个可以被 (g) 替换的元素 (x)。
4. 构造新解：

$$
OPT' = OPT-{x}\cup{g}
$$

1. 证明 (OPT') 仍然是可行解。
2. 证明 (OPT') 的质量不比 (OPT) 差。

如果是最大化问题，证明：

$$
value(OPT')\ge value(OPT)
$$

如果是最小化问题，证明：

$$
cost(OPT')\le cost(OPT)
$$

1. 因为 (OPT) 已经是最优解，所以 (OPT') 也一定是最优解。
2. 因此，存在一个包含贪心选择 (g) 的最优解。

这就说明贪心选择是安全的。

------

### 3. 活动选择问题中的交换论证

活动选择问题的贪心选择是：

每次选择结束时间最早的活动。

要证明它安全。

设当前活动集合为 (S_k)，令 (a_m) 是其中结束时间最早的活动。

我们要证明：

存在一个最大规模的兼容活动集合包含 (a_m)。

取一个最优解 (A_k)，令 (a_j) 是 (A_k) 中结束时间最早的活动。

如果：

$$
a_j=a_m
$$

那么结论成立。

如果：

$$
a_j\ne a_m
$$

则构造：

$$
A_k'=A_k-{a_j}\cup{a_m}
$$

因为 (a_m) 是整个集合中结束时间最早的活动，所以：

$$
f_m\le f_j
$$

而 (A_k) 中其他活动都在 (a_j) 之后开始，所以它们也一定能在 (a_m) 之后开始。

因此，把 (a_j) 换成 (a_m) 不会破坏兼容性。

而且：

$$
|A_k'|=|A_k|
$$

所以 (A_k') 也是一个最大规模兼容活动集合，并且包含贪心选择 (a_m)。

因此，选择结束时间最早的活动是安全的。

------

### 4. 活动选择的反向贪心

16.1-2 中的策略是：

每次选择开始时间最晚的活动。

这个策略和“最早结束”是对偶的。

最早结束是从左往右选，给后面的活动留下尽可能多的时间。

最晚开始是从右往左选，给前面的活动留下尽可能多的时间。

证明方式同样是交换论证。

设 (a_m) 是当前集合中开始时间最晚的活动。

取一个最优解 (A_k)，令 (a_j) 是 (A_k) 中开始时间最晚的活动。

如果 (a_j=a_m)，则结论成立。

如果不是，构造：

$$
A_k'=A_k-{a_j}\cup{a_m}
$$

因为 (a_m) 开始时间不早于 (a_j)：

$$
s_m\ge s_j
$$

而 (A_k) 中其他活动都在 (a_j) 开始前结束，所以它们也一定能在 (a_m) 开始前结束。

因此替换后仍然兼容，活动数量不变。

所以也存在一个最优解包含开始时间最晚的活动。

这说明反向贪心也是安全的。

------

### 5. 分数背包中的交换论证

分数背包的贪心选择是：

优先拿单位价值最高的物品。

单位价值为：

$$
\frac{v_i}{w_i}
$$

要证明这个选择安全。

设物品 (a) 的单位价值最高：
$$
\frac{v_a}{w_a}\ge \frac{v_b}{w_b}
$$
对于任意其他物品 (b)。

如果某个最优解没有尽可能多地拿 (a)，却拿了一部分单位价值更低的物品 (b)，那么我们可以做交换：

从 (b) 中拿掉一小部分重量 (\Delta)，换成同样重量的 (a)。

总重量不变。

价值变化为：

$$
\Delta\cdot \frac{v_a}{w_a}-\Delta\cdot \frac{v_b}{w_b}
$$

因为：

$$
\frac{v_a}{w_a}\ge \frac{v_b}{w_b}
$$

所以价值不会下降。

因此存在一个最优解，它尽可能多地选择单位价值最高的物品。

这就证明了分数背包具有贪心选择性质。

### 6. Huffman 编码中的交换论证

Huffman 编码的贪心选择是：

每次选择频率最低的两个字符合并。

要证明这个选择安全，需要证明：

频率最低的两个字符可以出现在某棵最优前缀编码树的最深层，并且互为兄弟。

设 (x,y) 是频率最低的两个字符。

取一棵最优前缀编码树 (T)。

在 (T) 中找一对最深的兄弟叶子，记为 (a,b)。

如果 (x,y) 已经在这个位置，则结论成立。

如果没有，就把 (x) 和 (a) 交换，把 (y) 和 (b) 交换。

为什么交换不会变差？

编码总代价为：

$$
B(T)=\sum_{c\in C}c.freq\cdot d_T(c)
$$

其中 (d_T(c)) 是字符 (c) 在树中的深度，也就是码字长度。

先交换 (x) 和 (a)。

只有这两个字符的贡献发生变化，所以：

$$
B(T)-B(T')
=
(a.freq-x.freq)(d_T(a)-d_T(x))
$$

因为 (x) 是最低频字符之一，所以：

$$
a.freq\ge x.freq
$$

又因为 (a) 在最深层，所以：

$$
d_T(a)\ge d_T(x)
$$

因此：

$$
B(T)-B(T')\ge 0
$$

即：

$$
B(T')\le B(T)
$$

交换后总代价不会增加。

同理交换 (y) 和 (b)，总代价也不会增加。

由于原树 (T) 已经是最优树，所以交换后的树仍然是最优树。

于是存在一棵最优树，其中最低频的两个字符位于最深层且互为兄弟。

这就证明了 Huffman 的贪心选择安全。

### 7. Huffman 16.3-5 中的交换论证

题目要求证明：

如果字符按频率单调递减排列，那么存在一个最优编码，使得码字长度单调递增。

也就是高频字符的码字长度不比低频字符更长。

设：

$$
c_i.freq\ge c_j.freq
$$

但在某棵最优树中却有：

$$
d_T(c_i)>d_T(c_j)
$$

也就是说，高频字符 (c_i) 反而比低频字符 (c_j) 更深。

交换 (c_i) 和 (c_j) 的位置，得到新树 (T')。

比较代价：

$$
B(T)-B(T')
=
(c_i.freq-c_j.freq)(d_T(c_i)-d_T(c_j))
$$

因为：

$$
c_i.freq\ge c_j.freq
$$

且：

$$
d_T(c_i)>d_T(c_j)
$$

所以：

$$
B(T)-B(T')\ge 0
$$

也就是：

$$
B(T')\le B(T)
$$

交换后总代价不会增加。

不断消除这种“高频字符在低频字符下面”的反序，就能得到一棵最优树，使得频率越高，码字长度越短或相等。

### 8. 交换论证的本质

交换论证其实是在证明：

当前的贪心选择不会比某个最优解中的对应选择更差。

如果最优解没有采用贪心选择，我们就把它改成采用贪心选择。

只要能保证：

可行性不破坏；

目标值不变差；

就说明贪心选择可以出现在某个最优解中。

因此，贪心选择是安全的。

------

### 9. 交换论证常见句式

写证明时可以套这个语言：

设 (OPT) 是一个最优解。

如果 (OPT) 已经包含贪心选择，则结论成立。

否则，设 (x) 是 (OPT) 中与贪心选择 (g) 对应的元素。

构造：

$$
OPT'=OPT-{x}\cup{g}
$$

下面证明 (OPT') 仍然可行。

再证明 (OPT') 的目标值不差于 (OPT)。

因此 (OPT') 也是最优解，并且包含贪心选择 (g)。

所以存在一个包含贪心选择的最优解，贪心选择是安全的。

------

### 10. 什么时候想到交换论证

看到下面这些情形时，可以优先想到交换论证：

题目让你证明一个贪心策略正确；

贪心选择看起来像“最大/最小/最早/最晚/最高频/最低频”；

你需要证明“存在一个最优解包含这个选择”；

问题中某个选择可以被另一个选择替换，并且不破坏可行性；

目标函数可以比较交换前后的变化。

典型例子：

活动选择：替换成结束更早的活动；

反向活动选择：替换成开始更晚的活动；

分数背包：用单位价值更高的物品替换单位价值更低的物品；

Huffman 编码：把低频字符换到更深的位置；

编码长度排序：把高频字符换到更浅的位置。

------

### 11. 交换论证和反例的关系

贪心策略不是靠直觉就能成立的。

比如活动选择中，下面这些策略看起来也合理，但都不安全：

选择持续时间最短的活动；

选择开始时间最早的活动；

选择与其他活动重叠最少的活动。

这些策略失败的原因是：

你无法证明任意最优解都能通过交换变成包含该选择的最优解。

甚至可以构造反例，使贪心第一步选择之后，最优解大小直接变小。

因此：

> 能不能做交换论证，是判断贪心策略是否可靠的重要信号。

------

### 12. 一句话总结

贪心算法的证明经常依赖交换论证。

交换论证的核心是：

先拿一个最优解，如果它没有贪心选择，就把其中某个部分换成贪心选择，并证明换完之后仍然可行且不变差。

如果能做到，就说明存在一个包含贪心选择的最优解，贪心选择是安全的。

所以贪心不是“凭感觉选最好的”，而是要证明：
$$
\text{局部最优选择可以被嵌入某个全局最优解。}
$$
