# 图算法 

### 一、 DFS (Depth First Search)：执着的迷宫探索者

**1. 物理直觉：**
想象你在走一个复杂的迷宫。DFS 的策略是：**“不撞南墙不回头”**。
*   你从起点出发，选定一条路一直往下走。
*   直到走到死胡同，或者发现这块地儿已经走过了，你才会**回溯（Backtrack）**到上一个路口，选另一条路再试。

**2. 核心工具：栈 (Stack) / 递归**
*   DFS 天然适合用**递归**来实现，因为递归本身就是利用系统栈来记录“回家的路”。

**3. 代码模板：**
```cpp
bool visited[MAXN]; // 标记数组，防止无限循环

void dfs(int u) {
    visited[u] = true; // 1. 标记当前点已访问
    cout << u << " ";  // 2. 执行操作（比如打印）

    for (int v : adj[u]) { // 3. 遍历所有邻居
        if (!visited[v]) { // 4. 如果邻居没去过
            dfs(v);        // 5. 递归潜入
        }
    }
}
```

### 二、 BFS (Breadth First Search)：扩散的涟漪

**2. 物理直觉：**
想象你往平静的湖面投了一块石头。BFS 的策略是：**“地毯式搜索”**。
*   你先访问离起点最近的一层点（1步能到的）。
*   再访问第二层点（2步能到的）……
*   **BFS 的核心优势：** 在无向图中，它第一次搜到某个点时，所经过的路径一定是**最短路径**。

**2. 核心工具：队列 (Queue)**
*   先进先出 (FIFO)。先发现的先排队，先排队的先处理邻居。

**3. 代码模板：**
```cpp
void bfs(int start) {
    queue<int> q;
    q.push(start);
    visited[start] = true;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        cout << u << " ";

        for (int v : adj[u]) {
            if (!visited[v]) {
                visited[v] = true; // 入队时立即标记，防止重复入队
                q.push(v);
            }
        }
    }
}
```

# Dijkstra 单源最短路径算法

## 1. 算法定义与适用场景

*   **定义**：给定一个带权图 $G=(V, E)$ 和源点 $s$，计算从 $s$ 到所有其他节点的最短距离。
*   **适用条件**：
    1.  **带权图**（边有长度）。
    2.  **权重必须为非负数**（这是 Dijkstra 的阿喀琉斯之踵，负权边请使用 Bellman-Ford 或 SPFA）。
*   **核心思想**：**贪心 (Greedy)** + **松弛 (Relaxation)**。

---

## 2. 核心原理

1.  **贪心策略**：每次从“未确定最短路径”的点集中，选出当前距离起点最近的点 $u$。此时，从起点到 $u$ 的最短路径已经确定，再也不会被更新。
2.  **松弛操作 (Relaxation)**：以 $u$ 为中转站，检查是否可以通过 $u$ 让起点到其邻居 $v$ 的距离变短。
    *   公式：`if (dist[u] + weight(u, v) < dist[v]) { dist[v] = dist[u] + weight(u, v); }`

---

## 3. 算法实现演进

### 版本 A：基础实现 (朴素版)

*   **思路**：使用两个嵌套循环，外层循环 $N$ 次，内层循环 $O(N)$ 寻找最小值。
*   **复杂度**：$O(N^2)$。
*   **适用场景**：稠密图（边数 $M \approx N^2$），或者 $N < 5000$ 的情况。

```cpp
// 核心逻辑摘录
for (int i = 1; i <= n; i++) {
    int u = -1, min_d = INF;
    for (int j = 1; j <= n; j++) { // O(N) 寻找最小点
        if (!visited[j] && dist[j] < min_d) {
            min_d = dist[j]; u = j;
        }
    }
    if (u == -1) break;
    visited[u] = true;
    for (auto& edge : adj[u]) { // 松弛邻居
        if (dist[u] + edge.second < dist[edge.first]) 
            dist[edge.first] = dist[u] + edge.second;
    }
}
```

### 版本 B：堆优化实现 (完全体)

*   **思路**：使用 **优先队列 (std::priority_queue)** 维护 `dist` 数组的最小值，将寻找最小值的时间从 $O(N)$ 降至 $O(1)$（维护开销 $O(\log N)$）。
*   **复杂度**：$O(M \log N)$ 或 $O(M \log M)$。
*   **适用场景**：稀疏图（$M \ll N^2$），节点数可达 $10^5 \sim 10^6$ 级别。

```cpp
// 核心逻辑摘录
priority_queue<pair<long long, int>, vector<pair<long long, int>>, greater<pair<long long, int>>> pq;
pq.push({0, start});
while (!pq.empty()) {
    long long d = pq.top().first; int u = pq.top().second; pq.pop();
    if (d > dist[u]) continue; // 关键：跳过过时信息
    for (auto& edge : adj[u]) {
        if (dist[u] + edge.second < dist[edge.first]) {
            dist[edge.first] = dist[u] + edge.second;
            pq.push({dist[edge.first], edge.first});
        }
    }
}
```

## 4. 深度思考 (拔尖班面试必考)

### Q1：为什么 Dijkstra 不能处理负权边？

*   **逻辑漏洞**：Dijkstra 的基石是“一旦一个点被标记为 `visited`，它的最短路径就确定了”。如果存在负权边，这个前提会瓦解。
*   **反例**：$A \to B$ 权重 2，$A \to C$ 权重 5，$B \to C$ 权重 -4。
    *   Dijkstra 第一步会选 $B$（距离2），第二步选 $C$（距离5，且标记为已确定）。
    *   实际上，通过 $B$ 到 $C$ 的距离是 $2 + (-4) = -2$，比 5 小。但 $C$ 已经被标记过，不会再被更新。

### Q2：如何记录最短路径的具体走法？

*   **方法**：增加一个 `parent[]` 数组。
*   **操作**：每当松弛成功时，记录 `parent[v] = u;`。
*   **回溯**：从终点开始，沿着 `parent` 指针倒着找，直到起点。

### Q3：Dijkstra 与 Prim 算法的关系？

*   两者都是**贪心算法**，结构惊人相似。
*   区别在于：
    *   **Dijkstra** 维护的是“该点到**起点**的距离”。
    *   **Prim** 维护的是“该点到**当前生成树**的距离”。

## 5. 工程实践建议 (GitHub 积累规范)

1.  **类型安全**：处理路径长度时，优先使用 `long long` 以防累加后溢出。
2.  **防御性编程**：在松弛前检查 `dist[u] != INF`，防止在有向图断连时发生 `INF + weight` 的溢出。
3.  **封装性**：将算法逻辑封装在函数中，输入输出保持独立，方便以后直接复用。
